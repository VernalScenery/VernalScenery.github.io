1 核心思想

决策树的本质是一个贪心算法，它通过在每一步都选择信息增益最大（相当于“梯度”）的特征来进行分裂，从而沿着一条“类梯度下降”的路径，来逐步降低整个系统的信息熵（相当于“损失函数”）。

信息增益：知道一个变量 X 后，另一个变量 Y 不确定性减少的量

决策树的构造

  • **根节点 (Root Node)**：树的最顶端，代表第一个要判断的问题（比如上图的“年收入”）。
  • **内部节点 (Internal Node) / 决策节点 (Decision Node)**：树中间的节点，代表一个用于判断的特征或属性。
  • **分支 (Branch)**：从一个节点延伸出的路径，代表一个判断问题的可能答案（比如“是”或“否”）。
  • **叶节点 (Leaf Node) / 终端节点 (Terminal Node)**：树的最底端，代表最终的决策结果或预测值（比如“批准贷款”、“拒绝贷款”）。

2 CART 算法

CART 是“Classification And Regression Tree”的缩 Omega，即分类与回归树。
顾名思义，CART 算法最核心的特点是它既可以解决分类问题，也可以解决回归问题，并且它生成的决策树都是二叉树。

无论用于分类还是回归，CART 的核心思想都是一样的：递归地将数据集二分。

它从根节点开始，包含所有训练样本。然后，它会遍历每一个特征的每一个可能的取值，尝试将数据集分裂成两个子集，并找到那个最佳的分裂点。分裂后，这个过程会在新的子节点上重复进行，直到满足某个停止条件为止。最终，整个特征空间被划分为一系列不重叠的矩形区域，每个区域对应一个叶子节点。

这个过程的关键在于，如何定义“最佳的”分裂？这取决于我们处理的是分类问题还是回归问题。

2.1 CART 用于分类 (Classification)

当用于分类问题时，CART 算法的目标是让分裂后的子节点尽可能地“纯净”（即子节点中的样本尽可能属于同一个类别）。它使用基尼不纯度（Gini Impurity作为衡量纯净度的标准。

对于一个节点，CART 会遍历所有特征以及特征的所有可能取值，将数据集 D 分成两个子集 $D_left$​ 和 $D_right$​。然后计算分裂后的基尼不纯度：

2.2 CART 用于回归 (Regression)

当用于回归问题时，CART 算法的目标是预测一个连续值（如房价、气温）。此时，我们不再关心节点的“纯度”，而是关心节点内样本的离散程度。它使用平方误差（Squared Error） 作为度量标准。

对于一个节点，其叶子节点的预测值通常是该节点内所有样本目标值的平均值。而平方误差就是衡量该节点内所有样本值与其平均值的差距。

4 过拟合：减枝

进行剪枝主要有两个核心目的：

  • **降低模型复杂度 (Reduce Model Complexity)**：一个未经剪枝的决策树可能会生长得非常“茂盛”，包含大量细节化的分支。剪枝可以去除一些分支，使模型变得更简单、更易于解释。
  • 解决过拟合 (Solve Overfitting)：这是剪枝最重要的目的。如果决策树过于复杂，它会学习到训练数据中特有的噪声和细节，导致模型在训练集上表现完美，但在新的、未见过的数据（测试集）上表现很差。这种现象就是过拟合。剪枝通过简化模型，提升了模型对新数据的泛化能力。

4.1 预剪枝（Pre-Pruning）

预剪枝是一种“悲观”或“谨慎”的策略。它在决策树生长过程中，对每个节点在分裂前都进行评估。如果当前节点的分裂不能带来泛化性能的提升，或者达到了某种预设的限制，就立即停止该节点的分裂，将其设为叶节点。

简单来说，就是“边建边剪”，提前刹车。

具体实现上：在决策树的生长过程中，通过一系列预先设定的超参数（Hyperparameters） 来提前终止其生长，从而限制树的复杂度。

scikit-learn 中常见的 API:

  • max_depth: 最大深度。限制决策树可以生长的最大层数。深度越小，模型越简单。
  • min_samples_split: 最小分裂样本数。规定一个节点必须要有多少个样本才能被允许进行分裂。如果一个节点的样本数少于这个阈值，它就不会再分裂，直接成为叶节点。
  • min_samples_leaf: 叶节点最小样本数。规定一个分裂操作后，产生的每个子节点（叶节点）中必须至少包含的样本数量。这可以防止产生一些样本量极少的、过于特殊的决策规则。
  • max_leaf_nodes: 最大叶节点数。直接限制最终生成的树最多能有多少个叶节点。
  • min_weight_fraction_leaf: 叶节点最小权重分数。与 min_samples_leaf 类似，但它是基于样本权重总和的比例来计算的，在处理带有权重的样本时非常有用。
  • 优点
    • 高效快速: 由于在树的生长过程中就不断剪枝，预剪枝使得许多不必要的分支根本没有机会生成。这大大降低了模型的训练时间和空间开销。
  • 缺点
    • 目光短浅（贪心本质）: 预剪枝最大的问题是其贪心的本质。某个节点当前的分裂可能无法立即带来性能提升，但它后续的再分裂却可能带来显著的性能提升。预剪枝策略可能会因为眼前的“无收益”而过早地终止了这条极具潜力的分支，从而导致“欠拟合”的风险。

4.2 后剪枝（Post-Pruning）

后剪枝是一种“乐观”的策略。它首先假设所有分裂都是有益的，先生成一棵尽可能完整、甚至有些过拟合的决策树。然后，再从下往上，系统性地考察每个非叶节点，评估如果将该节点下的子树替换为一个叶节点（即“剪掉”这个分支），是否能提升模型的泛化性能。

简单来说，就是“先建后剪”，秋后算账。

具体实现上：后剪枝通常需要一个独立的验证集来评估泛化性能。最经典的后剪枝算法是代价复杂度剪枝（Cost-Complexity Pruning, CCP），这也是 CART 树采用的方法。

  • 优点
    • 全局视野: 后剪枝是在一棵完全长成的树上进行的，它克服了预剪枝“目光短浅”的缺点，保留了更多有潜在价值的分支。通常情况下，后剪枝决策树的欠拟合风险较小。
    • 性能更优: 由于它基于全局信息和验证集性能进行剪枝，所以通常能够找到比预剪枝性能更好的子树，泛化能力更强。
  • 缺点
    • 计算成本高: 后剪枝需要先生成一棵完整的树，然后再自下而上地进行计算和评估，这个过程的训练开销要远大于预剪枝。

5 决策树的优缺点

方面 要点 详细说明与解决方案
优点 (Advantages) 1. 可解释性强 模型结构是类似 if-then 规则的流程图，决策路径清晰，非常易于人类理解和向他人解释。
2. 数据预处理要求低 无需进行特征归一化或标准化；可以同时处理数值型和分类型数据。
3. 灵活性高（非参数模型） 不对数据关系做预先假设，能够学习和表达复杂的非线性关系。
4. 预测速度快 一旦模型构建完成，对新数据的预测过程只需沿着树的路径进行简单比较，效率很高。
缺点 (Disadvantages) 1. 容易过拟合 倾向于生成过于复杂的树来完美匹配训练数据（包括噪声），导致泛化能力差。
解决方案：进行剪枝（预剪枝或后剪枝）。
2. 模型不稳定（高方差） 训练数据的微小变化可能导致生成一棵完全不同的树，模型结果难以复现。
解决方案：使用集成学习（Ensemble Learning），如随机森林、梯度提升树。
3. 贪心算法局限性 采用局部最优策略（如信息增益最大）来构建树，这无法保证能得到全局最优的决策树。
4. 难以学习某些关系 决策边界与坐标轴平行，因此很难高效地拟合简单的线性关系或对角线关系。
