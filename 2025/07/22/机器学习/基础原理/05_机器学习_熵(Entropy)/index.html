1 什么是熵 (Entropy)？

熵（shāng），一个在物理学、信息论乃至更广泛领域都至关重要的概念，其核心思想是衡量一个系统的混乱程度或不确定性。简单来说，系统越混乱、越无序，其熵值就越高；反之，系统越有序、越确定，熵值就越低。

这个概念最初由 19 世纪的物理学家鲁道夫·克劳修斯提出，用于描述热力学系统中能量的退化和不可用性。如今，熵的内涵已远超热力学范畴，成为理解自然演化、信息传播等众多现象的基石。

20 世纪，克劳德·香农将熵的概念引入信息论，用以衡量信息的不确定性。在这里，信息熵代表了一个随机事件或一个信息源所含有的平均信息量。

一个事件的发生概率越低，它所包含的信息量就越大。例如，“太阳从东边升起”这句话的信息量很低，因为它几乎是必然事件。而“明天会下彩虹雨”这句话的信息量就极高，因为它极不确定。

2 第一类：针对单个随机变量的度量

2.1 信息熵 (Information Entropy, H(p))

它衡量的是单一概率分布 p 的不确定性或混乱程度。一个分布越混乱、越不可预测，其信息熵就越高。

  • $p(x_i​)$: 这是事件 $x_i$​ 发生的概率。
  • $[−log2​p(x_i​)]$: 这部分可以被理解为事件 $x_i$​ 发生时，所带来的“信息量”或“惊讶程度”。

为什么用对数，还是负的对数？

  • 概率越小，信息量越大：一件非常不可能发生的事情（概率 p 很小）一旦发生了，会让我们非常惊讶，我们从中获得的信息量就很大。log 函数正好满足这个特性，当 $p(x_i​)$ 趋近于 0 时，$logp(x_i​)$ 趋近于负无穷。
  • 信息量必须是正数：概率 $p(x_i​)$ 是一个 0 到 1 之间的数，所以它的对数 $log2​p(x_i​)$ 是小于等于 0 的。为了让“信息量”成为一个正数，我们在前面加上一个负号，变成了 $−log2​p(x_i​)$。
  • 两个独立事件的信息量应该相加：假设你有两个独立的事件 A 和 B，它们同时发生的概率是 $p(A)⋅p(B)$。我们直观上认为，接收到这两个独立事件信息的总信息量，应该是它们各自信息量之和，即 $I(A and B)=I(A)+I(B)$。只有对数函数 log 满足这个性质：$log(ab)=log(a)+log(b)$。这就是选择对数函数最关键的数学原因。

现在我们理解了 $[−log2​p(x_i​)]$ 是单个事件 $x_i$​ 发生所带来的信息量

用每个结果的‘发生概率’去加权该结果的‘信息量’，最终得到整个系统信息量的平均期望。

这里的信息量也可以理解为 不确定性，概率越小的事件，不确定越大，然后对其加权求平均

信息熵的本质定义就是：一个随机变量所有可能结果带来的信息量的数学期望。

2.2 交叉熵 (Cross Entropy, L(p, q))

交叉熵 L(p, q) 是衡量你的预测 q 与现实 p 之间差距的一种方式。你的预测越不准，你需要付出的“额外成本”（编码长度）就越多，交叉熵的值就越大。

信息论有一个基本原则：给概率越高的事件分配越短的编码，总成本（平均编码长度）就越低。

  • p(xᵢ)：现实。即第 i 个事件实际发生的概率。它在这里充当权重。
  • -log q(xᵢ)：成本。即根据你的预测 q，为第 i 个事件编码所需要的长度。你预测的概率 q 越小，这个编码长度就越长，成本越高。

交叉熵衡量的是，当我们用一个预测的概率分布 q 来编码来自真实的概率分布 p 的事件时，所需要的平均成本（或编码长度）。

  • 成本越高（即交叉熵值越大），代表模型的预测分布 q 与真实的分布 p 之间的差距越大，因此模型的预测就越不准确。

2.3 相对熵 (Relative Entropy / KL 散度, D_KL(p||q))

相对熵可以由交叉熵和信息熵导出：

  • 信息熵 H(p) — 理论上的最低成本
  • 交叉熵 H(p,q) — 你实际付出的总成本

D_KL(p||q) 衡量的就是，当我们用预测分布 q 去近似真实分布 p 时，我们多付出的额外成本，亦或损失了多少信息

3 第二类：描述多个（以两个为例）随机变量之间关系的度量

3.1 联合熵 (Joint Entropy, H(X, Y))

联合熵衡量的是多个（这里以两个为例）随机变量共同组成的一个系统的总体不确定性。

如果说信息熵 $H(X)$ 是关于变量 X 的不确定性，那么联合熵 $H(X,Y)$ 就是关于 $(X, Y)$ 这一对 变量整体的不确定性。

  • $p(x,y)$：这是联合概率，即事件 X=x 和事件 Y=y 同时发生的概率。
  • $−logp(x,y)$：这是 $(x, y)$ 这一对事件同时发生所带来的信息量。
  • 双重求和符号：表示遍历所有可能的 x 和 y 的组合，将它们各自的信息量与其发生概率相乘后全部加起来，求得整个系统信息量的期望。

核心思想：联合熵的计算方式和信息熵几乎一样，只是把研究对象从单个事件的概率 $p(x)$ 换成了**一对事件同时发生的联合概率 $p(x,y)$**。

举个例子： 假设我们不仅关心“明天是否下雨”（变量 X），还关心“明天是否有风”（变量 Y）。

  • **信息熵 H(X)**：只衡量“下雨”这件事的不确定性。
  • **信息熵 H(Y)**：只衡量“有风”这件事的不确定性。
  • **联合熵 H(X, Y)**：衡量“明天天气状况（下雨情况和刮风情况）”这一整个系统的总不确定性。它考虑了所有可能的天气组合，如（晴，无风）、（晴，有风）、（雨，无风）、（雨，有风）。

3.2 条件熵 (Conditional Entropy, H(Y|X))

条件熵衡量的是，在已知一个随机变量 X 的情况下，另一个随机变量 Y 还剩下多少不确定性。它描述的是“消息中的消息”。

它的名字已经很好地解释了它的含义：“条件”+“熵”。

  • $p(y∣x)$：这是条件概率，即在事件 $X=x$ 发生的条件下，事件 $Y=y$ 发生的概率。
  • $logp(y∣x)$：是基于条件 x 下，事件 y 发生的信息量。
  • 整个公式计算的是“在 X 确定的情况下，Y 的不确定性”的数学期望。

核心思想：条件熵不是在单一条件下计算的，而是对所有可能的条件进行遍历，然后将每个条件下 Y 的不确定性，用该条件发生的概率进行加权，最后求和。

举个例子： 继续用天气的例子。假设我们想知道“下雨”这件事的不确定性。

  • $H(\text{是否下雨})$: 这是无任何前提下的不确定性。
  • $H(\text{是否下雨}∣\text{季节})$: 这就是条件熵。它衡量的是，当我们知道了现在是什么季节（比如夏天或冬天）之后，“是否下雨”这件事还剩下的不确定性。
    • 直观上，如果我知道现在是多雨的夏季，那么对于“是否下雨”这件事的猜测就变得更确定了（更倾向于会下雨），其不确定性（熵）就降低了。
    • 条件熵衡量的就是这种“知道一个信息后，对另一个信息不确定性降低”的平均程度。

log 左边乘的是 p(x,y) 为啥不是 p(y∣x)

我们先回顾一下熵的本质：信息量的数学期望。

所以，为了计算出整个系统中关于 Y 的条件不确定性的总期望，我们必须将每个 $(x,y)$ 组合所对应的信息量的值 $[−logp(y∣x)]$，乘以它在全局发生的真实概率 p(x,y)。

3.3 互信息 (Mutual Information, I(X, Y))

互信息衡量的是两个随机变量之间共享的信息量，或者说一个变量中包含的关于另一个变量的信息有多少。

它可以看作是：知道一个变量 X 后，另一个变量 Y 不确定性减少的量。

方式一（用熵的加减关系定义）：

  • $H(Y)$: 变量 Y 的原始不确定性。
  • $H(Y∣X)$: 知道了变量 X 之后，Y 剩下的不确定性。
  • $I(X,Y)$: 两者相减，得到的就是因为知道了 X 而消除掉的关于 Y 的不确定性。这个被消除掉的部分，就是 X 和 Y 共享的信息。

方式二（用联合概率和边缘概率计算）：

核心思想：互信息是两个变量之间关联程度的度量。

  • 如果 $I(X,Y)$ 很大，说明 X 和 Y 的相关性很强。知道了 X，就能很大程度上消除 Y 的不确定性。
  • 如果 $I(X,Y)=0$，说明 X 和 Y 是相互独立的。知道了 X 对猜测 Y 没有任何帮助。

举个例子：

  • 变量 X：学生花在学习上的时间
  • 变量 Y：学生的考试成绩
  • $H(Y)$: 考试成绩本身的不确定性。
  • $H(Y∣X)$: 如果我们知道了学生花了大量时间学习，那么对他成绩的猜测就变得更确定了（大概率会考得不错），此时成绩的“剩余不确定性”就降低了。
  • $I(X,Y)$: “学习时间”和“考试成绩”之间的互信息，就量化了这两个变量的关联程度。

在机器学习的决策树算法中，信息增益 (Information Gain) 这个概念，其本质就是互信息。算法每次选择特征进行分裂，就是为了选择那个能提供最大信息增益（即最大互信息）的特征，从而最有效地降低系统的不确定性。
