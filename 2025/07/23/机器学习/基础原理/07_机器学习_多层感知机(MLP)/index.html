1 核心思想

1.1 为什么需要多层 ？—— 单层感知机的局限性

最早的神经网络模型是单层感知机，它只有一个输入层和一个输出层。它本质上是一个线性分类器，意味着它只能在数据空间里画一条直线（或高维度的超平面）来进行分割。

这导致它只能解决线性可分的问题，比如逻辑“与”（AND）和“或”（OR）。但是，对于线性不可分的问题，单层感知机就无能为力了。最著名的例子就是异或问题（XOR）。

如上图，你永远无法用一条直线将 (0,1)、(1,0) 与 (0,0)、(1,1) 分开。这个看似简单的问题在早期 AI 发展中造成了巨大瓶颈，也直接催生了多层感知机的诞生。

1.2 多层感知机的架构

从结构上看，一个多层感知机由三部分组成：

  • 输入层 (Input Layer): 接收外部输入的数据，其节点数量通常与待处理数据的特征维度相等。
  • 隐藏层 (Hidden Layer): 位于输入层和输出层之间，是 MLP 的核心所在。可以有一个或多个隐藏层，每个隐藏层由若干个神经元（或称为节点）组成。正是这些隐藏层的存在，赋予了 MLP 学习非线性特征的能力。
  • 输出层 (Output Layer): 产生最终的计算结果，输出层的结构取决于具体的任务：
    • 回归任务（如预测房价）：通常只有一个神经元，且不使用（或使用线性）激活函数。
    • 二分类任务（如判断是否为垃圾邮件）：通常只有一个神经元，并使用 Sigmoid 激活函数，输出一个 0 到 1 之间的概率值。
    • 多分类任务（如识别图片是猫、狗还是鸟）：通常有 N 个神经元（N 为类别数），并使用 Softmax 激活函数，输出所有类别的概率分布。

1.3 MLP 的“秘密武器”：非线性激活函数

如果仅仅是堆叠网络层，而没有激活函数，那么多层网络和单层网络在数学上是等价的（多个线性变换的组合仍然是线性变换）。

MLP 之所以能够解决非线性问题，其真正的“秘密武器”是在每个隐藏层的神经元上都应用了一个非线性激活函数（Non-linear Activation Function）。

  • 作用：为模型引入非线性能力，使其能够学习和拟合复杂的曲线边界。
  • 常用函数：Sigmoid、Tanh，以及目前最流行的 ReLU (Rectified Linear Unit) 及其变体。

神经网络的“魔法”不在于它能学习到一个多么复杂的决策边界，而在于它能通过逐层对特征空间进行非线性变换，将一个原本线性不可分的问题，转化为一个在高维空间里线性可分的新问题，最后用一个简单的线性分类器就能轻松解决。

神经网络，尤其是深度神经网络，其最核心、最强大的能力，正是通过其多层次的神经元结构，自动地、从数据中学习和发现最优的特征工程。

  • 隐藏层 (Hidden Layer) 的激活函数：引入非线性，提升模型的表达能力。
  • 输出层 (Output Layer) 的激活函数：将网络最后一层的原始输出值，转换成符合任务需求的、有特定意义的格式。
层 (Layer) 激活函数的主要目的 常用选择
隐藏层 (Hidden Layer) 引入非线性，让网络能学习复杂模式。 ReLU (最常用)、Leaky ReLU、Tanh 等。
输出层 (Output Layer) 格式化输出，使其符合特定任务的要求。 Softmax (多分类)、Sigmoid (二分类)、无激活 (回归)。

2 激活函数

激活函数在神经网络中有两个主要作用：

  1. 引入非线性，提升模型表达能力：这是其核心作用。它使得神经网络能够学习数据中复杂的非线性模式，成为一个真正的“通用函数逼近器”。
  2. 控制信息流动，决定神经元是否“激活”：激活函数可以看作是一个“阀门”或“开关”。它根据输入信号的强度，来决定当前神经元应该在多大程度上被激活（即输出信号的强度），以及是否应该将信息传递给下一层。

如果没有激活函数 f，一个神经元的输出就是其输入的加权和 y = w₁x₁ + w₂x₂ + ... + b。这是一种线性变换。

当我们在每个神经元的加权和之后，应用一个非线性的激活函数 f 时，情况就完全不同了。

  • 激活函数为网络引入了非线性因素。每一层不再是简单的线性变换，而是对数据空间进行一次“扭曲”或“折叠”

常见的激活函数及其变体：

2.1 sigmoid 函数

  • 形状：一条平滑的“S”形曲线，能将任意输入值“压缩”到 (0, 1) 区间内。
  • 优点：输出范围有限，可以用作概率值；平滑可导。
  • 缺点：容易导致梯度消失（在反向传播时，当输入值很大或很小时，其梯度接近于 0，导致网络难以训练）；输出不是以 0 为中心。
  • 应用：在早期很流行，现在主要用于二分类任务的输出层。

2.2 tanh 函数

  • 形状：与 Sigmoid 类似，但输出范围被压缩到 (-1, 1) 区间内。
  • 优点：输出以 0 为中心，通常比 Sigmoid 收敛更快。
  • 缺点：仍然存在梯度消失的问题。

2.3 ReLU 函数

  • 优点：
    • 计算效率极高。
    • 在正数区间的梯度恒为 1，极大地缓解了梯度消失问题，使深度网络训练成为可能。
    • 能使网络具有稀疏性（让一些神经元输出为 0）。
  • 缺点：Dying ReLU 问题。如果一个神经元的输入恒为负，那么它将永远不会被激活，其梯度也永远为 0，导致该神经元“死亡”，无法再学习。
  • 应用：目前是绝大多数神经网络中隐藏层的首选和默认激活函数。

为了解决“Dying ReLU”问题，出现了很多变体，如 Leaky ReLU（允许负数输入有一个非常小的非零斜率）、PReLU、ELU 等。

2.4 Softmax 函数

Softmax 是专为多分类任务设计的输出层激活函数，它能将一组任意的数字分数，优雅地转换成一个“此消彼长”、总和为 1 的概率分布，让模型的预测结果变得直观且有意义。

2.4.1 Softmax 是什么？—— 一个“概率转换器”

从本质上讲，Softmax 函数的作用是将一个包含任意实数（可正可负）的向量，转换成一个概率分布向量。

一个合格的概率分布需要满足两个核心条件：

  1. 向量中的每一个元素值都在 0 到 1 之间。
  2. 向量中所有元素的总和必须等于 1。

Softmax 完美地实现了这两点。

2.4.2 为什么需要 Softmax？—— 从原始分数到可解释的概率

想象一下，我们正在训练一个神经网络来识别图片，任务是把它分为三个类别：“猫”、“狗”或“鸟”。

在经过所有隐藏层的计算后，网络最后一层可能会对这三个类别输出一组原始的、未处理过的分数（我们称之为 logits），比如：

[2.9, 1.2, 0.3]

这组分数告诉我们什么？

  • 我们可以直观地看出，模型认为“猫”的可能性最大（2.9 分），其次是“狗”（1.2 分），最后是“鸟”（0.3 分）。
  • 但是，这些分数并不是概率。它们不是在 0 到 1 之间，它们的和也不等于 1。我们无法直接说“这张图片有 290% 的可能是猫”，这毫无意义。

我们需要一种方法，将这些任意的分数转换成一个直观、有意义的概率分布。这正是 Softmax 函数要解决的问题。

2.4.4 Softmax vs. Sigmoid

一个常见的困惑是 Softmax 和 Sigmoid 的区别。

特性 Softmax 函数 Sigmoid 函数
应用场景 多分类问题（类别之间互斥，如猫/狗/鸟） 二分类问题（是/否），或多标签分类问题
输出 输出一个向量，所有元素之和为 1，代表一个完整的概率分布 输出一个标量（单一数值），在 0 到 1 之间，代表属于某个正类的概率
相互关系 输出是相互关联的，一个元素的增加必然导致其他元素的减少 （在多标签分类中）每个输出是相互独立的
本质 Sigmoid 函数可以看作是只有两个类别时的 Softmax 函数的特例

2.5 如何选择激活函数

在实践中，我们可以遵循一些通用的指导原则：

  1. 隐藏层的首选：ReLU
    • 在不确定用什么时，就用 ReLU。它是绝大多数现代神经网络的默认选择，因为它简单、高效且有效。
  2. 如果 ReLU 效果不佳，尝试其变体
    • 如果在训练中发现有大量神经元“死亡”，可以尝试使用 Leaky ReLU 或其他变体。
  3. 输出层的选择取决于任务类型
    • 二分类问题：在输出层使用 Sigmoid 函数，输出一个概率。
    • 多分类问题：在输出层使用 Softmax 函数，输出所有类别的概率分布。
    • 回归问题：在输出层不使用任何激活函数（或使用线性激活函数），直接输出预测值。
  4. 谨慎使用 Sigmoid 和 Tanh
    • 由于梯度消失问题，现在已经很少在较深的神经网络的隐藏层中使用 Sigmoid 或 Tanh。

3 训练过程

神经网络的训练就是不断重复以下四个步骤的过程：

  1. 正向传播 (Forward Pass): 输入一个批次的数据，通过网络计算出预测值 $\hat{y}$​。
  2. 计算损失 (Compute Loss): 用损失函数计算预测值 $\hat{y}$​ 和真实值 $y$ 之间的差距 $L$。
  3. 反向传播 (Backward Pass): 从损失函数开始，反向计算出网络中所有参数（$w$ 和 $b$）的梯度。
  4. 更新参数 (Update Parameters): 使用梯度下降法，根据计算出的梯度和学习率，对参数进行微小的调整。

这个循环会重复成千上万次，每一次循环，网络的参数都会被微调，使得损失值逐渐降低，网络的预测能力也随之增强，最终达到一个令人满意的精度。

3.1 正向传播 (Forward Propagation)

目标： 根据给定的输入数据和当前的网络参数（权重和偏置），计算出网络的预测输出。
过程： 这是一个从前到后、逐层计算的过程。

1、输入层 (Input Layer): 将你的原始数据（例如，一张图片的像素值，一段文本的词向量）输入到网络的第一层。

2、隐藏层 (Hidden Layers):

3、逐层向前： 第一个隐藏层的输出，会作为第二个隐藏层的输入。这个 加权求和 -> 激活函数 的过程会逐层重复，直到最后一层。

4、输出层 (Output Layer): 这是网络的最后一层，其激活函数的选择通常取决于具体的任务类型：

  • 回归任务 (如预测房价): 通常使用线性函数或不使用激活函数。
  • 二分类任务 (如判断猫狗图片): 通常使用 Sigmoid 函数，输出一个介于 0 和 1 之间的概率值。
  • 多分类任务 (如手写数字识别): 通常使用 Softmax 函数，输出所有类别的概率分布。

注意：

这里隐藏层中的每一个神经元都拥有自己专属的一套权重（weights）和偏置（bias），都会独立地执行一次这个完整的计算过程。

因为每个神经元的权重和偏置都是独立学习的，它们会从相同的输入数据中学会提取不同的信息。

  • 顶部神经元可能学会了识别一个“左上到右下”的边界模式。
  • 底部神经元可能学会了识别一个“左下到右上”的边界模式

当这两个神经元将它们各自提取出的不同特征（$a_{top}$ 和 $a_$）传递给输出层时，输出层就可以结合这两种不同的模式信息，从而创造出如图中所示的那种更复杂的、非线性的决策边界。

3.2 损失函数 (Loss Function)

在进行反向传播之前，我们需要一个衡量标准来判断我们“错得有多离谱”。这个标准就是损失函数（或成本函数，Cost Function）。

它通过比较网络的预测输出 $\hat{y}$​​ 和真实的标签 $y$ 来计算出一个“损失值”或“误差值”。

我们的终极目标是：通过调整网络的权重和偏置，让这个损失函数的值变得尽可能小。

3.3 反向传播 (Backward Propagation)

目标： 计算损失函数对网络中每一个参数（权重 w 和偏置 b）的梯度（偏导数）。这个梯度指明了我们应该朝着哪个方向、以多大的幅度去调整参数，才能最快地减小损失。

核心原理： **链式法则 (Chain Rule)**，这是微积分中的一个基本规则。由于整个网络（从输入到最终的损失）是由一系列复合函数构成的，链式法则允许我们将最终的损失，层层反向地分解，计算出它对网络中任意一个参数的偏导数。

过程： 这是一个从后向前、逐层计算梯度的过程。

关键思想： 每一层的梯度计算，都依赖于它后面那一层已经计算出的梯度。这就是“反向”传播的含义。

RMSProp 采用衰减平均的方式计算历史梯度，解决 Adagrad 的问题。 避免了学习率过早消失的问题，比 Adagrad 更灵活。 仍需设置一个全局学习率。 每个参数独立、自适应
Adadelta 类似于 RMSProp，但通过额外技巧消除了学习率这个超参数。 无需设置学习率。 实践中可能不稳定，目前使用较少。 每个参数独立、自适应
Adam 集大成者，结合了动量法和RMSProp的优点。 结合了两者的优点，收敛快、稳定，对多数问题效果好，是当前最常用的优化器。 理论上在某些情况下可能不收敛。 每个参数独立、自适应、带动量
  • 数据量不大选批量梯度下降法
  • 数据量大选小批量梯度下降法
  • 标配 Adam，但具体看效果
  • 其他最优化方法：牛顿法、拟牛顿法等等

4.3 梯度消失和梯度爆炸

问题 梯度消失 (Vanishing Gradient) 梯度爆炸 (Exploding Gradient)
核心原因 激活函数的导数持续小于 1 权重的数值持续大于 1
直接后果 连乘后梯度趋近于零 连乘后梯度趋近于无穷大
对训练的影响 导致网络退化，模型无法有效训练，早期层参数不更新 模型训练不稳定，参数更新过大导致崩溃
![image.png 665](https://picgocloud.oss-cn-shanghai.aliyuncs.com/picgo/20250723162916.png)
  • 图中以 Sigmoid 函数为例，它的导数最大值仅为 0.25，并且在输入值较大或较小时，其导数会趋近于 0。
  • 在反向传播的连乘计算中，如果大部分 $f^′(x_i​)$ 的值都是一个远小于 1 的小数（例如 0.2），多个这样的小数相乘，结果会以指数级速度快速缩小，最终趋近于 0。

  • 如果网络中的权重 wi​ 被初始化得较大，或者在训练中持续增大，导致它们的值普遍大于 1。
  • 在反向传播的连乘计算中，多个大于 1 的数相乘，结果会以指数级速度快速增大，最终形成一个巨大的数值。

为了解决这两个问题，研究人员提出了一系列有效的策略：

  1. 更换激活函数：使用 ReLU 及其变种作为激活函数（它们的导数在正区间恒为 1，缓解了梯度消失
  2. 设计新的网络架构（如残差网络 ResNet）
  3. 梯度剪切：设定一个梯度的上限阈值。在反向传播计算出梯度后，如果梯度的范数（可以理解为梯度向量的长度或大小）超过了这个阈值，就将其强制缩放到阈值的大小。
    • 左图展示了没有梯度剪切时，由于某次梯度过大（梯度爆炸），导致参数更新的“步子”迈得太大，直接“飞出”了最优解区域，使得训练不稳定。
    • 右图展示了使用梯度剪切后，过大的梯度被“修剪”回一个合理的范围，使得参数更新的步伐平稳可控，能够稳定地走向最优解。
  4. 采用更科学的权重初始化方法（如 Xavier/He 初始化）
  5. 使用批量归一化 (Batch Normalization)

5 神经网络优缺点

优点 (Advantages) 缺点 (Disadvantages)
1. 强大的学习能力
能够自动学习极其复杂的非线性模式和数据特征。		 1. 黑盒与不可解释性
决策过程不透明，难以理解模型“为什么”这么做，在关键领域应用受限。
2. 卓越的性能与准确率
在图像、语音、自然语言等领域是业界顶尖 (SOTA) 技术，准确率高。		 2. 训练成本高昂
需要强大的硬件（如 GPU/TPU），且训练大型模型耗时数天甚至数周。
3. 广泛的通用性
单一架构（如 Transformer）可应用于多种不同的任务和数据类型。		 3. 巨大的数据需求
通常需要海量的高质量标注数据才能达到理想性能，数据获取成本高。
4. 数据越多，性能越好
模型性能会随着数据量的增加而持续提升，非常适合大数据时代。		 4. 训练调试复杂
对网络结构、学习率等大量超参数的选择非常敏感，需要丰富经验和大量实验。
5. 容易过拟合
模型参数多，极易“记住”训练数据而非学习通用规律，需依赖正则化等技巧。
