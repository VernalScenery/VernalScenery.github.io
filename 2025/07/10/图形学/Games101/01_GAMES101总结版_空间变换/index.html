核心思想

把三维空间物体变为二维的一张照片

一共需要如下几个步骤：

模型坐标系 -> 世界坐标系 -> 观察坐标系（又称相机坐标系、视图坐标系）-> 裁剪坐标系 -> NDC 坐标系 -> 屏幕坐标系

变换阶段 核心作用 输入坐标空间 输出坐标空间
模型变换 将物体从局部坐标系转换到世界坐标系，定义物体在场景中的位置、朝向和大小。 局部坐标系（模型空间） 世界坐标系（世界空间）
视图变换 将世界坐标系中的物体转换到相机坐标系，模拟观察者视角。 世界坐标系 观察坐标系（视图空间）
投影变换 将 3D 观察空间投影到 2D 裁剪空间，建立透视关系或正交映射，并裁剪不可见区域。 观察坐标系 裁剪坐标系（裁剪空间）
齐次除法（透视除法） 将裁剪空间的齐次坐标转换为标准化设备坐标（NDC），范围为 ($[-1, 1]^3$。 裁剪坐标系 标准化设备坐标系（NDC）

平移无法用矩阵形式表示，所以，平移不是线性变换！

但我们不希望平移成为一种特殊情况，是否存在一种统一的方式来表示所有变换？（以及代价是什么?）

为了统一表示平移、旋转、缩放等变换，引入齐次坐标：在 2D 坐标基础上增加一个 w 坐标，区分 “点” 和 “向量”：

  • 2D 点（Point）：表示为 $(x,y,1)^T$（$w = 1$ ，体现位置属性 ）。
  • 2D 向量（Vector）：表示为 $(x,y,0)^T$（$w = 0$ ，体现方向 / 长度属性，无位置 ）。

2 旋转矩阵是正交矩阵

观察发现：$R_{-\theta}$ 其实就是 $R_{\theta}$​ 的转置矩阵，故有：

$$
R_{-\theta} = R_{\theta}^{-1} = R_{\theta}^{T}
$$

根据逆矩阵的定义, 这等价于：

$$
R_{\theta}^{T} \cdot R = R_{\theta}^{-1}\cdot R = I
$$

其中 $I$ 是 $n×n$ 的单位矩阵

3 绕任意轴旋转

3.1 核心思路

已知绕 x、y、z 轴的标准旋转方法，若要绕 任意轴 $u$ 旋转，可通过 3 步实现：

任意轴 $u$ 默认起点为原点，不在原点可以先通过平移实现

  • 先把 “任意轴 $u$” 旋转到 x、y、z 任意一轴 (比如 x 轴)，用矩阵 $R_1$ 实现；
  • 用 绕 x 轴的标准旋转矩阵 $R_x$ 完成旋转；
  • 再把轴 “逆旋转” 回原始位置，用 $R_1$ 的转置 ${R_1}^T$（因为旋转矩阵是正交矩阵，${R_1}^{-1}$ = ${R_1}^{T}$ ）实现。

整体变换公式：$R1 \cdot Rx \cdot R1^T \cdot (x, y, z)^T$

这里的 $R_x$ 是知道的，也就是 3 维下绕 x 轴旋转的矩阵，故只需要求出 $R_1$ 即可

3.2 矩阵推导

以 $u$ 为一轴构造 3 维正交坐标系，让 $u$ 与 $x$ 轴对齐，步骤如下：

  • 任取一个不与 $u$ 重合的方向 $t$，计算 $w=t×u$（叉乘，得到与 $u$ 垂直的向量 ）；
  • 再计算 $v=u×w$（叉乘，得到同时与 $u$、$w$ 垂直的向量 ）；
  • 此时 $u,w,v$ 构成新的正交坐标系（利用叉乘 “垂直性” 保证正交 ）。

把 $u,w,v$ 按列组成矩阵，就是 $R_1=(u,w,v)$ 。它的作用是：将原始坐标系 x,y,z 旋转到新坐标系 u,w,v

因为旋转矩阵是正交矩阵，${R_1}^{-1}$ = ${R_1}^{T}$
所以 ${R_1}^T$ 的作用与 $R_1$ 相反：将新坐标系 u,w,v 旋转回原始坐标系 x,y,z 。

最终绕任意轴 $u$ 旋转矩阵为：

$$
R(u,ϕ)=R_1​⋅R_x​(ϕ)⋅{R_1}^T​ =
\begin{pmatrix}
x_u & x_w & x_v \
y_u & y_w & y_v \
z_u & z_w & z_v
\end{pmatrix}
\begin{pmatrix}
\cos\phi & -\sin\phi & 0 \
\sin\phi & \cos\phi & 0 \
0 & 0 & 1
\end{pmatrix}
\begin{pmatrix}
x_u & y_u & z_u \
x_w & y_w & z_w \
x_v & y_v & z_v
\end{pmatrix}
$$

视图变换

将世界坐标系中的物体转换到相机坐标系，模拟观察者视角。

1 核心思想

现象： 如果相机（位置、朝向等）和所有物体同步运动（比如一起平移、旋转），最终 “拍出来的照片（渲染结果）会和原来一样”。

我们可以把相机强行变换到一个统一的 “标准观察坐标系”：

  • 位置：世界坐标系的原点 (0,0,0)
  • 上方向：Y 轴正方向
  • 观察方向：-Z 轴方向（即镜头朝向 Z 轴负方向）

视图矩阵的作用：将世界坐标系中的点转换为以相机为原点、观察方向为 −Z、上方向为 Y、右侧为 X 的观察坐标系。

要达成上述目标，$M_{\text{view}}$​ 需做一系列变换，对应上图里的步骤：

  • 平移（Translate e to origin）：先把相机的位置 $\vec{e}$（观察坐标系原点）移到世界坐标系原点，消除位置差。
  • 旋转对齐坐标轴：
    • 把相机的观察方向  $\vec{g}$​（g 向量）旋转到世界坐标系的 -Z 轴；
    • 把相机的上方向  $\vec{t}$（t 向量）旋转到世界坐标系的 Y 轴；
    • 利用叉乘  $\vec{g} \times \vec{t}$ 得到的向量，旋转对齐世界坐标系的 X 轴 (叉乘结果本就是垂直于 $\vec{g}$， $\vec{t}$ 的 “右方向”，用来对齐 X 轴，保证三个轴正交)

对应公式为：$M_{\text{view}} = R_{\text{view}}\cdot T_{\text{view}}$

投影变换

1 核心思想

投影矩阵（正交或透视）用于将观察空间（View Space）中的视体（视锥体 / 长方体），映射到一个标准化的立方体空间 [−1, 1]^3。

  • 视锥体（Frustum）：三维空间中的截头锥体，定义了相机的可视范围，是现实世界视角的几何描述。
  • 裁剪空间（Clip Space）：使用齐次四维坐标表示的空间，视锥体经过投影矩阵变换后被映射成一个以 $w$ 为尺度的立方体区域 ，满足 $−w≤x,y,z≤w$。
  • 投影矩阵：负责将三维视锥体变换成四维裁剪空间的立方体，使后续的裁剪和透视除法成为可能。
  • 透视除法：将裁剪空间中的四维坐标通过除以 $w$，映射到标准化设备坐标 （NDC）空间，范围固定为 $[-1, 1]^3$，为后续的视口变换和光栅化做准备。
    • 在透视投影中，w 分量中保留“深度感”信息，做除法后，近的点会大，远的点会小，于是就形成了 远小近大的透视效果。
    • 在正交投影中，w 恒为 1，无深度扭曲。

注意：投影矩阵不会直接得到 NDC，而是先把观察空间变换到 裁剪空间（齐次空间）， 然后通过除以 w（透视除法），才能进入 NDC 空间

在图形学中，顶点经过 模型 → 视图 → 投影 变换后，会被转换为一个四维向量 (x, y, z, w)，处于所谓的 裁剪空间（Clip Space）。

此时的每个分量意义如下：

分量 含义 用途
x, y 投影后的结果（但还未变为屏幕像素坐标） 决定物体在二维图像上的位置
z 深度值，也就是“离相机的远近程度” 后续用于深度测试（遮挡判断）
w 保留的“透视信息” 用于之后进行透视除法，获得标准化坐标

2 正交投影

正交投影是相对简单的一种，坐标的相对位置都不会改变，所有光线都是平行传播，我们把视体（长方体）映射到规范立方体 $[−1,1]^3$ 即可。

正交投影下，裁剪空间 = NDC，但两者含义不同。裁剪空间是矩阵变换的输出，NDC 是透视除法后的结果。

3 透视投影

3.1 整体思路

透视投影的几何形状是视锥体（Frustum），类似人眼所看东西的方式，遵循近大远小，直接处理较复杂。因此采用 “两步法”：

在挤压视锥体的过程中需要注意如下几点：

  1. 近平面上任意一点 (x, y, n) 不变
  2. 远平面上任意一点 (x, y, f) 的 z 不变, 一直为 f
  3. 远平面中心点 (0, 0, f) 不变

这三个条件共同约束了挤压变换矩阵 $M_{\text{persp}\to\text{ortho}}$ 的形式，确保：

  1. 近平面作为成像平面的坐标保真；
  2. 远平面深度值的正确性；
  3. 光轴中心对称性的保持

2.2.1 算法思想

它叫 “逐边裁剪法”，意思就是像给多边形 “修边”，一条边一条边地处理，把多边形超出 “窗口（可视范围，比如屏幕框 ）” 的部分一点点剪掉 。

  1. 循环处理窗口边：窗口不是有四条边嘛（假设是长方形窗口 ），算法会一条边一条边地拿来裁剪多边形，循环着把四条边都用一遍 。
  2. 准备两个 “点队伍”：每次处理一条窗口边时，弄一个空的 “点数组 DST（新队伍 ）”，原来多边形的顶点放在 “数组 SRC（原队伍 ）” 。
  3. 顶点配对测试：把原队伍（SRC ）里的顶点，从第一个开始，两两配对，比如顶点 S 是当前点，顶点 P 是 S 后面那个点 。然后拿这对顶点（S 和 P ），对着当前要处理的窗口边，看看咋处理，规则看下面这 4 种情况（对应图里的情况 ）：
    • 情况 1（无输出 ）：S 和 P 都在窗口边的 “外侧”（也就是要剪掉的那侧 ），那这俩点都不要，DST 里不加 。
    • 情况 2（输出交点 ）：S 在外侧，P 在内侧（窗口该留的一侧 ），这时候找 S 到 P 的连线和窗口边的 “交点 I”，把交点 I 放进 DST 。
    • 情况 3（输出交点 + P ）：S 在内侧，P 在外侧，先找 S 到 P 连线和窗口边的交点 I，把 I 放进 DST，再把 P 也放进 DST
    • 情况 4（输出 P ）：S 和 P 都在内侧，直接把 P 放进 DST 。
  4. 更新队伍：处理完一轮（一条窗口边 ），把 DST 变成新的 SRC，继续用下一条窗口边裁剪，直到四条边都处理完，剩下的 DST 里的点，就组成裁剪后的多边形啦 。

2.2.3 属性插值

  1. 找边界的 “规矩”（边界线方程 ）： 先确定平面边界的数学表达式（比如直线方程，像之前的 $ax + by = d$），这是判断点在里在外、算距离的依据 。
  2. 算 S 和 P 到边界的 “有向距离”： 把 S 点、P 点的坐标代入边界线方程，算出它们到边界的距离 $d_s$（S 到边界的距离 ）和 $d_p$（P 到边界的距离 ）。注意是 “有向距离”，意思是距离有正负，能区分在边界内侧还是外侧（比如内侧正、外侧负，或者反过来 ）。
  3. 定个 “混合比例”（weight ）： 用距离算一个 “权重”，决定 S 和 P 的属性在交点 $I$ 里占多少。公式里的 $weight = \frac{l_s}{l_s - l_p} = \frac{d_s}{d_s - d_p}$，其实就是看 S 和 P 到边界的距离关系，确定 $I$ 更靠近谁，权重就代表 $P$ 的属性占比（剩下的 $1 - weight$ 就是 S 的占比 ）

2.3 三维

核心是把 NDC 的范围限制，转化为齐次裁剪空间的 6 个平面方程，让 GPU / 渲染管线 可以：

  1. 用线性代数方法快速判断顶点在裁剪空间内 / 外（代入平面方程看符号）。
  2. 通过有向距离插值，计算顶点被裁剪平面切割时的交点，实现 Sutherland - Hodgman 算法的三维裁剪。

二维里是用 “边” 裁剪多边形，三维里变成用 “平面” 裁剪 3D 模型（比如三角形、立方体 ）。

而且选了个叫 “NDC 构成的 -1 到 1 盒体” 的范围裁剪，你可以把这盒体想成摄像头能看到的 “有效空间”，超出这个盒子的模型部分，就要被剪掉 。

  • 顶点 z 值不为 0：$z_e = 0$ 就相当于顶点和摄像机（眼睛 ）“贴在一起” 了，这时候计算容易出问题，所以得保证顶点 z 值不是 0 ，避免这种极端情况 。
  • 顶点 z > 0，说明它在摄像机后方（或与摄像机重合），需被裁剪。

最终，6 个裁剪条件转化为 6 个齐次裁剪空间的平面方程。

把每个裁剪条件（如 xₑ + wₑ > 0 ）看作 四维空间（xₑ, yₑ, zₑ, wₑ）中的平面，形式为 axₑ + byₑ + czₑ + dwₑ + e = 0（这里 e=0 ）。

  • 平面的法向量：取系数 (a, b, c, d)，比如 xₑ + wₑ > 0 对应法向量 (1, 0, 0, 1) 。
  • 有向距离：顶点代入平面方程 axₑ + byₑ + czₑ + dwₑ 后，结果符号表示 “在平面哪一侧”，绝对值可看作到平面的 “有向距离”，用于插值计算裁剪交点（类似二维 Sutherland - Hodgman 的交点计算）。

边界情况处理：

  • 本质：处理多边形顶点的 “循环性”（最后一个顶点要和第一个顶点连起来）时，算法需要只计算一次首尾之间的交点，避免因 “回头遍历” 重复计算。
  • 实现：通过标记 “是否是最后一个顶点”，在遍历到 P = size - 1（最后一个顶点）时，特殊处理与 S（第一个顶点）的衔接，保证交点计算只触发一次，且结果正确闭合。

3 为什么要在透视除法之前裁剪？

场景设定

你有一个点 $(x_c, y_c, z_c, w_c)$，这是已经经过了投影矩阵（透视投影矩阵）变换的坐标，但还没做除法。
我们最终要判断：这个点是否能被渲染，也就是说——它是否在可见范围内？

可见范围定义

GPU 认为一个点在屏幕上可见，要满足：

这个区域叫做 NDC 立方体（标准化设备空间立方体），是一个中心在原点、每边长 2 的立方体。

问题

如果你直接在 NDC 空间裁剪（除完 $w$ 再裁剪）会出问题：

  • 除以 $w_c$ 是非线性的操作，不能在线性插值（比如三角形顶点插值）之前做。
  • 比如三角形有两个顶点在 NDC 内部，一个在外部，这时必须插值计算裁剪点 —— 如果你已经除以了 $w$，就无法正确线性插值了。

因为裁剪发生在透视除法之前，此时还在齐次裁剪空间，$w$ 还没有被除掉，几何和属性都还处在投影前的线性空间中，插值是线性的、没有透视畸变，所以属性可以直接线性插值。

解决方法：在除法之前裁剪！

我们要想个办法，在除法之前判断点是否在 NDC 范围内。

我们来把这个判断不等式：

$-1 < \frac{x_c}{w_c} < 1$

等价变形一下（乘上 $w_c$，前提是 $w_c > 0$）：

$-w_c < x_c < w_c \implies x_c + w_c > 0,\ -x_c + w_c > 0$

这样我们就可以在裁剪阶段，用线性方法裁剪多边形，不会破坏插值！

透视矫正

  • 但是投影之后，从 3D 空间变为了 2D 平面，再根据平面的重心坐标去插值属性，就没有近大远小的效果了

假设你从 3D 世界中拍一张照片：

  • 裁剪时：你还没按快门，相机在看世界，东西直的就是直的，插值是线性的。
  • 光栅时：你已经拍完照片了，要在照片上贴图，近大远小，线性插值已经不再“真实”，你得按比例调整（就是透视矫正）。

你在 GPU 里绘制一个三角形，三个顶点分别有不同的纹理坐标，比如：

A $(x_1,y_1,z_1,w_1)$ $(u_1,v_1)$
B $(x_2,y_2,z_2,w_2)$ $(u_2,v_2)$
C $(x_3,y_3,z_3,w_3)$ $(u_3,v_3)$

用线性插值两个量，构建一个非线性的比值关系，从而得到正确的透视下的插值效果。

  • **裁剪阶段保留 $w$**，此时齐次空间是线性的，方便直接做线性插值生成新顶点。
  • 透视矫正阶段是先除以 $w$，把属性压到透视变形空间中，再用线性插值，然后再乘回来还原出真实的非线性插值结果。
阶段 空间形状 操作 属性插值 为什么
透视投影之前 视锥体（frustum） 保留 $w$，空间是锥形 无 真实世界线性空间，保留原始关系
除以 $w$ 之后 单位立方体（NDC） 除以 $w$ ⇒ 展平锥体为立方体 做线性插值 在立方体中插值是线性的
还原结果 恢复锥体映射下的位置 乘回 $w$ ⇒ 回到透视空间 结果变为非线性 让属性跟随几何透视变化

裁剪阶段是在4D 齐次坐标空间，此时：

  • 空间是“锥体”的形状（因为 $x,y,z$ 都还没除 $w$，整体是齐次坐标）
  • 所有顶点位置、属性（包含 $u,v$）都是齐次坐标形式
  • 插值基于齐次坐标线性插值（包括 $w$ 本身）
  • 这就是为什么裁剪时可以直接线性插值的原因：齐次空间本身就是线性的

透视除法之后，空间变为NDC 立方体空间：

  • 这个空间是 3D，但已经除去了 $w$，从四维压缩到三维
  • 在这个空间里，原来的锥形被“压平”成立方体（$x,y,z$ 范围都在 [-1,1]）
  • 这时候对属性（$u,v$ 等）直接线性插值就会产生错误，因为属性本质是由齐次坐标决定的非线性投影关系
  • 透视矫正通过插值 $\frac{u}{w}$ 和 $\frac{1}{w}$ 并再还原来修正这个非线性问题

总结

这个插值过程实际上模拟了在三维空间里插值再投影，而不是先投影再插值。

举个比喻：你拿一张纸（是三角形）在空间里看，然后它投影到眼睛前是扭曲的，如果你在投影后再“线性插值”，那插值出来的点不是真的在那张纸上，而是在歪了的三角形里。

视口变换

把规范立方体 NDC 的 “标准化坐标”，转成实际屏幕的 “像素位置” 。

  • 与 z 轴无关 。因为屏幕是 2D 的，这一步只处理 XY 平面的坐标映射，z 轴信息（比如深度 ）会在其他阶段处理（如深度测试 ），这里暂时不管 。
  • 把 [-1, 1]² 范围的坐标，映射到屏幕的 [0, width]×[0, height] 范围 。规范立方体的 XY 坐标是 [-1, 1]（左右、上下对称 ），而屏幕像素的 XY 坐标是从 (0,0) 到 (width, height)（比如分辨率 800×600 ，就是 [0,800]×[0,600] ），需要做 “缩放 + 平移” 变换，把标准化坐标转成屏幕像素位置 。

视口变换矩阵：

  • X 轴变换：
    原 X 坐标是 x ∈ [-1, 1] ，矩阵第一行 [width/2, 0, 0, width/2] 作用后：
    $x’ = \frac{width}{2} \cdot x + \frac{width}{2}$
    当 x = -1 时，x' = 0 ；当 x = 1 时，x' = width 。实现了从 [-1, 1] 到 [0, width] 的映射 。

  • Y 轴变换：
    原 Y 坐标是 y ∈ [-1, 1] ，矩阵第二行 [0, height/2, 0, height/2] 作用后：
    $y’ = \frac{height}{2} \cdot x + \frac{height}{2}$
    当 y = -1 时，y' = 0 ；当 y = 1 时，y' = height 。实现了从 [-1, 1] 到 [0, height] 的映射 。

  • Z 轴与齐次坐标：
    第三行 [0, 0, 1, 0] 保持 Z 坐标不变（因为 z 轴无关，这里可暂存深度信息 ）；第四行 [0, 0, 0, 1] 保证齐次坐标的 w 分量始终为 1 ，维持齐次坐标规则 。
