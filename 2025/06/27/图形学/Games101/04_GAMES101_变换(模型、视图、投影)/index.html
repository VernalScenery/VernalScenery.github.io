Why study transformation？

模型变换

视图变换

视图变换的目标是：把三维场景中的物体，转换为 “从相机视角观察” 的效果（类似人眼 / 相机拍摄时，如何定位、看向哪里、以哪个方向为 “上” ）。要实现这个变换，第一步是明确定义相机的状态。

1 定义相机的状态

2 简化视图变换的思路

现象： 如果相机（位置、朝向等）和所有物体同步运动（比如一起平移、旋转），最终 “拍出来的照片（渲染结果）会和原来一样”。

基于上面的观察，图形学里用一个巧妙的方法简化计算：

  1. 固定相机到 “标准位置”
    我们可以把相机强行变换到一个统一的 “标准观察坐标系”：
    • 位置：世界坐标系的原点 (0,0,0)
    • 上方向：Y 轴正方向（Up at Y）
    • 观察方向：-Z 轴方向（Look at -Z，即镜头朝向 Z 轴负方向）
  2. 让物体 “跟着相机同步变换”
    既然 “相机和物体同步运动不影响结果”，那么：
    • 对相机做的变换（比如从任意位置移动到原点），反向施加给所有物体。
    • 这样就等效于“相机移动到标准位置，物体跟着动”，但实际计算时，只需要对物体做变换，就能模拟出 “相机在标准位置观察” 的效果。

  • X、Y、Z 轴定义明确（原点 (0,0,0)，上方向 Y，观察方向 -Z ）。
  • 所有三维场景的视图变换，最终都可以转换为 “把物体变换到这个标准坐标系下的观察结果”，方便统一渲染计算。

3 构造视图矩阵

3.1 核心思想

视图矩阵的作用：将世界坐标系中的点转换为以相机为原点、观察方向为 −Z、上方向为 Y、右侧为 X 的观察坐标系。

要达成上述目标，$M_{\text{view}}$​ 需做一系列变换，对应上图里的步骤：

  • 平移（Translate e to origin）：先把相机的位置 $\vec{e}$（观察坐标系原点）移到世界坐标系原点，消除位置差。
  • 旋转对齐坐标轴：
    • 把相机的观察方向  $\vec{g}$​（g 向量）旋转到世界坐标系的 -Z 轴；
    • 把相机的上方向  $\vec{t}$（t 向量）旋转到世界坐标系的 Y 轴；
    • 利用叉乘  $\vec{g} \times \vec{t}$ 得到的向量，旋转对齐世界坐标系的 X 轴 (叉乘结果本就是垂直于 $\vec{g}$， $\vec{t}$ 的 “右方向”，用来对齐 X 轴，保证三个轴正交)

3.2 视图矩阵的推导

3.2.1 整体思路

$M_{\text{view}} = R_{\text{view}}\cdot T_{\text{view}}$

视图变换要让相机（观察坐标系）挪到世界坐标系的标准位置（原点、Y 上、-Z 前 ），分两步：

  • 先平移（$T_{\text{view}}$​）：把相机位置 $\vec{e}$ 移到世界原点；
  • 再旋转（$R_{\text{view}}$​）：把相机的观察方向、上方向等，对齐世界坐标系的 -Z、Y、X 轴。

这里平移矩阵很简单，主要是旋转矩阵的推导。

3.2.2 旋转矩阵的推导

$R_{\text{view}}$​​：对齐坐标轴的 “逆向推导”

目标是把相机的三个轴：

  • 观察方向 $\vec{g}$​ → 世界 $-Z$ 轴
  • 上方向 $\vec{t}$​ → 世界 $Y$ 轴
  • 右方向 $\vec{g} \times \vec{t}$（叉乘，垂直于 $\vec{g}$， $\vec{t}$ 的轴 ）→ 世界 $X$ 轴

直接推导旋转矩阵麻烦，所以反过来想：如果能找到 “世界坐标轴 → 相机坐标轴” 的旋转矩阵 $R_{\text{view}}^{-1}$，那么 $R_{\text{view}}$​ 就是它的逆矩阵（因为旋转矩阵是正交矩阵，逆 = 转置）。

投影变换

投影的本质是：把3D 空间中的物体 / 场景，映射到2D 平面（比如屏幕、画布 ）的过程，对应 “3D to 2D” 。

两类投影方式：

  • 正交投影常用在 CAD 软件、建筑可视化（需要精确尺寸传递 ）；
  • 透视投影常用在游戏、电影、VR 等追求真实视觉效果的场景。

1 正交投影 (Orthographic Projection)

1.1 A simple way of understanding

核心逻辑：“丢弃 Z 坐标 → 规整化 2D 坐标”

1.1.1 前置约定：相机的标准位置

  • 先假设相机（观察点）放在世界坐标系的原点，朝向 -Z 方向（即往 Z 轴负方向看 ），“上方向” 对齐世界坐标系的 Y 轴。
  • 这和视图变换（View Transformation）的标准约定一致（让相机位置 / 方向标准化，方便后续计算 ）

1.1.2 Drop Z coordinate（丢弃 Z 坐标）

  • 3D 点的坐标是 (x,y,z)，投影时直接去掉 Z 分量，得到 (x,y) 。
  • 这一步的几何意义：把 3D 点垂直投影到 Z=0 平面（投影平面，projection plane），相当于用平行于 Z 轴的光线 “照射” 物体，在 Z=0 平面留下影子（正交投影的本质 ）

1.1.3 Translate and scale to [−1,1]^2（平移 + 缩放至规范矩形）

投影得到的 2D 点 (x,y) 范围可能不规则，需要：

  • 平移：把点的中心移到坐标原点附近；
  • 缩放：把点的范围缩放到 [−1,1]×[−1,1](即 x 和 y 都在 -1 到 1 之间), 这一步是为了后续和 “视口变换”（Viewport Transformation）衔接，让图形能适配屏幕坐标。

1.2 In general

目标：映射到规范立方体

  • 需求：图形学中，为了统一后续的裁剪、视口变换，需要把 3D 场景中的 “感兴趣区域”（通常是一个长方体，比如正交投影的视体、透视投影的平截头体 ），映射到一个标准立方体：[−1,1]^3（即 x、y、z 坐标都在 -1 到 1 之间 ）。
  • 符号定义：要映射的长方体是 [l,r]×[b,t]×[f,n] ，其中：
    • l/r：x 轴方向的左 / 右边界；
    • b/t：y 轴方向的下 / 上边界；
    • f/n：z 轴方向的远 / 近边界
  • 注意：
    • 这里是右手坐标系，Z 轴指向外面，所以 n 可能比 f 小，比如 n 是 -5，f 是 -10。
    • 实际图形 API（如 OpenGL ）为了简化理解，改用左手坐标系，避免 n>f 的反直觉问题

两步变换：Translate（平移）→ Scale（缩放）

核心逻辑：“平移消偏移 → 缩放归一化” ，让任意 3D 区域变成标准立方体，方便后续的裁剪（只有在 [−1,1]^3 内的点才会被渲染 ）和视口变换（映射到屏幕坐标 ）

1.3 正交投影矩阵的构造

核心思路：先平移、后缩放

正交投影的目标是把 3D 长方体 [l,r]×[b,t]×[f,n] 映射到规范立方体 [−1,1]^3，分两步：

  • 第一步（Translate）：把长方体的中心平移到世界坐标系原点：要让中心移到原点，需要反向平移（减去中心坐标 ）

  • 第二步（Scale）：把长方体的边长缩放到 2（因为规范立方体在 x/y/z 轴的范围是 [−1,1]，边长为 2 ）。

后续通过视口变换（Viewport Transformation），再映射到屏幕坐标（如 [0,width]×[0,height] ）。

2 透视投影

2.1 回顾：齐次坐标的性质

齐次坐标的核心性质

  • 只要四个分量成比例，就代表同一个点。比如：$xz:yz:z^2:z = x:y:z:1$

几何意义：“缩放不变性”

2.2 如何实现透视投影

2.2.1 整体思路

透视投影的几何形状是视锥体（Frustum）（近平面小、远平面大，类似相机的视野范围 ），直接处理较复杂。因此采用 “两步法”：

在挤压视锥体的过程中需要注意如下几点：

  1. 近平面上任意一点 (x, y, n) 不变
  2. 远平面上任意一点 (x, y, f) 的 z 不变, 一直为 f
  3. 远平面中心点 (0, 0, f) 不变

这三个条件共同约束了挤压变换矩阵 $M_{\text{persp}\to\text{ortho}}$ 的形式，确保：

  1. 近平面作为成像平面的坐标保真；
  2. 远平面深度值的正确性；
  3. 光轴中心对称性的保持

  • 近平面：距离相机 n 单位
2. 几何关系：相似三角形

同理，对 x 坐标，也存在相同的相似三角形关系：

n 是近平面距离相机的 Z 坐标，z 是原始点的 Z 坐标，且 z>n ，点在远侧

3 视锥体的参数定义

  • 理想情况是直接明确指定这些参数（l 左、r 右、b 底、t 顶 ），这样能精准定义近平面范围。
  • 实际应用里，人们常简化操作：用垂直视场角（vertical field-of-view，简称 fovY ） 和纵横比（aspect ratio ） 来间接确定 。这里默认近平面是 “对称” 的，也就是左边界 l = -r（左右对称 ），底边界 b = -t（上下对称 ） 。

通过简单的三角函数（正切）和比例关系，结合垂直视场角（fovY）、纵横比（aspect）以及近平面距离（|n| ），就能算出透视投影近平面的四个边界（l, r, b, t ）

由上图可得：

  • r = aspect * t = aspect * |n| * tan(fovY/2)
  • t = |n| * tan(fovY/2)

因为默认近平面是对称的（左右对称、上下对称 ）：

  • 左边界 l = -r（右是 r ，左就是负的 r ）。
  • 下边界 b = -t（上是 t ，下就是负的 t ）。

MVP 矩阵

  • 模型变换：对 3D 模型进行平移、旋转、缩放等操作，将其放置到场景中的合适位置，例如把游戏中的 “剑” 模型放到角色 “手中”。
  • 视图变换：确定相机（视点）的位置和角度，决定观察场景的视角，如 FPS 游戏中第一人称或第三人称视角的切换。
  • 投影变换：将 3D 空间物体投影到 2D 屏幕，分为正交投影和透视投影。正交投影无透视效果，用于工程制图、UI 渲染等；透视投影模拟人眼透视，远物显小，常用于游戏、影视 3D 场景。
  • 规范立方体到屏幕的转换：将经过投影变换得到的规范立方体 [-1,1]³ 中的坐标映射到屏幕的像素坐标，此步骤为视口变换。通过视口变换矩阵，把规范立方体在 xy 平面的坐标 [-1,1]² 转换为 [0,width]×[0,height]，从而将 3D 物体绘制到 2D 屏幕上。

视口变换

把规范立方体的 “标准化坐标”，转成实际屏幕的 “像素位置” 。

1 像素的坐标规则

  • 索引形式：像素的索引是 (x, y) ，而且 x 和 y 都是整数 （比如 (0,0) (1,2) ，对应像素在阵列里的行列位置 ）。
  • 索引范围：从 (0, 0) 到 (width - 1, height - 1) 。假设屏幕分辨率是 width×height（比如 800×600 ），那 x 最大到 799 ，y 最大到 599 ，覆盖所有像素 。
  • 中心坐标：像素 (x, y) 的 “中心位置” 是 (x + 0.5, y + 0.5) 。因为像素是小方块，为了计算精准，常把 “中心点” 当像素的代表坐标（比如后续做光栅化时，判断图形覆盖哪个像素，会用到中心坐标 ）。
  • 屏幕覆盖范围：屏幕整体覆盖的坐标范围是 (0, 0) 到 (width, height) 。比如 800×600 的屏幕，x 从 0 到 800 ，y 从 0 到 600 ，刚好把所有像素的小方块 “包起来” 。

2 规范立方体转屏幕

  • 与 z 轴无关 。因为屏幕是 2D 的，这一步只处理 XY 平面的坐标映射，z 轴信息（比如深度 ）会在其他阶段处理（如深度测试 ），这里暂时不管 。
  • 把 [-1, 1]² 范围的坐标，映射到屏幕的 [0, width]×[0, height] 范围 。规范立方体的 XY 坐标是 [-1, 1]（左右、上下对称 ），而屏幕像素的 XY 坐标是从 (0,0) 到 (width, height)（比如分辨率 800×600 ，就是 [0,800]×[0,600] ），需要做 “缩放 + 平移” 变换，把标准化坐标转成屏幕像素位置 。

视口变换矩阵：

  • X 轴变换：
    原 X 坐标是 x ∈ [-1, 1] ，矩阵第一行 [width/2, 0, 0, width/2] 作用后：
    $x’ = \frac{width}{2} \cdot x + \frac{width}{2}$
    当 x = -1 时，x' = 0 ；当 x = 1 时，x' = width 。实现了从 [-1, 1] 到 [0, width] 的映射 。

  • Y 轴变换：
    原 Y 坐标是 y ∈ [-1, 1] ，矩阵第二行 [0, height/2, 0, height/2] 作用后：
    $y’ = \frac{height}{2} \cdot x + \frac{height}{2}$
    当 y = -1 时，y' = 0 ；当 y = 1 时，y' = height 。实现了从 [-1, 1] 到 [0, height] 的映射 。

  • Z 轴与齐次坐标：
    第三行 [0, 0, 1, 0] 保持 Z 坐标不变（因为 z 轴无关，这里可暂存深度信息 ）；第四行 [0, 0, 0, 1] 保证齐次坐标的 w 分量始终为 1 ，维持齐次坐标规则 。

总结

变换阶段 核心作用 输入坐标空间 输出坐标空间
模型变换 将物体从局部坐标系转换到世界坐标系，定义物体在场景中的位置、朝向和大小。 局部坐标系（模型空间） 世界坐标系（世界空间）
视图变换 将世界坐标系中的物体转换到相机坐标系，模拟观察者视角。 世界坐标系 观察坐标系（视图空间）
投影变换 将 3D 观察空间投影到 2D 裁剪空间，建立透视关系或正交映射，并裁剪不可见区域。 观察坐标系 裁剪坐标系（裁剪空间）
齐次除法（透视除法） 将裁剪空间的齐次坐标转换为标准化设备坐标（NDC），范围为 ($[-1, 1]^3$。 裁剪坐标系 标准化设备坐标系（NDC）
视口变换 将 NDC 坐标映射到屏幕像素坐标，确定物体在屏幕上的显示位置和大小。 标准化设备坐标系 屏幕坐标系（屏幕空间）

投影矩阵作用：

  • 它把 观察空间中的锥形视锥体 映射到 一个立方体空间（裁剪空间）。
  • 输出一个四维坐标 (x, y, z, w)。
  • w 保留了“深度/远近”的信息 —— 所以你还不能丢掉它

透视除法：

才真正把 视锥体压缩为标准立方体，这时候你才得到了我们理解的“被投影到屏幕”的位置。
