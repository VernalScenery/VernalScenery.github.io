初识 Linux

1 Linux 组成

Linux 系统由 “Linux 系统内核” 和 “系统级应用程序” 两部分组成，是典型的 “底层核心 + 上层应用” 架构

  • 内核提供系统最核心的功能
    如：调度 CPU、调度内存、调度文件系统、调度网络通讯、调度 IO 等。
  • 系统级应用程序，可以理解为出厂自带程序，可供用户快速上手操作系统
    如：文件管理器、任务管理器、图片查看、音乐播放等。

Linux 内核是免费开源的，任何人都可以下载内核源码并查看且修改。
可以通过：The Linux Kernel Archives 去下载 Linux 内核

2 Linux 发行版

提供了内核 + 系统级程序的完整封装，称之为 Linux 发行版

  • RedHat: IBM 公司的产品, 功能强大, 收费.
  • CentOS: RedHat 的免费版.
  • Ubantu: 做界面做的比较好的, 适合小白.
  • 中标麒麟, 红旗: 国内的.

3 虚拟机

  • 通过虚拟化技术，在电脑内，虚拟出计算机硬件，并给虚拟的硬件安装操作系统，即可得到一台虚拟的电脑，称之为虚拟机
  • 在现有系统里，虚拟出一个独立的 “Linux 电脑环境 “

虚拟机和宿主机的关系：完全隔离

Linux 命令

1 Linux 命令基础格式

Linux 命令通用格式为 command [-options] [parameter] ，其中：

  • command 是命令本身（如 ls cp ）；
  • [-options] 是可选的 “选项”，用来调整命令细节（如 -l 让 ls 列详细列表，-r 让 cp 递归复制 ）；
  • [parameter] 是可选的 “参数”，指定命令作用的目标（如目录路径、文件名 ）。

示例：

  • ls -l /home/itheima， ls 是命令本身， -l 是选项， /home/itheima 是参数
    意思是以列表的形式，显示/home/itheima 目录内的内容
  • cp -r test1 test2 ，cp 是命令本身， -r 是选项， test1 和 test2 是参数
    意思是复制文件夹 test1 成为 test2

2 Linux 目录

Linux 的目录结构是一个树型结构

  • Windows 系统可以拥有多个盘符, 如 C 盘、D 盘、E 盘
  • Linux 没有盘符这个概念, 只有一个根目录 /, 所有文件都在它下面

目录名 作用说明
bin 存基础系统命令的可执行文件（如 ls 看目录、cp 复制、cat 查看文件内容 ），系统启动 / 修复时必需，普通用户也能执行。
boot 存系统启动相关文件（内核、启动引导程序 grub 等 ），删错会导致系统无法启动！
dev 把硬件设备抽象成文件（如硬盘是 /dev/sda，鼠标 / 键盘也对应这里的文件 ），程序通过操作这些 “文件” 控制硬件。
etc 存系统级、服务级的配置文件（网络配置、软件参数、用户权限等 ），改配置基本都动这里。
home 普通用户的 “家目录”，每个用户（如 alice/bob ）登录后，默认进入自己的 home 子文件夹，存个人数据、配置。
root 管理员（root 用户）的家目录，和 home 类似，但只有 root 能访问（普通用户没权限 ）。
run 存系统运行时的临时数据（如进程 PID 文件、服务运行状态 ），重启后会清空，用于快速读写。
sbin 存管理员（root）专用的系统命令（如 fsck 修复硬盘、ifconfig 改网络 ），普通用户一般没权限执行。
tmp 存临时文件，系统 / 程序运行中临时生成的文件放这，重启或定期会自动清理，谁都能读写（注意别存重要数据 ）。
usr 可理解为 “系统软件资源目录”，类似 Windows 的 Program Files：
- usr/bin：存非系统核心的命令（如编译器、办公软件 ）
- usr/local：用户手动编译安装的软件（自己装的程序默认放这里 ）
- usr/sbin：存面向管理员的非核心系统命令
var 存经常变化的文件（日志、数据库、缓存等 ）：
- var/log 是最常用的系统 / 程序日志（排错查问题全靠它 ）
- var/tmp 也是临时文件，但比 /tmp 更 “持久”（重启不一定删 ）

vi / vim 编辑器

1 概览

  • vi / vim 是 visual interface 的简称, 是 Linux 中最经典的文本编辑器
  • vim 是 vi 的加强版本，兼容 vi 的所有指令，相比 vi，增加了语法检查、代码高亮等功能，更适合程序员编写脚本和程序。

2 三种模式

模式名称 核心作用 操作限制 / 特点 模式切换方式
命令模式 执行 “快捷操作”（复制、粘贴、删除、跳转、替换…），是 vi/vim 的 “总指挥” 不能直接输入文字，按键 = 命令 - 启动 vi/vim 后默认进入
- 其他模式按 ESC 返回
输入模式 真正 “写内容” 的模式，像记事本一样自由编辑文本 只能打字、删字，无快捷命令 命令模式下按 i/a/o 等进入（看下方示例）
底线命令模式 执行 “文件级操作”（保存、退出、查找替换、设置语法…） 以 : 开头输入命令，按回车执行 命令模式下按 : 进入

  • 启动：vim 文件名 → 直接进入 命令模式（如 vim test.txt 打开文件）。
  • 命令模式 → 输入模式：
    • i：在光标当前位置前插入文字（常用！）。
    • a：在光标当前位置后插入文字。
    • o：在光标当前行下方新建一行并进入输入模式。
  • 输入模式 → 命令模式：按 ESC 键（退出编辑，回到 “总指挥” 模式）。
  • 命令模式 → 底线命令模式：按 : 键（左下角会出现 :，开始输入保存、退出等命令 ）。
  • 底线命令模式 → 命令模式：按 ESC 键（退出底线命令，回到命令模式 ）。

在底线命令模式下，输入以下命令后按回车：

  • :w → 保存文件（write）。
  • :q → 退出编辑器（quit）。
  • :wq → 保存并退出（常用！）。
  • :q! → 强制退出（不保存修改，小心用！）。

Linux 用户和权限

1 Linux 用户

1.1 概览

无论是 Windows、MacOS、Linux 均采用多用户的管理模式进行权限管理。

  • 在 Linux 系统中，拥有最大权限的账户名为：root（超级管理员）
  • 而在前期，我们一直使用的账户是普通的用户：itheima

普通用户权限边界

  • HOME 目录内自由：普通用户在自己的 HOME 目录（比如 itheima 的家目录）里，操作基本不受限（创建、修改文件等都能做 ）。
  • HOME 外受限：离开 HOME 目录，普通用户对多数系统目录，只有 “只读、执行” 权限，没法修改内容，防止误操作搞坏系统。

切换用户, “ 临时 “ 借调权限相关命令

1.2 切换用户

1.2.1 1. su 命令：用户切换工具

  • 作用：用于在 Linux 系统中切换用户身份，英文全称 Switch User 。
  • 语法：su [-] [用户名]
    • [-] 可选：带 - 会加载目标用户的环境变量（比如 root 的环境配置），建议日常使用时加上，让切换更完整；
    • [用户名] 可选：填要切换的用户，省略则默认切换到 root 。

1.2.2 切换后的退出方式

切换用户后，想回到上一个用户，有两种办法：

  • 命令：输入 exit ；
  • 快捷键：按 ctrl + d 。

1.2.3 切换的密码规则

  • 普通用户 → 其他用户（含 root）：需要输入目标用户的密码（比如普通用户切 root，要输 root 密码 ）；
  • root 用户 → 其他用户：无需密码，直接切换（因为 root 是超级管理员，权限最大 ）。

1.3  借调权限

1.3.1 sudo 命令基础

  • 作用：在已知 root 密码时，su 可切换到 root 获最大权限，但长期用 root 易致系统损坏，sudo 能让普通用户临时以 root 身份执行特定命令，平衡权限与安全。
  • 语法：sudo 其它命令 ，在普通命令前加 sudo ，可临时赋予该命令 root 授权 。
  • 限制：不是所有普通用户都能直接用 sudo，需为普通用户配置 sudo 认证 ，才有权限使用。

1.3.2 普通用户 sudo 认证配置步骤

  1. 切换到 root 并打开配置文件：切换到 root 用户，执行 visudo 命令，会通过 vi 编辑器打开 /etc/sudoers 文件（该文件管理 sudo 权限规则 ）。
  2. 添加权限规则：在 /etc/sudoers 文件最后添加配置

    其中 NOPASSWD: ALL 表示该用户（这里是 itheima ）用 sudo 执行命令时无需输入密码（也可根据需求配置需输密码的规则 ）。
  3. 保存配置：在 vi 编辑器中，通过 wq 操作保存修改。
  4. 验证配置：切回普通用户（如 itheima ），执行带 sudo 的命令（如 sudo ls /root、sudo mkdir /test ），命令会以 root 身份运行，且按配置（如无需输密码）执行 。

2 用户和用户组

2.1 概览

Linux 系统中可以：

  • 配置多个用户
  • 配置多个用户组
  • 用户可以加入多个用户组中

  • 针对用户组的权限（比如用户组 B 里的所有用户，统一有某个目录的访问权限 ）

4 !命令前缀 自动匹配并执行上一个以该前缀开头的命令

6 ctrl + a / ctrl + e 光标快速移动到命令行开头或结尾
7 ctrl + ← / ctrl + → 光标向左 / 右跳过一个单词
8 ctrl + l / clear 清空终端屏幕内容
9 ctrl + u 清空当前行已输入的命令
10 方向键↑ 调出上一条历史命令
11 tab 命令或文件名补全：单次按下自动补全，快速按两次显示备选选项
  • history >> 1.txt 把历史记录追加到 1.txt 文件中

3 服务管理：systemctl 命令

用于控制 Linux 系统的服务（含系统内置、部分第三方软件服务 ），实现服务的启动、停止、状态查看、开机自启配置

  • start 启动
  • stop 关闭
  • status 查看状态
  • enable 开启开机自启
  • disable 关闭开机自启

win10 系统下的服务如下图：

Linux 系统内置服务，用于支撑基础功能，例如：

  • NetworkManager/network：主、副网络服务，管理网络连接
  • firewalld：防火墙服务，管控网络访问规则
  • sshd：SSH 服务，支持远程登录（如 FinalShell 工具依赖此服务 ）

除了内置的服务以外,部分第三方软件安装后也可以以 systemctl 进行控制｡

  • Apache 服务器（httpd）：

4 软连接 和 硬链接

  • 软连接：类似于 windows 的快捷方式
  • 硬链接：类似于 动态备份 (修改 a.txt, c.txt 也会同步变化)

4.1 ln 命令创建软连接

在系统中创建软链接, 可以将文件､文件夹链接到其它位置｡

4.2 ln 命令创建硬连接

不加 -s 就是创建硬连接

  • 物理存储：硬链接相当于为同一个文件创建多个 “别名”，这些别名（链接文件）与原文件共享相同的 inode（索引节点，存储文件元数据和物理地址）。
    • 例如：原文件 a.txt 的 inode 指向磁盘某区域，硬链接 c.txt 通过相同的 inode 指向同一区域，两者本质是同一文件的不同引用。
  • 修改同步：修改 a.txt 或 c.txt 任一文件，内容会实时同步到另一个，因为它们操作的是同一物理数据。

这里 2 1 2 是硬连接数，硬链接数表示有多少个文件名指向同一个 inode(索引节点)。

  • 当创建硬链接时，硬链接数加 1；删除文件时，硬链接数减 1。
  • 当硬链接数为 0 且无进程引用时，数据才会被真正删除。

5.2 端口号

  • ip：设备在网络中的唯一标识。
  • 端口号：程序在设备上的唯一标识

Linux 系统支持 65535 个端口 ，按用途分为 3 大类 ：

  • 公认端口（1~1023）：
    • 用途：系统内置服务、知名程序 “预留” 使用（比如 SSH 用 22 端口、HTTPS 用 443 端口 ）。
    • 建议：非特殊需求，别占用这类端口，避免冲突。
  • 注册端口（1024~49151）：
    • 用途：可 “随意使用”，给普通程序 / 服务做长期绑定（比如图里计算机 B 的微信，长期用 5678 端口等别人连 ）。
  • 动态端口（49152~65535）：
    • 用途：程序对外联网时，临时用的 “出口” ，不长期绑定（比如图里计算机 A 的微信连 B 时，临时用 50001 端口 ）。

  • A 发起连接 → 用 动态端口（50001） 当 “临时出口”。
  • B 接收连接 → 用 注册端口（5678） 长期绑定，等别人连。

简单说，Linux 端口分三类，各司其职：公认端口给系统 / 知名服务，注册端口给普通程序长期用，动态端口给程序临时联网当出口

可以通过 netstat 命令，查看指定端口的占用情况

8 压缩 和 解压缩

市面有很多压缩格式，不同系统常用的还不一样：

  • 跨平台通用：zip（Linux/Windows/MacOS 都能用 ）。
  • Windows 专属常用：7zip、rar（WinRAR、Bandizip 这些软件支持 ）。
  • Linux/MacOS 专属常用：tar、gzip 。

这里我们主要学习如何在 Linux 系统中操作: tar､gzip､zip 这三种压缩格式

tar 命令（处理 .tar、.tar.gz 格式）

Linux/Mac 里最常用的 “打包 + 压缩” 工具，能处理两种格式：

  • .tar：只打包，不怎么压缩体积，相当于 “文件夹打包”。
  • .tar.gz：用 gzip 算法压缩，体积大幅减小，常用！

语法：tar [选项] 目标文件 源文件

选项 作用 场景举例
-c 创建压缩包（打包 / 压缩） tar -cvf xxx.tar 文件
-x 解压压缩包（解包 / 解压） tar -xvf xxx.tar
-v 显示过程（压缩 / 解压时打印日志） 加了 -v 能看到实时进度
-f 指定文件名（必须放最后！） 所有 tar 命令都得带 -f
-z 用 gzip 压缩 / 解压（处理 .tar.gz） tar -zcvf xxx.tar.gz 文件
-C 指定解压目录（解压时用） tar -xvf xxx.tar -C /目标路径

-f 必须放最后！比如 tar -zcvf xxx.tar.gz 文件 是对的，tar -fzv ... 就会报错。
-z 负责启用 gzip 压缩，处理 .tar.gz 时必须加。
-C 记得和其他参数分开写

常用操作示例

  • 压缩：
    • 打包成 .tar：tar -cvf test.tar 1.txt 2.txt（把 1.txt、2.txt 打包成 test.tar ）
    • 压缩成 .tar.gz：tar -zcvf test.tar.gz 1.txt 2.txt（用 gzip 算法压缩，体积更小 ）
  • 解压：
    • 解压 .tar 到当前目录：tar -xvf test.tar
    • 解压 .tar 到指定目录：tar -xvf test.tar -C /home/xxx
    • 解压 .tar.gz 到指定目录：tar -zxvf test.tar.gz -C /home/xxx

zip 命令（处理 .zip 格式）

跨平台通用（Linux/Windows/Mac 都支持），压缩成 .zip 格式，适合和其他系统交互时用。

语法： zip [选项] 压缩包名 源文件

选项 作用 场景举例
-r 递归压缩（含文件夹时必须加！） zip -r test.zip 文件夹/

压缩包含文件夹的内容时，必须加 -r ，否则文件夹里的子文件不会被压缩进去。

常用操作示例

  • 压缩：
    • 压缩多个文件：zip test.zip a.txt b.txt（把 a.txt、b.txt 压成 test.zip ）
    • 压缩文件夹 + 文件：zip -r test.zip 文件夹/ a.txt（递归压缩文件夹和文件 ）
  • 解压（用 unzip 命令）：
    • 解压到当前目录：unzip test.zip
    • 解压到指定目录：unzip test.zip -d /home/xxx

解压到指定目录：tar 用 -C ，unzip 用 -d
