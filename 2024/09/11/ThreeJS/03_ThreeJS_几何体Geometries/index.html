几何体

1 BufferGeometry(基类)

是面片、线或点几何体的有效表述。包括顶点位置，面片索引、法相量、颜色值、UV 坐标和自定义缓存属性值。使用 BufferGeometry 可以有效减少向 GPU 传输上述数据所需的开销。

1.1 构造函数

BufferGeometry()：创建一个新的 BufferGeometry. 同时将预置属性设置为默认值.

threejs的长方体BoxGeometry、球体SphereGeometry等几何体都是基于BufferGeometry类构建的，BufferGeometry是一个没有任何形状的空几何体，你可以通过BufferGeometry自定义任何几何形状，具体一点说就是定义顶点数据

.attributes : Object：通过 hashmap 存储该几何体相关的属性，hashmap 的 id 是当前 attribute 的名称，值是相应的 buffer

.index : BufferAttribute：顶点索引

1.3 方法

1.3.1 .setFromPoints ( points : Array ) : this

该方法可以把数组pointsArr中坐标数据提取出来赋值给几何体。具体说就是把pointsArr里面坐标数据提取出来，赋值给geometry.attributes.position属性

三维向量Vector3表示顶点坐标
二维向量Vector2表示顶点坐标
Curve类可通过.getPoints 方法获取坐标数据

1.3.2  旋转、缩放、平移几何体

BufferGeometry通过.scale()、.translate()、.rotateX()、.rotateY()等方法可以对几何体本身进行缩放、平移、旋转,这些方法本质上都是改变几何体的顶点数据。

2 BufferAttribute(基类)

这个类用于存储与BufferGeometry相关联的 attribute（例如顶点位置向量，面片索引，法向量，颜色值，UV坐标以及任何自定义 attribute ）。 利用 BufferAttribute，可以更高效的向GPU传递数据。

如下 ：

2.1 构造函数

BufferAttribute( array : TypedArray, itemSize : Integer, normalized : Boolean )

  • array ： 顶点数组
  • itemSize：几个为一组
  • normalized -- (可选)
    指明缓存中的数据如何与GLSL代码中的数据对应。例如，如果array是 UInt16Array类型，且normalized的值是 true，则队列中的值将会从 0 - +65535 映射为 GLSL 中的 0.0f - +1.0f。

  • . array: TypedArray

3 常见几何体(子类)

threejs默认的大部分几何体都有三角形的顶点索引数据，可以通过如下代码查看其顶点数据和索引数据

3.1 网格模型(三角形概念)

网格模型 Mesh 其实就一个一个三角形(面)拼接构成。使用网格模型 Mesh 渲染几何体 geometry，就是几何体所有顶点坐标三个为一组，构成一个三角形，多组顶点构成多个三角形，就可以用来模拟表示物体的表面。

3.1.1 正反面

  • 正面：逆时针
  • 反面：顺时针

空间中一个三角形有正反两面，那么 Three.js 的规则是如何区分正反面的？非常简单，你的眼睛(相机)对着三角形的一个面，如果三个顶点的顺序是逆时针方向，该面视为正面，如果三个顶点的顺序是顺时针方向，该面视为反面。

3.2 几何体细分数

即为用多少个三角形去逼近

Three.js很多几何体都提供了细分数相关的参数，这里以矩形和球体为例

3.2.1 矩形平面几何体PlaneGeometry

矩形平面几何体至少需要两个三角形拼接而成。

把一个矩形分为2份，每个矩形2个三角形，总共就是4个三角形

把一个矩形分为4份，每个矩形2个三角形，总共就是8个三角形

3.2.2 球体SphereGeometry

球体SphereGeometry参数2、3分别代表宽、高度两个方向上的细分数，默认32,16，具体多少以你所用版本为准。

如果球体细分数比较低，表面就不会那么光滑。
