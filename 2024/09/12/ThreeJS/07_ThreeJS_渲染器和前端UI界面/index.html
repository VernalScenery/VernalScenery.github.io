ThreeJS_渲染器和前端UI界面

1 Canvas 画布布局

设置 canvas 画布的尺寸，注意在浏览器窗口文档区域整体宽高基础上，减掉其他顶部和左侧 div 元素和布局间隙的尺寸。

Canvas 随着窗口变化同理

2 UI 交互界面与 Canvas 画布叠加

利用绝对定位和事件绑定实现

3 Canvas 画布背景透明度

canvas 画布完全透明，可以透过 canvas 画布看到画布后面叠加的 HTML 元素图文，呈现出来一种三维模型悬浮在网页上面的效果

三种方法设置：

3.1 背景透明alpha: true

通过Three.js渲染器WebGLRenderer的alpha属性值设置为true就可以，WebGL渲染器的alpha属性默认值是false。

3.2 .setClearColor()方法

.setClearColor()方法的参数2，可以用来设置背景颜色透明度。

3.3 .setClearAlpha()方法

改变背景透明度值

4 Canvas 画布保存为图片

4.1 配置webgl渲染器preserveDrawingBuffer:true

4.2 Cavnas方法.toDataURL()

Canvas画布通过.toDataURL()方法可以获取画布上的像素信息。canvas.toDataURL("image/png");表示以png格式获取像素数据，可以直接赋值给超链接元素a的.herf属性下载到本地

5 模型冲突

两个模型重叠，电脑GPU分不清谁在前谁在后，这种现象，可以称为深度冲突Z-fighting。

适当偏移，解决深度冲突，偏移尺寸相对模型尺寸比较小，视觉上两个平面近似还是重合效果。

5.1 webgl 渲染器设置对数深度缓冲区

当一个三维场景中有一些面距离比较近，有深度冲突，你可以尝试设置webgl渲染器设置对数深度缓冲区logarithmicDepthBuffer: true来优化或解决。logarithmicDepthBuffer: true作用简单来说，就是两个面间距比较小的时候，让 threejs 更容易区分两个面，谁在前，谁在后。

有一点要注意，当两个面间隙过小，或者重合，你设置 webgl 渲染器对数深度缓冲区也是无效的。
