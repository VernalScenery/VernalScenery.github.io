2.3 WebGLPointsLayer 点精灵

WebGLPointsLayer 在地图上显示大量点。该层具有 JSON 格式的样式，允许对最终表示进行一定级别的自定义。可以使用以下运算符：

2.3.1 读取运算符

  1. **[‘band’, bandIndex, xOffset, yOffset]**（仅限瓦片图层）：
    • 从源数据的指定波段（band）中获取像素值。波段索引（bandIndex）从1开始。
    • 返回的像素值范围在0到1之间。
    • xOffset 和 yOffset 是可选的，允许在x和y方向上指定像素偏移量，用于从相邻像素中采样数据（仅限WebGL）。
    • TileImage 源通常有四个波段：红色、绿色、蓝色和透明度（alpha）。
    • DataTileSource 源的波段数量取决于底层数据源和配置。
  2. **[‘get’, attributeName]**：
    • 获取要素（feature）的属性值，类似于 feature.get('attributeName')。
    • 用于从要素的属性中检索数据。
  3. **[‘get’, attributeName, keyOrArrayIndex, …]**（仅限Canvas）：
    • 访问要素属性的嵌套属性或数组项。
    • 当指定键或索引处没有内容时，返回 undefined。
    • 允许在Canvas渲染过程中深入要素属性结构。
  4. **[‘geometry-type’]**：
    • 返回要素的几何类型作为字符串，如 'LineString'、'Point' 或 'Polygon'。
    • 对于 Multi* 类型，返回其单数形式（例如，'MultiLineString' 返回 'LineString'）。
    • 圆形几何体返回为 'Polygon'。
    • GeometryCollection 几何体返回集合中第一个找到的几何体的类型（仅限WebGL）。
  5. **[‘resolution’]**：
    • 返回当前的分辨率（通常指地图上一个像素代表的地面距离）。
    • 用于根据当前缩放级别动态调整样式或数据展示。
  6. **[‘time’]**（仅限WebGL）：
    • 返回自图层创建以来的时间（以秒为单位）。
    • 通常用于动画或基于时间的图层更新。
  7. **[‘var’, ‘varName’]**：
    • 从样式变量中检索值。
    • 如果变量未定义，将抛出错误。
    • 用于在样式中重用或动态调整值。
  8. **[‘zoom’]**（仅限WebGL）：
    • 返回当前的缩放级别。
    • 用于根据缩放级别动态调整样式或数据展示。

2.3.2 变换运算符

  1. ['case', condition1, output1, ...conditionN, outputN, fallback]
    这个函数用于根据一系列条件选择输出。它会按顺序检查每个条件（conditionX），如果某个条件为真（boolean true），则返回对应的输出（outputX）。如果所有条件都不满足，则返回回退值（fallback）。 示例：['case', ['==', ['get', 'type'], 'restaurant'], 'restaurant icon', ['==', ['get', 'type'], 'park'], 'park icon', 'default icon']
    如果属性type等于restaurant，则返回'restaurant icon'；如果等于park，则返回'park icon'；否则返回'default icon'。
  2. ['match', input, match1, output1, ...matchN, outputN, fallback]
    这个函数用于将输入值与一系列匹配值进行比较，返回与第一个有效匹配相对应的输出。如果没有匹配项，则返回回退值。输入和匹配值必须类型相同（数字或字符串）。 示例：['match', ['get', 'type'], 'restaurant', 'restaurant icon', 'park', 'park icon', 'default icon']
    如果type属性的值为restaurant，则返回'restaurant icon'；如果为park，则返回'park icon'；否则返回'default icon'。
  3. ['interpolate', interpolation, input, stop1, output1, ...stopN, outputN]
    这个函数用于在两个或多个点之间进行插值，以获取输入值对应的输出值。插值方法可以是线性（'linear'）或指数（'exponential'，需指定基数）。 示例（线性插值）： ['interpolate', ['linear'], ['get', 'elevation'], 0, '#0000ff', 1000, '#ff0000']
    如果elevation的值在0到1000之间，则颜色从蓝色（#0000ff）渐变到红色（#ff0000）。
  4.  ['string', value1, value2, ...]
    这个函数返回列表中第一个评估为字符串的值。它常用于确保某个值（如从数据源获取的）是字符串类型，并提供默认值。 示例（Canvas API）：['string', ['get', 'name'], 'Unknown']
    如果name属性存在且为字符串，则返回其值；否则返回'Unknown'。
  5.  ['number', value1, value2, ...]
    这个函数返回列表中第一个评估为数字的值。它同样用于确保值是数字类型，并提供默认值。 示例（Canvas API）：['number', ['get', 'population'], 0]
    如果population属性存在且为数字，则返回其值；否则返回0。
  6.  ['coalesce', value1, value2, ...]
    这个函数返回列表中第一个非空（null或undefined）的值。它常用于提供默认值，特别是在数据源可能返回空值的情况下。 示例（Canvas API）：['coalesce', ['get', 'optionalField'], 'default value']
    如果optionalField存在且不为null或undefined，则返回其值；否则返回'default value'。

2.3.3 逻辑运算符

  • ['<', value1, value2]：如果value1严格小于value2，则返回true，否则返回false。
    示例：['<', 5, 10] 返回 true。
  • ['<=', value1, value2]：如果value1小于或等于value2，则返回true，否则返回false。
    示例：['<=', 5, 5] 返回 true。
  • ['>', value1, value2]：如果value1严格大于value2，则返回true，否则返回false。
    示例：['>', 10, 5] 返回 true。
  • ['>=', value1, value2]：如果value1大于或等于value2，则返回true，否则返回false。
    示例：['>=', 10, 10] 返回 true。
  • ['==', value1, value2]：如果value1等于value2，则返回true，否则返回false。
    示例：['==', 'hello', 'hello'] 返回 true。
  • ['!=', value1, value2]：如果value1不等于value2，则返回true，否则返回false。
    示例：['!=', 5, 10] 返回 true。
  • ['!', value1]：如果value1是true或大于0，则返回false；否则返回true。
    示例：['!', 0] 返回 true。
  • ['all', value1, value2, ...]：如果所有输入都是true，则返回true；否则返回false。
    示例：['all', true, true, false] 返回 false。
  • ['any', value1, value2, ...]：如果任何输入是true，则返回true；否则返回false。
    示例：['any', false, false, true] 返回 true。
  • ['between', value1, value2, value3]：如果value1在value2和value3之间（包括边界值），则返回true；否则返回false。
    示例：['between', 5, 1, 10] 返回 true。
  • ['in', needle, haystack]：如果needle在haystack数组中，则返回true；否则返回false。注意haystack必须是数字或字符串的数组，且目前只支持字面量数组。
    示例：['in', 'a', ['a', 'b', 'c']] 返回 true。
    对于字符串数组，使用['literal', ...]来明确指定这是一个数组字面量，特别是在表达式解析器中可能需要这种区分。

2.3.4 转换运算符

  1. [‘array’, value1, …valueN]
    • 创建一个由数字值组成的数组。
    • 目前，这个表达式的值数量（N）只能是2、3或4（仅限WebGL）。
    • 这在需要指定一组固定数量的数值时非常有用，例如，在定义颜色数组或数据数组时。
  2. [‘color’, red, green, blue, alpha] 或 [‘color’, shade, alpha]
    • 创建一个颜色值。
    • red、green、blue 或 shade（代表灰度值）组件必须是0到255之间的数值。
    • alpha（透明度）是可选的，如果不指定，则默认为1（完全不透明）。如果指定，alpha 必须是0到1之间的数值。
    • 第一个形式允许直接指定红、绿、蓝和透明度值。
    • 第二个形式允许通过单个灰度值（shade）和可选的透明度值来创建颜色。
    • 这在需要动态设置颜色时非常有用，例如在根据数据值改变颜色时。
  3. [‘palette’, index, colors]
    • 从颜色数组中使用给定的索引选择一个颜色值。
    • index 表达式必须计算为一个数字，用于从 colors 数组中选择颜色。
    • colors 数组中的项目必须是字符串，表示十六进制颜色（如'#86A136'）、使用rgba[a]函数表示法的颜色（如'rgb(134, 161, 54)'或'rgba(134, 161, 54, 1)'）、命名颜色（如'red'）或包含3（[r, g, b]）或4（[r, g, b, a]）个值的数组字面量（其中r、g、b在0-255范围内，a在0-1范围内）。
    • 这在需要根据索引（如数据分类）从预定义颜色集合中选择颜色时非常有用。
  4. [‘to-string’, value]
    • 将输入值转换为字符串。
    • 如果输入是布尔值，则结果为"true"或"false"。
    • 如果输入是数字，则根据ECMAScript语言规范的"NumberToString"算法将其转换为字符串。
    • 如果输入是颜色（但请注意，这个用法可能不是所有环境都支持，因为颜色通常作为对象或特定格式处理），则尝试将其转换为"rgba(r,g,b,a)"形式的字符串（但请注意，这通常是Canvas特有的用法，因为WebGL通常使用不同的颜色表示法）。
    • 这个函数在需要将数据值转换为字符串表示以进行进一步处理或显示时非常有用。

2.3.5 操作符求值

在地图渲染或类似环境中，值（Values）可以是字面量（Literals）或另一个操作符（Operator），并且它们会被递归地评估。这意味着，如果一个值本身是一个操作符，那么它的操作数（operands）也会被评估，直到所有的值都是字面量为止。以下是字面量值可以具有的类型：

  1. boolean：布尔值，可以是true或false。
  2. number：数值，可以是整数或浮点数，用于表示数量、大小、比例等。
  3. **number[]**：数值数组，其长度只能是2、3或4（这通常受限于WebGL的限制，因为WebGL中的某些函数或API可能只接受特定长度的数组作为参数）。这种数组通常用于表示向量（如二维或三维空间中的点或方向）或颜色（在某些情况下，如果颜色以RGBA形式表示，并且每个分量是一个0到255之间的整数）。
  4. string：字符串，用于表示文本或标识符。在地图渲染环境中，字符串可能用于定义图层名称、属性键、样式规则等。
  5. Color：颜色值，尽管在严格意义上它可能不是一个字面量类型，但在某些上下文中（如WebGL或Canvas API），颜色可以以特定的格式（如十六进制字符串'#RRGGBB'、'#RRGGBBAA'、rgb(r,g,b)、rgba(r,g,b,a)或数组[r, g, b, a]）直接表示。这里的Color可以看作是一个特殊的值类型，它封装了表示颜色的数据。

需要注意的是，在实际编程中，Color可能不是一个内置的字面量类型，而是由库或框架提供的一个类、结构体或数据类型。因此，当你需要表示颜色时，你需要根据你所使用的库或API的文档来正确地创建和使用颜色对象或值。
