  • value: 属性的值，默认为 undefined。
  • writable: 表示是否可以通过赋值操作改变属性的值，默认为 false。

2.3 Object的一些方法对 obj 实现限制

2.3.1 Object.preventExtensions

Object.preventExtensions 是 JavaScript 中的一个方法，用于防止向对象添加新的属性。这意味着一旦对象被 Object.preventExtensions 处理后，就无法再向该对象添加任何新的属性，但仍然可以修改现有属性的值（如果这些属性是可写的），并且可以删除现有属性。

属性描述符的变化

当对象被 Object.preventExtensions 处理后，对象本身不会有任何变化，只是不能再添加新的属性。现有的属性描述符保持不变。

2.3.2 Object.seal

Object.seal 是 JavaScript 中的一个方法，用于密封一个对象。密封一个对象意味着该对象将不能再添加新的属性或删除现有的属性，但仍然可以修改现有属性的值（如果这些属性是可写的）。

作用

  • 防止扩展：不能向对象添加新属性。
  • 防止删除：不能从对象中删除现有属性。
  • 保留写权限：如果属性是可写的，则仍可以修改其值。
检查对象是否被密封

您可以使用 Object.isSealed 方法来检查一个对象是否已经被密封

属性描述符的变化

当对象被密封后，所有属性的 configurable 描述符都会被设置为 false，而 writable 和 enumerable 描述符保持不变。

通过 Object.seal 方法，您可以确保对象的结构不会被意外修改，但仍允许对现有属性的值进行更改。这对于保护对象的状态非常有用。

2.3.3 Object.freeze

Object.freeze 是 JavaScript 中的一个方法，用于冻结一个对象。冻结一个对象意味着该对象将不能再添加新的属性、删除现有的属性，也不能修改现有属性的值（包括数据描述符和访问器描述符）。此外，对于嵌套的对象，如果需要完全冻结整个对象树，还需要递归地调用 Object.freeze。

作用

  • 防止扩展：不能向对象添加新属性。
  • 防止删除：不能从对象中删除现有属性。
  • 防止修改：不能修改现有属性的值。
  • 防止配置：不能修改现有属性的描述符（如 writable、configurable 等）。
完全冻结嵌套对象

为了确保整个对象树都被冻结，需要递归地调用 Object.freeze

检查对象是否被冻结

您可以使用 Object.isFrozen 方法来检查一个对象是否已经被冻结

属性描述符的变化

当对象被 Object.freeze 处理后，所有属性的 writable 和 configurable 描述符都会被设置为 false，而 enumerable 描述符保持不变。

3 创建对象的方案

3.1 工厂模式

工厂模式的缺点

  1. 类型识别问题：工厂方法创建的对象在打印时，其类型都是 Object 类型，这使得很难区分不同类型的对象。
  2. 缺乏统一的原型链：工厂方法创建的对象没有共享同一个原型链，导致无法有效地继承方法和属性。

3.2 构造函数

在 JavaScript 中，构造函数是一种特殊类型的函数，用于创建和初始化对象。构造函数的主要特点是它们通常以首字母大写的形式命名，并且通过 new 操作符来调用。下面详细介绍 JavaScript 构造函数的各个方面。

3.2.1 基本概念

定义
  • 构造函数：一种特殊的函数，用于创建特定类型的对象。
  • 语法：构造函数的定义与其他函数类似，但通常首字母大写，以示区分。

3.2.2 使用 new 操作符

步骤

当使用 new 操作符调用构造函数时，JavaScript 引擎会执行以下步骤：

  1. 创建一个新对象：生成一个新的空对象。
  2. 设置原型：将新对象的 [[Prototype]] 属性指向构造函数的 prototype 对象。
  3. **绑定 this**：将构造函数内部的 this 绑定到这个新对象。
  4. 执行构造函数代码：执行构造函数中的代码，给新对象添加属性和方法。
  5. 返回新对象：如果没有显式返回其他对象，则返回新创建的对象。

3.2.3 原型链

原型
  • prototype 属性：每个函数都有一个 prototype 属性，它是一个对象，包含可以被该函数的所有实例共享的属性和方法。
  • 继承：通过原型链，子对象可以继承父对象的方法和属性。

3.2.4 返回值

  • 默认返回：如果构造函数没有显式返回任何值或返回的是原始值（如数字、字符串等），则返回新创建的对象。
  • 显式返回对象：如果构造函数显式返回一个对象，则返回该对象而不是新创建的对象。

3.2.5 类型识别

  • instanceof 运算符：用于检查对象是否是由某个构造函数创建的。

3.2.6 构造函数 vs 工厂函数

  • 工厂函数：普通函数，返回一个新对象。缺点是无法识别对象的具体类型。
  • 构造函数：通过 new 调用，可以识别对象的具体类型。

4 原型

4.1 隐式原型__对象的原型

在 JavaScript 中，每个对象都有一个内部属性 [[Prototype]]，通常称为对象的原型。原型是一个对象，它允许对象继承其他对象的属性和方法。以下是关于原型的一些关键点：

  1. 原型的概念：

    • 每个对象都有一个 [[Prototype]] 属性，指向另一个对象，这个对象就是原型。
    • 原型对象也可以有自己的原型，形成一个原型链。
    • 最终，原型链会终止于 null，因为 null 没有原型。

  2. 查看原型：

    • 早期的 ECMAScript 规范没有提供标准的方法来访问 [[Prototype]]，但大多数浏览器提供了 __proto__ 属性。

    • ES5 之后，推荐使用 Object.getPrototypeOf 方法来获取对象的原型。

  3. 原型的作用：

    • 当从一个对象中获取某个属性时，会触发 [[Get]] 操作。

    • [[Get]] 操作的步骤如下：

      1. 在当前对象中查找对应的属性，如果找到则直接返回。

      2. 如果未找到，则沿着原型链向上查找，直到找到属性或到达原型链的末端（即 null）。

  4. 原型链：

    • 原型链是指从一个对象开始，通过其原型对象，再到原型对象的原型对象，依次向上查找，直到找到 null。

  5. 设置原型：

    • 可以使用 Object.create 方法创建一个新对象，并指定其原型。

    • 也可以在对象字面量中使用 __proto__ 属性（虽然不推荐）。

4.2 显示原型__函数的原型

在 JavaScript 中，每个函数都有一个 prototype 属性，这个属性是一个对象，称为显示原型。显示原型的主要作用是定义可以被通过该函数创建的对象继承的属性和方法。

4.2.1 特点

  1. 每个函数都有 prototype 属性：
    • 即使是空函数，也会有一个 prototype 属性。
    • prototype 属性默认是一个对象，包含一个 constructor 属性，指向该函数本身。
  2. 用于继承：
    • 当使用 new 关键字调用函数创建对象时，新创建的对象的 __proto__ 属性会被设置为该函数的 prototype 属性。
    • 这样，新创建的对象就可以访问 prototype 上定义的属性和方法。
  3. 修改 prototype 影响所有实例：
    • 修改函数的 prototype 属性会影响所有通过该函数创建的对象。

4.3 隐式原型和显式原型的区别

在 JavaScript 中，对象的原型可以通过两种方式来表示和访问：隐式原型 (__proto__) 和显式原型 (prototype)。下面是它们的区别：

4.3.1 隐式原型 (__proto__)

  • 定义：每个对象都有一个 __proto__ 属性，它指向该对象的原型对象。
  • 用途：用于访问对象的原型，从而查找继承的属性和方法。
  • 特点：
    • __proto__ 是一个对象的属性，不是所有环境中都支持（如某些旧版本的浏览器）。
    • 通过 Object.getPrototypeOf 方法可以更安全地获取对象的原型。

4.3.2 显式原型 (prototype)

  • 定义：每个函数都有一个 prototype 属性，它是一个对象，用于作为通过该函数创建的对象的原型。
  • 用途：用于在构造函数中定义可以被实例继承的属性和方法。
  • 特点：
    • prototype 是函数的属性，不是对象的属性。
    • 通过 new 关键字创建的对象，其 __proto__ 属性会指向构造函数的 prototype 对象。

4.3.3 总结

  • **隐式原型 (__proto__)**：每个对象都有，指向该对象的原型对象，用于属性和方法的继承。
  • **显式原型 (prototype)**：每个函数都有，用于定义可以被实例继承的属性和方法，通过 new 关键字创建的对象的 __proto__ 会指向构造函数的 prototype。

4.4 构造函数创建对象的内存表现

  • **构造函数 Person**：
    • Person 函数本身占用一块内存。
    • Person.prototype 占用另一块内存，指向原型对象。
  • **实例对象 p1 和 p2**：
    • 每个实例对象 p1 和 p2 占用各自的内存。
    • 每个实例对象的 __proto__ 属性指向 Person.prototype。

5 原型链

5.1 原型链的基本概念

  • **原型 (prototype)**：每个函数都有一个 prototype 属性，指向一个对象，这个对象可以被该函数创建的所有实例共享。
  • **隐式原型 (__proto__)**：每个对象都有一个 __proto__ 属性，指向它的原型对象。

5.2 原型链的查找机制

  • 当访问一个对象的属性时，JavaScript 引擎会首先在该对象本身查找该属性。
  • 如果没有找到，引擎会沿着 __proto__ 链向上查找，直到找到该属性或到达原型链的顶端（即 null）。

5.3 顶层原型

原型链最顶层的原型对象就是Object的原型对象

顶层原型的来源

在 JavaScript 中，顶层原型指的是所有对象最终继承的原型对象，即 Object.prototype。以下是关于顶层原型的一些关键点：

  1. **Object.prototype**：
    • Object.prototype 是所有对象的顶层原型。
    • 它是 Object 构造函数的 prototype 属性值。
    • 所有通过 Object 构造函数创建的对象，或者使用对象字面量 {} 创建的对象，最终都会继承自 Object.prototype。
  2. __proto__ 属性：
    • 每个对象都有一个 __proto__ 属性，指向其原型对象。
    • 对于普通对象，__proto__ 默认指向 Object.prototype。
    • Object.prototype 的 __proto__ 属性值为 null，表示原型链的终点。
  3. constructor 属性：
    • Object.prototype 上有一个 constructor 属性，指向 Object 构造函数。
  4. Object.prototype 的属性和方法：
    • Object.prototype 提供了一些常用的方法，如 toString、valueOf、hasOwnProperty 等。

  1. 继承的属性可见:

    • 子类可以在创建实例时向父类构造函数传递参数，增强了灵活性。

    • 由于 Student.prototype 被设置为 p（一个 Person 实例），子类实例的原型对象上会多出一些不必要的属性，这些属性在子类实例中已经存在。

8.1 原型式继承-对象

  • 定义一个内部构造函数 Fn，它没有任何自定义属性或方法。

8.2 寄生式继承-对象

相较于原型式继承仅仅返回一个空对象，寄生式继承则在新对象上添加一些新的属性或方法，增强新对象的功能。

  • 新对象不仅继承了原型对象的属性和方法，还在创建过程中添加了新的属性和方法。
  • 每次调用 createStudent 函数时，都会创建一个新的对象，并且每个对象都有自己的 name 和 studying 方法。
  • 这种方式使得每个创建的对象都可以有自己独特的属性和方法，而不仅仅是共享原型对象的属性和方法。

实例对象

    • 原型对象默认有一个 constructor 属性，指向该函数本身。

10 JS中的 Function 和 Object

JS 中的所有函数内部实现都是由new Function得到的，故函数也是一个实例对象

  • 作为实例对象，其具有__proto__属性，指向 Function的prototype
  • 作为一个函数，其具有prototype属性，它是所有通过 new Foo() 创建的实例的原型。
  • Foo.prototype 是一个对象，它是所有通过 new Foo() 创建的实例的原型。
  • Foo.__proto__ 是 Foo 函数对象的原型，指向 Function.prototype。
  • Foo.prototype.constructor 指向 Foo 构造函数。
  • Foo.__proto__.constructor 指向 Function 构造函数。
  • Foo.prototype 和 Foo.__proto__ 是不同的对象，因此它们不相等。
  • Foo.prototype 和Function的prototype是一个对象

而Function的原型是全局Object

首先

Object和Function作为函数实例，其__proto__都指向 Function的prototype

Object和Function作为一个函数，其具有prototype属性

  • 一个是所有通过 new Function() 创建的实例的原型，其constructor指向 Function 构造函数。
