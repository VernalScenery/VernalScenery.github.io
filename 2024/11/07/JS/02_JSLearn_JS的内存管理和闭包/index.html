内存管理和垃圾回收

1 认识内存管理

不管什么样的编程语言，在代码的执行过程中都是需要给它分配内存的，不同的是某些编程语言需要我们自己手动的管理内存，某些编程语言会可以自动帮助我们管理内存：

不管以什么样的方式来管理内存，内存的管理都会有如下的生命周期：

  • 第一步：分配申请你需要的内存（申请）；
  • 第二步：使用分配的内存（存放一些东西，比如对象等）；
  • 第三步：不需要使用时，对其进行释放；

2 JS 的内存管理

在JavaScript中，内存分配主要涉及到两个方面：栈（Stack）和堆（Heap）。

  • 栈（Stack）:
    • 栈用于存储原始值（如布尔值、数字、字符串、null、undefined）。
    • 函数调用时，参数和局部变量会被压入栈中，当函数执行完毕后，这些数据会从栈中弹出。
    • 栈的操作遵循“后进先出”（LIFO, Last In First Out）的原则。
  • 堆（Heap）:
    • 堆用于存储对象，包括数组、对象字面量等复杂数据结构。
    • 当创建一个对象时，会在堆中分配内存，并返回一个指向该对象的引用，这个引用通常存储在栈中。
    • 堆的内存管理相对复杂，涉及垃圾回收机制来释放不再使用的对象占用的内存。

3 垃圾回收

垃圾回收的英文是Garbage Collection，简称GC

JavaScript 是使用垃圾回收的语言，也就是说执行环境负责在代码执行时管理内存。

基本思路很简单：确定哪个变量不会再使用，然后释放它占用的内存。这个过程是周期性的，即垃圾回收程序每隔一定时间（或者说在代码执行过程中某个预定的收集时间）就会自动运行。

垃圾回收程序会周期性运行，如果内存中分配了很多变量，则可能造成性能损失，因此垃圾回收的时间调度很重要。

3.1 标记策略

垃圾回收程序必须跟踪记录哪个变量还会使用，以及哪个变量不会再使用，以便回收内存。如何标记未使用的变量也许有不同的实现方式。不过，在浏览器的发展史上，用到过两种主要的标记策略：标记清除和引用计数。

3.1.1 标记清除

JavaScript 最常用的垃圾回收策略是标记清除（mark-and-sweep）。当变量进入上下文，比如在函数内部声明一个变量时，这个变量会被加上存在于上下文中的标记。而在上下文中的变量，逻辑上讲，永远不应该释放它们的内存，因为只要上下文中的代码在运行，就有可能用到它们。当变量离开上下文时，也会被加上离开上下文的标记。

3.1.2 引用计数

另一种没那么常用的垃圾回收策略是引用计数（reference counting）。其思路是对每个值都记录它被引用的次数。声明变量并给它赋一个引用值时，这个值的引用数为 1。如果同一个值又被赋给另一个变量，那么引用数加 1。类似地，如果保存对该值引用的变量被其他值给覆盖了，那么引用数减 1。当一个值的引用数为 0时，就说明没办法再访问到这个值了，因此可以安全地收回其内存了。垃圾回收程序下次运行的时候就会释放引用数为 0的值的内存。

但这种策略会遇到很严重的问题：循环引用。所谓循环引用，就是对象 A有一个指针指向对象 B，而对象B也引用了对象A。

4 优化内存占用

优化内存占用的最佳手段就是保证在执行代码时只保存必要的数据。如果数据不再必要，那么把它设置为 null，从而释放其引用。这也可以叫作解除引用。这个建议最适合全局变量和全局对象的属性。局部变量在超出作用域后会被自动解除引用。如下面的例子所示：

4.1 通过 const 和 let 声明提升性能

因为const和let 都以块（而非函数）为作用域，所以相比于使用var，使用这两个新关键字可能会更早地让垃圾回收程序介入，尽早回收应该回收的内存。

4 理解闭包

闭包是指一个函数能够记住并访问它的词法作用域，即使这个函数在其词法作用域之外执行。简单来说，闭包就是函数和与其相关的引用环境组合的一个整体。

4.1 工作原理

  1. 词法作用域：JavaScript中的函数在定义时，会捕获其所在的词法作用域。这意味着函数可以访问其定义时所在作用域中的变量和函数。
  2. 嵌套函数：当一个函数内部定义了另一个函数时，内部函数可以访问外部函数的变量和参数。
  3. 持久化作用域：即使外部函数已经执行完毕，内部函数仍然可以访问外部函数的作用域。这是因为内部函数保留了对外部函数作用域的引用。

闭包可以捕获并保留外部作用域中的对象，使其在外部作用域中仍然可访问。

对上面这句话的理解

  1. 闭包跟函数最大的区别在于，当捕捉闭包的时候，它的自由变量(eg:"name")会在捕捉时被确定，这样即使脱离了捕捉时的上下文(eg:foo)，它也能照常运行；
  2. 闭包在实现上是一个结构体，它存储了一个函数(eg:bar)和一个关联的环境(eg:"name")（相当于一个符号查找表）；
  3. 闭包是在支持 头等函数 的编程语言中，实现词法绑定的一种技术

    • 词法绑定：在词法解析时就确定了父级作用域
