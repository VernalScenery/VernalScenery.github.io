1 效果合成器（EffectComposer）

用于在 three.js 中实现后期处理效果。该类管理了产生最终视觉效果的后期处理过程链。 后期处理过程根据它们添加/插入的顺序来执行，最后一个过程会被自动渲染到屏幕上。

1.1 构造函数

  • renderer – 用于渲染场景的渲染器。EffectComposer  需要一个渲染器来执行后处理效果。渲染器负责将场景渲染到屏幕上，并且  EffectComposer  会使用这个渲染器来处理各种后处理效果。
  • renderTarget – （可选）一个预先配置的渲染目标，内部由 EffectComposer 使用。

1.2 属性

  • .passes : Array：一个用于表示后期处理过程链（包含顺序）的数组

1.3 方法

  • .addPass ( pass : Pass ) : undefined：添加一个通道
    pass – 将被添加到过程链的过程
    将传入的过程添加到过程链。
  • .insertPass ( pass : Pass, index : Integer ) : undefined：指定位置添加一个通道
    pass – 将被插入到过程链的过程。
    index – 定义过程链中过程应插入的位置。
  • .removePass ( pass : Pass ) : undefined：移除一个通道
    pass – The pass to remove from the pass chain.
    Removes the given pass from the pass chain.
  • .render ( deltaTime : Float ) : undefined：执行渲染
    deltaTime – The delta time value.
    执行所有启用的后期处理过程，来产生最终的帧，
  • .setPixelRatio ( pixelRatio : Float ) : undefined
    pixelRatio – 设备像素比
  • .setSize ( width : Integer, height : Integer ) : undefined
    width – EffectComposer 的宽度。
    height – EffectComposer 的高度。
  • .dispose () : undefined：释放由此实例分配的与 GPU 相关的资源

2 渲染器通道RenderPass

2.1 构造函数

通过EffectComposer(renderer)指定了需要后处理的渲染器WebGLRenderer，渲染器通道RenderPass的作用是指定后处理对应的相机camera和场景scene。

  • scene: THREE.Scene：当前的场景对象。RenderPass 需要知道当前场景中的所有对象，以便在渲染时将它们绘制到帧缓冲区中。
  • camera: THREE.Camera：当前使用的摄像机对象。RenderPass 需要知道当前摄像机的位置和视角，以便正确地从摄像机的角度渲染场景。

给EffectComposer添加一个渲染器通道RenderPass。

  • composer: EffectComposer 对象，用于管理多个后处理效果。
  • renderPass: RenderPass 对象，用于将场景渲染到帧缓冲区中。

2.2 后处理链的顺序

  1. RenderPass
    • 作用：将场景中的所有对象渲染到帧缓冲区中。
    • 位置：通常是后处理链中的第一步，因为它提供了后续后处理效果的基础图像。
  2. 其他后处理效果
    • 作用：对  RenderPass  渲染的结果进行进一步处理，如添加光晕、轮廓、模糊等效果。
    • 位置：在  RenderPass  之后，按顺序执行

2.4 OrbitControls  与渲染循环的关系

  1. 响应用户输入:

    • OrbitControls  会监听用户的鼠标和触摸事件，根据用户的操作更新摄像机的位置和方向。
    • 这些更新需要在每一帧中应用，以确保用户看到的是最新的视图。

  2. 持续更新:

    • 渲染循环确保每一帧都重新渲染场景，这样  OrbitControls  的更新才能立即反映在屏幕上。
    • 如果没有渲染循环，即使  OrbitControls  更新了摄像机的位置，也不会看到任何变化，因为场景不会重新渲染。

  • scene: THREE.Scene`` 当前的场景对象。OutlinePass`  需要知道当前场景中的所有对象，以便在渲染时应用轮廓效果。

4.1 构造函数

从左往右依次为：

  • resolution：THREE.Vector2：屏幕分辨率，用于确定后处理效果的输出尺寸。通常情况下，这个值应该是渲染器的尺寸。
