1.1 函数执行上下文

当一个函数被调用时，JavaScript 引擎会为该函数创建一个函数执行上下文Functional Execution Context（FEC），并且压入到ECS中。这个执行上下文包括以下几个部分：

  • 变量对象（Variable Object，VO）：在函数执行上下文中，变量对象被称为活动对象（Activation Object，AO）。
  • 作用域链（Scope Chain）：用于解析标识符（变量、函数等）。
  • this 值：当前执行上下文的 this 值。this 值的确定取决于函数的调用方式。

总结：

  • this 是一个关键字，用于引用当前执行上下文中的对象。
  • this 的值取决于函数的调用方式，而不是函数的定义位置。

this的绑定是函数执行时确定的，而具体的绑定过程有许多的规则

2.3.1 引入

使用 call、apply 或 bind 方法可以显式地指定 this 的值。

  • call：立即调用函数，并传入指定的 this 值和参数列表。
  • apply：立即调用函数，并传入指定的 this 值和参数数组。
  • bind：返回一个新的函数，该函数的 this 值被永久绑定到指定的对象。

5.3 箭头函数

5.3.1 箭头函数的 this

普通函数的 this 绑定

普通函数在调用时会创建自己的 this 上下文，this 的值取决于函数的调用方式。我们已经讨论过几种主要的 this 绑定规则：

  1. 默认绑定（独立函数调用）：this 绑定到全局对象（非严格模式）或 undefined（严格模式）。
  2. 隐式绑定（方法调用）：this 绑定到调用该方法的对象。
  3. 显式绑定（使用 call、apply、bind）：this 绑定到显式指定的对象。
  4. 构造函数调用：this 绑定到新创建的对象。
箭头函数的 this 绑定

箭头函数不会创建自己的 this 上下文，而是继承外部作用域的 this。这意味着箭头函数的 this 值是在定义时确定的，而不是在调用时确定的。

  • 定义时确定：箭头函数的 this 值在其定义时就已经确定，而不是在调用时确定。
    • 如果外部也是一个函数，且是一个普通函数，那这个外部作用域的 this，即这个普通函数的 this 由4种绑定规则决定
  • 继承外部作用域：箭头函数的 this 继承自其外部作用域的 this。

  1. (b = person.sayName)();
    • 赋值表达式（独立函数调用）：b = person.sayName 将 person.sayName 赋值给 b，然后立即调用 b。由于 b 是一个独立函数调用，this 绑定到全局对象（在浏览器中是 window，在 Node.js 中是 global）。在严格模式下，this 会被绑定到 undefined。

6.2 面试题二

  1. foo1 方法：
    • person1.foo1();：隐式绑定，this 指向 person1，输出 person1。
    • person1.foo1.call(person2);：显示绑定，this 指向 person2，输出 person2。
  2. foo2 方法：
    • person1.foo2();：箭头函数，this 继承自外部作用域，外部作用域是全局对象 window，输出 window。
    • person1.foo2.call(person2);：箭头函数的 this 绑定在定义时已经确定，不会因为 call 方法而改变，仍然输出 window。
  3. foo3 方法：
    • person1.foo3()();：返回一个普通函数，this 在调用时绑定，外部作用域是全局对象 window，输出 window。
    • person1.foo3.call(person2)();：返回一个普通函数，this 在调用时绑定，外部作用域是全局对象 window，输出 window。
    • person1.foo3().call(person2);：返回一个普通函数，**this 在调用时绑定**，call(person2) 显式绑定 this，输出 person2。
  4. foo4 方法：
    • person1.foo4()();：返回一个箭头函数，箭头函数的 this 继承自外部作用域，外部作用域是 person1(因为普通函数的this是调用时绑定的)，，输出 person1。
    • person1.foo4.call(person2)();：返回一个箭头函数，箭头函数的 this 继承自外部作用域，外部作用域是 person2，输出 person2。
    • person1.foo4().call(person2);：返回一个箭头函数，箭头函数的 this 继承自外部作用域，外部作用域是 person1，输出 person1。
      • 首先是person1.foo4（），此时 foo4的 this就隐式绑定为 person1了，
      • 而箭头函数的 this 是定义时就确定的，注意只有执行 foo4()，才会创建 AO，箭头函数才会被定义。
      • 故无论 call 谁，this 都是继承的外部作用域 foo4，即 person1.foo4（）这时确定的 person1

生成一个对象本身不会创建一个新的作用域，对象的作用域取决于其创建和使用的上下文。

  • 创建上下文：对象的创建上下文决定了对象的初始作用域。对象可以在全局作用域、函数作用域或块作用域中创建。
    • 全局作用域：在全局作用域中创建的对象及其属性和方法在全局范围内可访问。
    • 函数作用域：在函数内部创建的对象及其属性和方法只在该函数的作用域内可访问。
    • 块作用域：在块内部创建的对象及其属性和方法只在该块的作用域内可访问。
  • 使用上下文：对象的使用上下文决定了对象在哪些地方可以被访问和操作。通过返回值、闭包等方式，对象可以在其创建作用域之外的地方使用。

6.4 面试题四

person1.obj.foo1()()

  • foo1 是一个普通函数，返回另一个普通函数。
  • 第一次调用 person1.obj.foo1() 返回一个普通函数。
  • 第二次调用返回的普通函数时，this 绑定到全局对象（在浏览器中是 window）。
  • 因此，输出 window。

person1.obj.foo1.call(person2)()

  • foo1 是一个普通函数，返回另一个普通函数。
  • person1.obj.foo1.call(person2) 将 foo1 的 this 绑定到 person2，但返回的仍然是一个普通函数。
  • 第二次调用返回的普通函数时，this 绑定到全局对象（在浏览器中是 window）。
  • 因此，输出 window。

person1.obj.foo1().call(person2)

  • foo1 是一个普通函数，返回另一个普通函数。
  • 第一次调用 person1.obj.foo1() 返回一个普通函数。
  • 第二次调用返回的普通函数时，使用 call(person2) 将 this 绑定到 person2。
  • 因此，输出 person2。

person1.obj.foo2()()

  • foo2 是一个普通函数，返回一个箭头函数。
  • 箭头函数没有自己的 this 绑定，而是继承外层作用域的 this。
  • 外层作用域的 this 是 person1.obj，其 name 属性为 'obj'。
  • 因此，输出 obj。

person1.obj.foo2.call(person2)()

  • foo2 是一个普通函数，返回一个箭头函数。
  • person1.obj.foo2.call(person2) 将 foo2 的 this 绑定到 person2。
  • 箭头函数继承 person2 的 this 绑定，其 name 属性为 'person2'。
  • 因此，输出 person2。

person1.obj.foo2().call(person2)

  • foo2 是一个普通函数，返回一个箭头函数。
  • 第一次调用 person1.obj.foo2() 返回一个箭头函数。
  • 箭头函数继承外层作用域的 this，即 person1.obj，其 name 属性为 'obj'。
  • 即使使用 call(person2) 也不会改变箭头函数的 this 绑定。
  • 因此，输出 obj。

arguments

在JavaScript中，arguments 是一个特殊的类数组对象，它包含了函数调用时传递的所有参数。arguments 对象在每个函数内部都是可用的，即使函数没有显式地定义参数列表。通过 arguments 对象，你可以在函数内部访问和操作传递给函数的所有参数。

1 普通函数

1.1 主要特点

  1. 类数组对象：
    • arguments 对象不是真正的数组，但它具有类似数组的属性和方法。
    • 它有一个 length 属性，表示传递给函数的参数数量。
    • 你可以通过索引index访问参数，例如 arguments[0] 表示第一个参数，arguments[1] 表示第二个参数，依此类推。
    • 但是它却没有数组的一些方法，比如forEach、map等；
  2. 动态参数：
    • arguments 对象允许你在不知道具体参数数量的情况下处理参数。
    • 这在编写通用函数时非常有用，特别是那些需要处理可变数量参数的函数。

1.2 arguments.callee

arguments.callee 是一个特殊的属性，它指向当前正在执行的函数。这个属性在某些情况下非常有用，尤其是在匿名函数中，因为匿名函数没有名称，无法通过函数名来引用自己。arguments.callee 提供了一种方式来引用当前函数。

用途

  1. 递归调用：在匿名函数中进行递归调用。
  2. 自调用：在某些设计模式中，如立即执行函数表达式（IIFE）中，可能需要引用当前函数。
