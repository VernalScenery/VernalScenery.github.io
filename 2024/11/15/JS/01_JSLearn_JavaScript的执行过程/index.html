一些概念

1 执行上下文(EC)

执行上下文（Execution Context）是 JavaScript 引擎在执行代码时创建的一个环境，用于管理和跟踪代码的执行状态。简单来说，执行上下文就像是一个“工作空间”，在这个空间里，JavaScript 引擎可以管理变量、函数、作用域等信息。

想象你在做一道数学题，你需要一张纸来记录你的计算过程、中间结果和最终答案。这张纸就是你的“工作空间”。同样，JavaScript 引擎在执行代码时也需要一个“工作空间”来管理代码的执行过程，这个“工作空间”就是执行上下文。

1.1 执行上下文的组成部分

  1. 变量对象（Variable Object，VO）：
    • 这是执行上下文的核心部分，用于存储变量、函数声明和参数。
    • 在全局执行上下文中，变量对象就是全局对象（GO）（window 或 global）。
    • 在函数执行上下文中，变量对象被称为活动对象（AO），包含函数参数、局部变量、函数声明和 arguments 对象。
  2. 作用域链（Scope Chain）：
    • 用于解析变量和函数的引用。
    • 作用域链是一个链表，包含了当前执行上下文及其父执行上下文的变量对象。
    • 通过作用域链，JavaScript 引擎可以查找变量和函数的定义。
  3. this 值：
    • 当前执行上下文中的 this 值。
    • this 值根据函数的调用方式确定，例如在全局执行上下文中，this 指向全局对象（window 或 global）；在方法调用中，this 指向调用该方法的对象。

1.2 执行上下文的生命周期

  1. 创建阶段：
    • 创建变量对象（VO）。
    • 初始化作用域链。
    • 确定 this 值。
  2. 编译阶段：
    • 变量声明：初始化为 undefined。
    • 函数声明：初始化为函数对象，并且为函数对象分配内存空间，并指向这个内存空间中的函数对象
  3. 执行阶段：
    • 逐行执行代码，更新变量对象中的值。

2 执行上下文栈(ECS)

执行上下文栈（Execution Context Stack）是 JavaScript 引擎用来管理代码执行过程中不同执行上下文的一种数据结构。它是一个后进先出（LIFO，Last In First Out）的栈结构，用于跟踪当前正在执行的代码的位置和状态。每当进入一个新的执行上下文（如调用一个函数），该上下文会被压入栈顶；当执行上下文完成时，它会被从栈顶弹出。

2.1 执行上下文栈的基本概念

  1. 全局执行上下文：程序启动时创建，包含全局变量和函数。
  2. 函数执行上下文：每次调用函数时创建，包含函数的局部变量、参数、函数声明等。
  3. 执行上下文栈：用于管理这些执行上下文的栈结构。

2.2 执行上下文栈的工作原理

  1. 程序启动：创建全局执行上下文并将其压入执行上下文栈。
  2. 函数调用：每次调用函数时，创建一个新的函数执行上下文并将其压入执行上下文栈。
  3. 函数返回：当函数执行完毕，其执行上下文从栈顶弹出，控制权返回到前一个执行上下文。

JS代码执行逻辑

1 全局代码执行逻辑

1.1 预解析（Hoisting）

  • 在代码执行之前，JavaScript引擎会进行预解析，将所有的变量声明和函数声明提升到当前作用域的顶部。
  • 变量声明会被提升，但初始化不会提升；函数声明会被完全提升。

1.2 创建全局执行上下文

  • 全局代码执行时，JavaScript引擎会创建一个全局执行上下文（Global Execution Context）。
  • 全局执行上下文是所有其他执行上下文的基础，它在整个程序运行期间一直存在。

1.3 变量对象初始化

  • 全局执行上下文的变量对象（Variable Object）包含所有全局变量和函数声明。
  • 变量声明会被初始化为undefined，直到它们被赋值。
  • 函数声明会被完全初始化，可以直接调用。

1.4 作用域链建立

  • 全局执行上下文的作用域链只有一个条目，即全局变量对象。
  • 全局变量可以在整个程序中访问。

1.5 this绑定

  • 在全局执行上下文中，this关键字绑定到全局对象。
    • 浏览器环境中，全局对象是window。
    • Node.js环境中，全局对象是global。

1.6 代码执行

  • 逐行执行全局代码。
  • 访问变量时，JavaScript引擎会从全局变量对象中查找。

1.7 执行上下文销毁

  • 全局执行上下文不会被销毁，因为它在整个程序运行期间一直存在。
  • 但是，当程序结束时，全局执行上下文及其相关的变量对象会被销毁。

2 全局执行上下文(GEC)

2.1 创建全局执行上下文

当 JavaScript 引擎开始执行全局代码时，首先会创建一个全局执行上下文Global Execution Context（GEC）。并且压入到ECS中。这个全局执行上下文包括以下几个部分：

  1. 变量对象（Variable Object，VO）：初始化全局变量和函数。在全局执行上下文中，VO通常被称为全局对象（Global Object，GO）
  2. 作用域链（Scope Chain）：全局执行上下文的作用域链只有一个条目，即全局对象本身。
  3. this 值：在全局执行上下文中，this 值指向全局对象（window 或 global）

2.2 编译阶段（Creation Phase）

在编译阶段，JavaScript 引擎会初始化全局执行上下文，创建变量对象（VO）并设置初始值。这个阶段的主要任务包括：

  • 初始化全局变量：声明的全局变量会被初始化为 undefined。
  • 初始化全局函数：声明的全局函数会被初始化为对应的函数对象，并且为函数对象分配内存空间，并指向这个内存空间中的函数对象
  • 初始化全局对象：全局对象（window 或 global）会被创建，并包含内置的全局属性和方法

2.3 执行阶段

  • 执行代码：逐行执行全局代码，更新变量对象中的值。

注意：

  • GO包含Date、Array、String、Number、setTimeout、setInterval等等，其中还有一个window属性指向自己；
  • 编译阶段只会初始化全局变量和全局函数，函数内的函数不会被编译，只会给全局函数对象分配内存空间，并指向这个内存空间中的函数对象

3 函数代码执行逻辑

3.1 函数声明与提升

  • 在代码执行之前，JavaScript引擎会进行预解析（Hoisting），将函数声明提升到当前作用域的顶部。
  • 这意味着即使函数定义在调用之后，也不会导致引用错误。

3.2 创建执行上下文

  • 每次调用函数时，JavaScript引擎都会为该函数创建一个新的执行上下文（Execution Context）。
  • 执行上下文包括变量对象（Variable Object）、作用域链（Scope Chain）和this绑定。

3.3 变量对象初始化

  • 变量对象（Variable Object）包含函数的所有局部变量、函数参数和内部变量。
  • 函数参数和变量声明会被初始化为undefined，直到它们被赋值。

3.4 作用域链建立

  • 作用域链（Scope Chain）决定了变量的查找路径。
  • 作用域链从当前执行上下文的变量对象开始，向上查找父级执行上下文的变量对象，直到全局执行上下文。

3.5 this绑定

  • this关键字的绑定取决于函数的调用方式。
  • 常见的绑定方式包括默认绑定、隐式绑定、显式绑定和new绑定。

3.6 代码执行

  • 在执行上下文中，逐行执行函数体内的代码。
  • 访问变量时，JavaScript引擎会沿着作用域链查找变量。

3.7 返回值

  • 函数执行完毕后，会返回一个值（如果有的话）。
  • 如果没有明确的返回语句，函数默认返回undefined。

3.8 执行上下文销毁

  • 函数执行完毕后，其执行上下文会被销毁，局部变量和参数会被从内存中移除。
  • 如果存在闭包，某些变量可能会继续保留在内存中。

4 函数执行上下文(FEC)

4.1 创建执行上下文（Execution Context）

当一个函数被调用时，JavaScript 引擎会为该函数创建一个函数执行上下文Functional Execution Context（FEC），并且压入到ECS中。这个执行上下文包括以下几个部分：

  • 变量对象（Variable Object，VO）：在函数执行上下文中，变量对象被称为活动对象（Activation Object，AO）。
  • 作用域链（Scope Chain）：用于解析标识符（变量、函数等）。
  • this 值：当前执行上下文的 this 值。this 值的确定取决于函数的调用方式。

4.2 编译阶段（Creation Phase）

在编译阶段，JavaScript 引擎会初始化执行上下文，创建变量对象（AO）并设置初始值。这个阶段的主要任务包括：

  • 创建变量对象（AO）：
    • 初始化函数参数。
    • 初始化函数声明，并且为函数对象分配内存空间，并指向这个内存空间中的函数对象
    • 初始化变量声明（但不赋值）。
    • 创建 arguments 对象（对于普通函数）。

4.3 执行阶段（Execution Phase）

在执行阶段，JavaScript 引擎会逐行执行函数内的代码。这个阶段的主要任务包括：

  • 执行代码：执行函数体内的每一行代码，包括赋值、调用其他函数等。
  • 更新变量对象（AO）：根据代码的执行结果，更新变量对象中的值。

5 内存分配

5.1 栈和堆

在JavaScript中，内存分配主要涉及到两个方面：栈（Stack）和堆（Heap）。

  • 栈（Stack）:
    • 栈用于存储原始值（如布尔值、数字、字符串、null、undefined）。
    • 函数调用时，参数和局部变量会被压入栈中，当函数执行完毕后，这些数据会从栈中弹出。
    • 栈的操作遵循“后进先出”（LIFO, Last In First Out）的原则。
  • 堆（Heap）:
    • 堆用于存储对象，包括数组、对象字面量等复杂数据结构。
    • 当创建一个对象时，会在堆中分配内存，并返回一个指向该对象的引用，这个引用通常存储在栈中。
    • 堆的内存管理相对复杂，涉及垃圾回收机制来释放不再使用的对象占用的内存。

5.2 AO和函数声明的内存分配

创建AO(或GO)和初始化函数声明在JavaScript中都会在内存中开辟存储空间

  • 每次函数调用时，都会创建一个新的 AO。
  • 函数声明会在内存中开辟存储空间，函数调用则是根据地址去调用，但不会开辟新存储空间

概念对比

1 全局执行上下文、函数执行上下文的区别

  • 变量对象（VO）：
    • 全局执行上下文：变量对象就是全局对象（window 或 global）。
    • 函数执行上下文：变量对象被称为活动对象（AO），包含函数参数、局部变量、函数声明和 arguments 对象。
  • 作用域链：
    • 全局执行上下文：作用域链只有一个条目，即全局对象。
    • 函数执行上下文：作用域链包含当前函数的作用域和其父作用域。
  • this 值：
    • 全局执行上下文：this 值指向全局对象（window 或 global）。
    • 函数执行上下文：this 值根据函数的调用方式确定。

2 作用域、作用域链

2.1 作用域（Scope）

作用域是指变量或函数在程序中可被访问的区域。JavaScript 中主要有两种作用域：

  1. 全局作用域（Global Scope）：
    • 在全局作用域中定义的变量和函数在整个程序中都可以被访问。
    • 全局作用域通常只有一个，除非在浏览器中每个窗口或标签页有一个独立的全局作用域。
  2. 局部作用域（Local Scope）：
    • 在函数内部定义的变量和函数只能在该函数内部被访问。
    • 每次调用函数时都会创建一个新的局部作用域。
  3. 块级作用域（Block Scope）：
    • 在 {} 块中定义的变量和常量只能在该块内部被访问。块级作用域主要通过 let 和 const 关键字实现。

2.2 作用域链（Scope Chain）

作用域链是 JavaScript 引擎在查找变量时使用的一系列作用域。它是一个包含多个作用域的链表，用于解析变量和函数引用。作用域链的形成和使用如下：

  1. 形成：
    • 作用域链在函数创建时确定，包含函数创建时所在的所有外层作用域的变量对象。
    • 每个函数都有一个内部属性 [[Scope]]，这个属性保存了函数创建时的作用域链。
  2. 使用：
    • 当函数被调用时，JavaScript 引擎会创建一个新的执行上下文，并使用函数的 [[Scope]] 属性来构建该执行上下文的作用域链。
    • 作用域链用于在函数内部查找变量和函数引用，查找顺序是从内到外。

块级作用域的作用域链

块级作用域通过 let 和 const 关键字定义的变量和常量，它们的作用域仅限于所在的代码块。虽然块级作用域本身不是一个独立的作用域链节点，但它会影响作用域链的查找过程。

3 执行上下文、变量对象（VO）、作用域、this

  1. 执行上下文（Execution Context）：
    • 定义：执行上下文是JavaScript引擎为了管理和执行一段代码而创建的一个抽象概念。
    • 组成：
      • 变量环境（Variable Environment）：存储变量和函数声明。
      • 词法环境（Lexical Environment）：存储变量和函数声明的引用。
      • this绑定：确定当前执行上下文中this的值。
    • 生命周期：
      • 创建阶段：在函数调用之前，JavaScript引擎会创建执行上下文，并初始化变量环境和词法环境。
      • 执行阶段：在函数调用过程中，执行上下文中的代码被执行。
      • 销毁阶段：函数执行完毕后，执行上下文被销毁，释放相关资源。
  2. 变量对象（VO）：执行上下文的一部分，用于存储函数作用域内的变量和函数声明。
  3. 作用域（Scope）：
    • 定义：作用域决定了变量、函数或类在程序中的可访问范围。
      • JavaScript中有两种主要的作用域：全局作用域和局部作用域（也称为函数作用域或块作用域）。
      • 全局作用域中的变量在整个程序中都可以访问。
      • 局部作用域中的变量只能在其定义的函数或块内访问
    • 类型：
      • 全局作用域：在整个程序中都可访问。
      • 局部作用域：仅在特定的函数或块内可访问。
    • 与执行上下文的关系：
      • 作用域是由词法环境决定的。词法环境是执行上下文的一部分，因此可以说作用域与执行上下文密切相关。
      • 每个执行上下文都有一个词法环境，词法环境决定了该上下文中的作用域链（Scope Chain）。
  4. this关键字：
    • 定义：this是一个特殊的关键字，它在运行时被赋予一个值，这个值取决于函数的调用方式。
    • 与执行上下文的关系：
      • this 是一个关键字，用于引用当前执行上下文中的对象。
      • this的值取决于函数的调用方式，而不是函数的定义方式。

总结

  • 执行上下文包含词法环境和变量环境，这两个环境共同决定了作用域。
  • 执行上下文还包含this绑定，决定了this**的值。
  • 作用域是由词法环境决定的，而词法环境是执行上下文的一部分。
  • this的值是在执行上下文创建时确定的，因此this与执行上下文密切相关。
