BOM

1 认识 BOM

BOM（Browser Object Model，浏览器对象模型）是用于与浏览器窗口进行交互的对象集合。它提供了许多对象来控制浏览器的行为和外观，而不仅仅是文档内容。

BOM主要包括的对象模型

  • window：包括全局属性、方法，控制浏览器窗口相关的属性、方法；
  • location：浏览器连接到的对象的位置（URL）；
  • history：操作浏览器的历史；
  • document：当前窗口操作文档的对象；
  • Screen：提供了关于用户屏幕的信息，包括屏幕的尺寸和颜色深度等。
  • Navigator：提供了关于浏览器的信息，包括浏览器的名称、版本、平台、语言设置等。

2 Window 对象

2.1 Window 作为 JS 全局对象

2.2 Window 作为浏览器窗口对象

详情见 Window - Web API | MDN

常用属性、方法、事件

EventTarget

Window继承自EventTarget，所以会继承其中的属性和方法：

  • addEventListener：注册某个事件类型以及事件处理函数；
  • removeEventListener：移除某个事件类型以及事件处理函数；
  • dispatchEvent：派发某个事件类型到EventTarget上；

3 Location 对象

Location 对象提供了关于当前文档 URL 的信息，并且允许你操作 URL。

常用属性：

  • href: 完整的 URL 字符串。
  • protocol: 协议部分（例如 http: 或 https:）。
  • host: 主机名和端口号（如果有）。
  • hostname: 主机名（不包括端口号）。
  • port: 端口号（如果没有指定则为空字符串）。
  • pathname: URL 的路径部分。
  • search: 查询字符串部分，以问号 ? 开头。
  • hash: 片段标识符部分，以井号 # 开头。
  • username: URL 中的用户名部分（许多浏览器已禁用）。
  • password: URL 中的密码部分（许多浏览器已禁用）。

常用方法：

  • assign(url): 加载新的文档，并在浏览器历史记录中留下一条记录。
  • replace(url): 用新的文档替换当前文档，不会在浏览器历史记录中留下一条记录。
  • reload(forceGet): 重新加载当前文档，可以指定是否从服务器强制加载。

4 History 对象

History 对象允许我们访问浏览器的会话历史记录，并提供了多种方法来操作这些记录。

常用属性：

  • length: 会话中的历史记录条数。
  • state: 当前状态对象。

常用方法：

  • back(): 返回上一页，等价于 history.go(-1)。
  • forward(): 前进下一页，等价于 history.go(1)。
  • go(n): 加载历史中的某一页。
  • pushState(state, title, url): 添加一个新的历史记录。
  • replaceState(state, title, url): 替换当前的历史记录。

DOM

浏览器用来展示网页，而网页中最重要的是其中的各种标签元素。JavaScript 通过 Document Object Model (DOM) 来操作这些元素。DOM 提供了一系列的模型和对象，使得我们可以方便地操作 Web 页面。

整体架构

1 EventTarget

EventTarget 是一个基础接口，表示可以接收事件的对象，并且可以为其注册事件监听器。几乎所有与 DOM 相关的节点都实现了 EventTarget 接口，包括 Element、Document、Window 等。

通过 EventTarget 接口，你可以方便地为各种 DOM 节点注册、移除和触发事件，实现丰富的交互效果。

  • addEventListener：注册某个事件类型以及事件处理函数；
  • removeEventListener：移除某个事件类型以及事件处理函数；
  • dispatchEvent：派发某个事件类型到EventTarget上；

2 Node节点

在 DOM（文档对象模型）中，Node 是所有节点类型的基类。每个节点（如元素、文本、注释等）都是 Node 的实例。Node 提供了许多属性和方法，用于操作和查询节点。

所有的DOM节点类型都继承自Node接口。Node - Web API | MDN

常用属性和方法：

  1. 获取节点属性：
    • nodeName 返回节点的名称，对于元素节点，返回标签名。
    • nodeType 返回节点的类型，常见的有 1（元素节点）、3（文本节点）等。
    • nodeValue 返回节点的值，对于文本节点，返回文本内容；对于其他类型的节点，通常为 null。
  2. 操作子节点：
    • childNodes 返回一个 NodeList，包含当前节点的所有子节点。
    • 通过 childNodes 可以访问子节点的属性和方法，例如 nodeValue。
  3. 添加、插入、删除和替换节点：
    • appendChild(node) 将一个节点添加到当前节点的最后一个子节点位置。
    • insertBefore(newNode, referenceNode) 在指定的参考节点之前插入一个新节点。
    • removeChild(node) 从当前节点中删除一个子节点。
    • replaceChild(newNode, oldNode) 用一个新节点替换当前节点的一个子节点。

3 Document

Document 对象是 DOM（文档对象模型）的根节点，代表整个 HTML 或 XML 文档。它提供了许多属性和方法，用于访问和操作文档中的元素和数据。

Document节点表示的整个载入的网页

常用属性和方法：

  1. 获取文档的基本信息：
    • document.body 获取 <body> 元素。
    • document.title 获取和设置文档的标题。
    • document.head 获取 <head> 元素。
    • document.children 获取文档的子节点集合。
  2. 获取和操作 URL：
    • window.location 和 document.location 都表示当前文档的 URL，它们引用同一个 Location 对象。
  3. 创建和获取元素：
    • document.createElement(tagName) 创建一个新的元素节点。
    • document.createTextNode(text) 创建一个新的文本节点。
    • document.getElementById(id) 根据 ID 获取元素。
    • document.getElementsByTagName(tagName) 根据标签名获取元素集合。
    • document.getElementsByName(name) 根据 name 属性获取元素集合。
    • document.querySelector(selector) 根据 CSS 选择器获取第一个匹配的元素。
    • document.querySelectorAll(selector) 根据 CSS 选择器获取所有匹配的元素。

4 Element

Element 对象是 Node 对象的一个子类，代表文档中的一个 HTML 或 XML 元素。它提供了许多属性和方法，用于访问和操作元素的各种特性。

我们平时创建的div、p、span等元素在DOM中表示为Element元素

常用属性和方法：

  1. 获取和设置属性：
    • getAttribute(name) 获取指定属性的值。
    • setAttribute(name, value) 设置指定属性的值。
    • removeAttribute(name) 移除指定属性。
    • hasAttribute(name) 检查元素是否具有指定属性。
  2. 操作类名：
    • className 获取和设置元素的 class 属性。
    • classList 提供了一个 DOMTokenList 对象，可以进行增删查改操作。
  3. 获取元素尺寸和位置：
    • clientWidth 和 clientHeight 分别表示元素的可见宽度和高度。
    • offsetLeft 和 offsetTop 分别表示元素相对于其偏移父元素的左边缘和上边缘的距离。

5 事件监听

5.1 认识事件监听

5.2 事件流

事件流的概念确实非常重要，它描述了当一个事件发生时，事件是如何在DOM树中传播的。具体来说，事件流分为两个主要阶段：事件捕获阶段和事件冒泡阶段。

5.2.1 为什么会产生事件流？

当你在浏览器上点击一个元素时，实际上你点击的是一个嵌套的元素结构。例如，一个<span>元素可能嵌套在一个<div>元素中，而<div>元素又嵌套在<body>元素中。因此，事件不仅仅发生在被点击的那个元素上，还会涉及到其父元素以及更高层次的祖先元素。这就是为什么我们需要事件流来定义事件是如何从顶层元素传递到目标元素，然后再返回到顶层元素的过程。

5.2.2 事件流的两个阶段

早期浏览器开发时，微软（IE）和网景（Netscape）公司在处理事件流的问题上采取了不同的策略，导致了事件冒泡和事件捕获这两种不同的事件流机制。

  1. 事件捕获阶段（Capture Phase）：

    • 在事件捕获阶段，事件从最外层的祖先元素开始，逐级向下传递到目标元素。

  2. 事件冒泡阶段（Bubble Phase）：

    • 在事件冒泡阶段，事件从目标元素开始，逐级向上传递到最外层的祖先元素。

默认情况下，addEventListener监听的是事件冒泡阶段。

在现代浏览器中，你可以通过在addEventListener方法的第三个参数中设置为true来监听事件捕获过程。

  • event：事件名称（如"click"）。
  • listener：事件处理函数。
  • useCapture：布尔值，true表示在捕获阶段监听事件，false或省略表示在冒泡阶段监听事件。

如果一个 DOM 元素同时有事件冒泡和事件捕获的监听，那么会优先监听到事件捕获的

5.3 事件对象event

当一个事件发生时，浏览器会创建一个Event对象，并将其传递给事件处理函数。这个Event对象包含了与事件相关的各种信息和方法，帮助你更好地处理和响应事件。

事件参考 | MDN

5.3.1 常见的属性

  1. type：事件的类型，例如 "click"。
  2. target：事件最初发生的元素，即触发事件的元素。
  3. currentTarget：当前处理事件的元素，即绑定事件处理函数的元素。
  4. offsetX 和 offsetY：点击位置相对于事件目标元素的偏移量。
  5. clientX 和 clientY：鼠标指针相对于视口的坐标。
  6. pageX 和 pageY：鼠标指针相对于整个页面的坐标。
  7. screenX 和 screenY：鼠标指针相对于屏幕的坐标。
  8. which 或 button：表示鼠标按钮或键盘键，例如 1 表示左键，2 表示滚轮，3 表示右键
  9. keyCode 或 key：表示键盘按键的键码或键值。
  10. altKey、ctrlKey、shiftKey、metaKey：表示是否按下了 Alt、Ctrl、Shift 或 Meta 键。

5.3.2 常见的方法

  1. **preventDefault()**：取消事件的默认行为。
  2. **stopPropagation()**：阻止事件的进一步传播。
  3. **stopImmediatePropagation()**：阻止事件的进一步传播，并且阻止同一个事件处理程序中的其他监听器被调用。
