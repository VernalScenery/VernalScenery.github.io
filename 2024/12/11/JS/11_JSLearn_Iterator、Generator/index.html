迭代器 (Iterator)

1 什么是迭代器

迭代器

  • 迭代器：是一种设计模式，允许用户遍历容器对象（如数组、链表等）中的元素，而无需了解其内部结构。
  • 在JavaScript中，迭代器也是一个具体的对象，这个对象需要符合迭代器协议（iterator protocol）

迭代器协议要求

  • next 方法：必须有一个 next 方法，该方法返回一个对象，返回的对象包含两个属性：
    • done：布尔值，表示迭代是否完成。
      • false：还可以继续迭代。
      • true：迭代已完成，此时 value 可以省略。
    • value：当前迭代返回的值，可以是任何 JavaScript 值。

2 什么是可迭代对象

可迭代对象（Iterable Object）

  • 定义：可迭代对象是指实现了 iterable protocol 协议的对象。
  • 协议要求：必须实现 @@iterator 方法，该方法返回一个迭代器对象。
  • 访问方法：在 JavaScript 中，@@iterator 方法可以通过 Symbol.iterator 访问。

优点

  • 简化代码：将创建迭代器的过程封装在对象内部，使得外部使用更加简洁。
  • 兼容性：可迭代对象可以用于 for...of 循环等内置迭代操作，提高代码的可读性和可维护性。

概念区分

  • 迭代器是一个对象，它定义了一个 next 方法，该方法返回一个包含 done 和 value 属性的对象。
    • 用于遍历数据结构中的元素，每次调用 next 方法返回当前元素，并指示是否已经遍历完所有元素。
  • 可迭代对象是一个实现了 iterable protocol 协议的对象，即该对象必须有一个 @@iterator 方法（在 JavaScript 中通过 Symbol.iterator 访问）。
    • 使得对象可以被 for...of 循环等内置迭代操作使用。
  • 二者关系：可迭代对象 通过实现 [Symbol.iterator] 方法返回一个 迭代器对象。

4 可迭代对象的应用

  • for...of 循环：用于遍历可迭代对象。
  • 展开语法（Spread Syntax）：用于将可迭代对象展开为单个元素。
  • 解构赋值（Destructuring Assignment）：用于从可迭代对象中提取多个值。
  • 创建对象：许多内置构造函数接受可迭代对象作为参数。
  • 方法调用：许多内置方法接受可迭代对象作为参数，用于处理异步操作或转换数据。

生成器 (Generator)

1 什么是生成器

生成器是ES6中新增的一种函数控制、使用的方案，它可以让我们更加灵活的控制函数什么时候继续执行、暂停执行等

平时我们会编写很多的函数，这些函数终止的条件通常是返回值或者发生了异常

如果想在函数执行到某一行时暂停，就需要用到生成器

2 生成器函数

1、定义方式：

  • 生成器函数使用 function* 关键字定义。

2、使用 yield 关键字

  • yield 关键字用于暂停函数的执行，并返回一个值。
  • 当生成器函数被调用时，它不会立即执行函数体中的代码，而是返回一个生成器对象。
  • 通过调用生成器对象的 next 方法，可以逐步执行函数体中的代码，每次遇到 yield 时暂停。

3、返回值

  • 生成器函数返回一个生成器对象，这个生成器对象是一个特殊的迭代器。
  • 生成器对象有 next 方法，用于控制生成器函数的执行。
  • next 方法返回一个对象，该对象包含 value 和 done 两个属性：
    • value：当前 yield 表达式的值。
    • done：布尔值，表示生成器是否已经执行完毕。

3 通过 yield 中断执行

yield 关键字可以让生成器停止和开始执行，也是生成器最有用的地方。

  • 生成器函数在遇到 yield关键字之前会正常执行。遇到这个关键字后，执行会停止，函数作用域的状态会被保留；
  • 停止执行的生成器函数只能通过在生成器对象上调用next()方法来恢复执行；

此时的yield关键字有点像函数的中间返回语句，它生成的值会出现在next()方法返回的对象里。

  • 通过 yield 关键字退出的生成器函数会处在 done: false 状态；
  • 通过 return 关键字退出的生成器函数会处于 done: true 状态；

生成器函数内部的执行流程会针对每个生成器对象区分作用域。在一个生成器对象上调用next()不会影响其他生成器：

yield 关键字只能在生成器函数内部使用，用在其他地方会抛出错误。类似函数的 return 关键字，yield 关键字必须直接位于生成器函数定义中，出现在嵌套的非生成器函数中会抛出语法错误

4 生成器传递参数 – next函数

生成器函数的一个重要特性是可以在调用 next 方法时传递参数。

  • 这些参数会作为上一个 yield 表达式的返回值。
  • 这种机制使得生成器函数可以在不同的执行阶段接收外部输入，从而实现更复杂的逻辑控制。

  1. next 方法传递参数：
    • 在调用 next 方法时传递的参数会作为上一个 yield 表达式的返回值。
    • 第一次调用 next 时传递的参数会被忽略，因为生成器函数刚开始执行时没有 yield 表达式。
  2. yield 表达式的返回值：
    • yield 表达式会暂停生成器函数的执行，并返回一个值。
    • 下一次调用 next 方法时传递的参数会作为 yield 表达式的返回值。
  3. done 属性：
    • done 属性表示生成器是否已经执行完毕。
    • 当 done 为 true 时，表示生成器已经执行完毕，不会再有新的值返回。

5 生成器提前结束 – return函数

生成器函数除了可以通过 next 方法传递参数外，还可以通过 return 方法传递参数并提前终止生成器函数的执行。

  • return 方法会立即终止生成器函数的执行，并返回一个指定的值。
  • 一旦生成器函数被 return 方法终止，后续调用 next 方法将不再继续执行生成器函数体内的代码

第二段代码的执行, 使用了return，相当于在第一段代码的后面加上return, 就会提前终端生成器函数代码继续执行

6 生成器抛出异常 – throw函数

生成器函数除了可以通过 next 和 return 方法传递参数外，还可以通过 throw 方法向生成器函数内部抛出异常。

  • throw 方法会在生成器函数的当前暂停点抛出一个异常，生成器函数内部可以使用 try...catch 语句来捕获并处理这个异常；
  • 如果生成器函数抛出异常，并通过try...catch语句成功捕获并处理这个异常，那么后续调用next方法将继续执行生成器函数体内的代码；
