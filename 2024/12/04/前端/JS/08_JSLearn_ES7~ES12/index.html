2.5 Object Descriptors

ES8（ECMAScript 2017）引入了一个新的静态方法 Object.getOwnPropertyDescriptors，它返回一个对象，该对象的所有自身属性描述符（包括不可枚举属性）都作为其自身的属性存在。

这个方法的主要用途是配合 Object.create、Object.defineProperties 等方法，以便更灵活地复制和修改对象的属性。

用途

  1. 深克隆对象
    • 可以使用 Object.getOwnPropertyDescriptors 和 Object.create 来实现深克隆对象，包括其 getter 和 setter。
  2. 修改对象属性
    • 可以先获取对象的属性描述符，然后修改某些属性描述符，再重新定义这些属性。
  3. 继承和扩展
    • 在继承和扩展对象时，可以使用 Object.getOwnPropertyDescriptors 来确保子类正确继承父类的属性，包括 getter 和 setter。

2.6 Async Function：async、await

ES8（ECMAScript 2017）引入了 async 和 await 关键字，用于简化异步编程，使其更易于阅读和编写。async 和 await 提供了一种基于 Promise 的语法糖，使得异步代码看起来更像同步代码。

3 ES9

3.1 Async iterators

ES2018（ECMAScript 2018）引入了异步迭代器（Async Iterators），用于处理异步数据流。异步迭代器允许你使用 for await...of 循环来遍历异步数据源，如异步生成器、异步读取文件等。

3.2 对象展开运算符（Object Spread Operator）

ES2018（ECMAScript 2018）引入了对象展开运算符（Object Spread Operator），用于将一个对象的所有可枚举属性复制到另一个对象中。对象展开运算符使用三个点 (...) 表示。

1、对象浅拷贝

  • 浅拷贝：只复制对象的第一层属性，对于嵌套的对象或数组，只会复制其引用，不会递归复制其内部结构。
  • 深拷贝：不仅复制对象的第一层属性，还会递归地复制所有嵌套的对象或数组，确保新对象与原对象完全独立。

对于基本数据类型，会被直接复制到新的对象 copy 中

  • 使用 { ...original } 创建 original 的浅拷贝。
  • copy 和 original 是不同的对象，因此 original === copy 返回 false。

对于嵌套对象，内部对象仍然是引用

  • copy 是 original 的浅拷贝。
  • nested 对象仍然是引用，因此 original.nested === copy.nested 返回 true。

3.3 Promise finally

finally 方法是 ES2018 引入的一个 Promise 方法，用于指定无论 Promise 被 resolve 还是 reject，都会执行的回调函数。这个方法通常用于清理资源或执行一些无论成功还是失败都需要执行的操作。

4 ES10

4.1 flat flatMap

ES2019 引入了 flat 和 flatMap 方法，用于处理多维数组。这两个方法提供了更简洁的方式来展平数组并进行映射操作。

4.1.1 flat 方法

flat 方法用于将多维数组展平成一维数组。你可以指定展平的深度，默认深度为 1。

  • depth：可选参数，表示展平的深度，默认值为 1。

4.1.2 flatMap 方法

flatMap 方法首先对数组中的每个元素执行一个映射函数，然后将结果展平成一维数组。相当于先使用 map 方法，再使用 flat 方法（深度为 1）。

  • callback：必需，生成新数组元素的函数，可以接受三个参数：
    • currentValue：当前正在处理的数组元素。
    • index：可选，当前正在处理的数组元素的索引。
    • array：可选，调用 flatMap 的数组。
  • thisArg：可选，执行 callback 函数时使用的 this 值。

  • Object.fromEntries 则将键值对数组转换回对象

5 ES11

5.1 BigInt

BigInt 是 ES2020 引入的一种新的数据类型，用于表示任意大小的整数。JavaScript 中的 Number 类型只能安全地表示 -2^53 到 2^53 之间的整数（即 -9007199254740991 到 9007199254740991）。超出这个范围的整数可能会导致精度丢失。BigInt 解决了这个问题，可以表示任意大小的整数

ES11 之前 max_safe_integer

  • Number.MAX_SAFE_INTEGER 表示 Number 类型的最大安全整数
  • 当超过这个范围时，加法运算会导致精度丢失

ES11 之后: BigInt

  • 使用 n 后缀创建 BigInt 类型的字面量
  • BigInt 类型的加法运算不会导致精度丢失

不支持隐式转换，需要显示转换

  • 使用 BigInt(num) 将 Number 类型转换为 BigInt 类型，以便进行混合运算。
  • 将 BigInt 转换为 Number 时，如果 BigInt 的值超出了 Number 的范围，结果会是 Infinity。

5.2 空值合并运算符 (Nullish Coalescing Operator)

Nullish Coalescing Operator（空值合并运算符）是 ES2020 引入的一个新运算符，用于处理 null 和 undefined 值。它提供了一种更精确的方式来设置默认值，与传统的逻辑或运算符 || 相比，?? 运算符不会将 0、false、NaN 等“假值”视为 null 或 undefined。

  • value：要检查的值。
  • defaultValue：当 value 为 null 或 undefined 时返回的默认值。

5.3 可选链 (Optional Chaining)

Optional Chaining 是 ES2020 引入的一个新特性，用于安全地访问嵌套对象的属性。它允许你在访问对象的属性时，如果某个中间属性不存在或为 null 或 undefined，则整个表达式会短路并返回 undefined，而不会抛出错误。

减少 if 判断

5.4 全局对象 (Global This)

globalThis 是 ES2020 引入的一个新特性，用于在任何 JavaScript 环境中获取全局对象。无论是在浏览器、Node.js 还是其他 JavaScript 运行环境中，globalThis 都能提供一种统一的方式来访问全局对象，避免了不同环境下的差异。

  • globalThis 是一个标准的、跨环境的方式，可以在任何 JavaScript 环境中获取全局对象。
  • 无论是在浏览器、Node.js 还是其他环境中，globalThis 都会返回当前环境的全局对象。

获取某一个环境下的全局对象 (Global Object)

  • 跨环境兼容性：在编写跨平台的 JavaScript 代码时，使用 globalThis 可以避免因不同环境下的全局对象名称不同而导致的问题。

5.5 for..in 标准化

  1. 可枚举属性：
    • for...in 只会遍历对象的可枚举属性，包括对象自身的可枚举属性和从原型链继承的可枚举属性。
    • 可以使用 Object.prototype.propertyIsEnumerable() 方法来检查属性是否可枚举。
  2. 顺序：
    • for...in 不保证属性的遍历顺序。虽然在现代 JavaScript 引擎中，通常会按照属性的插入顺序遍历，但这并不是标准要求的一部分。
    • 如果需要有序遍历对象的属性，建议使用 Object.keys() 或 Object.entries() 方法。
  3. 原型链属性：
    • for...in 会遍历对象自身的属性以及从原型链继承的可枚举属性。
    • 为了只遍历对象自身的属性，可以使用 hasOwnProperty() 方法进行过滤。
  4. 性能：
    • for...in 循环在遍历对象属性时可能会比 for...of 或 forEach 等方法稍慢，因为它需要处理原型链上的属性。
    • 对于数组，推荐使用 for...of 或 forEach 方法。

5.6 动态导入 (Dynamic Import)

动态导入是 ES2020 引入的一个新特性，允许你在运行时按需加载模块。这有助于提高应用的性能，特别是在大型应用中，可以减少初始加载时间。动态导入使用 import() 函数，返回一个 Promise，该 Promise 解析为模块的内容。

动态导入：使用 import() 函数在运行时按需加载模块。

  • 优点：
    • 提高应用性能，减少初始加载时间。
    • 支持代码分割和懒加载。
  • 应用场景：
    • 按需加载模块。
    • 代码分割。
    • 懒加载功能或组件。
  • modulePath：要导入的模块的路径。
  • 返回一个 Promise，解析为模块对象。

5.7 Promise.allSettled

Promise.allSettled 是 ES2020 引入的一个新的 Promise 方法，用于处理多个 Promise 并等待它们全部完成（无论是成功还是失败）。与 Promise.all 不同，Promise.allSettled 不会在任何一个 Promise 失败时立即拒绝，而是会等待所有 Promise 完成，并返回一个包含每个 Promise 结果的数组。

**Promise.allSettled**：等待所有 Promise 完成，并返回一个包含每个 Promise 结果的数组。

  • 优点：
    • 不会在任何一个 Promise 失败时立即拒绝。
    • 可以了解每个 Promise 的最终状态和结果。
  • 应用场景：
    • 并行操作。
    • 错误处理。
    • 日志记录。
  • promises：一个包含多个 Promise 的可迭代对象（如数组）。
  • 返回一个 Promise，解析为一个结果数组。

5.8 import meta

import.meta 是 ES2020 引入的一个特性，用于提供关于当前模块的元数据。它是一个对象，包含与模块相关的各种信息。最常用的属性是 url，它表示模块的 URL。

**import.meta**：提供关于当前模块的元数据。

  • 常用属性：
    • url：模块的 URL 字符串。
  • 应用场景：
    • 获取模块的 URL，用于动态加载资源或进行相对路径计算。
    • 在不同环境中获取模块的元数据，便于调试和配置。

6 ES12

6.1 FinalizationRegistry

FinalizationRegistry 是 ES2021 引入的一个新特性，用于在对象被垃圾回收时执行清理操作。它允许开发者注册回调函数，当注册的对象被垃圾回收时，这些回调函数会被调用。这在处理原生资源或需要显式清理的操作时非常有用。

  • callback：当目标对象被垃圾回收时调用的函数。
  • target：要注册的目标对象。
  • heldValue：传递给回调函数的值。
  • unregisterToken：可选参数，用于取消注册。

注意事项：

  1. 垃圾回收时机：垃圾回收的具体时机无法精确控制，因此回调函数的调用时间也是不确定的。
  2. 性能影响：频繁使用 FinalizationRegistry 可能会对性能产生一定影响，应谨慎使用。
  3. 内存泄漏：确保不要在 heldValue 中持有对目标对象的引用，否则可能导致内存泄漏。

unregisterToken 参数用于取消注册，防止对象过早被垃圾回收。

由于 obj 被取消注册，即使 obj 被设置为 null，回调函数也不会被调用。

6.2 WeakRef

WeakRef 是 ES2021 引入的一个新特性，用于创建对对象的弱引用。弱引用不会阻止对象被垃圾回收，因此在对象不再被强引用时，它可以被正常回收。WeakRef 主要用于需要访问对象但又不希望阻止其被垃圾回收的场景。

  • 弱引用：不会增加对象的引用计数，允许对象在不再被强引用时被垃圾回收。
  • deref 方法：用于尝试获取弱引用指向的对象。如果对象已被垃圾回收，则返回 undefined。

如果我们默认将一个对象赋值给另外一个引用，那么这个引用是一个强引用

由于 info 的强引用，故 obj 不会被垃圾回收机制销毁

如果我们希望是一个弱引用的话，可以使用 WeakRef

  • target：要创建弱引用的对象。
  • weakRef.deref()：尝试获取弱引用指向的对象，如果对象已被垃圾回收，则返回 undefined。

6.3 逻辑赋值运算符 (logical assignment operators)

ES2021 引入了三种新的逻辑赋值运算符：||=、&&= 和 ??=。这些运算符可以在条件满足时进行赋值操作，使代码更加简洁和易读。

6.3.1  ||= 逻辑或赋值运算符

||= 运算符用于在左侧变量为 false、null、undefined、0、NaN 或空字符串 "" 时，将其赋值为右侧的值。

6.3.2  &&= 逻辑与赋值运算符

&&= 运算符用于在左侧变量为 true 且非 false、null、undefined、0、NaN 或空字符串 "" 时，将其赋值为右侧的值。

6.3.3 3. ??= 逻辑空赋值运算符

??= 运算符用于在左侧变量为 null 或 undefined 时，将其赋值为右侧的值。与 ||= 不同，??= 不会将 false、0、NaN 或空字符串 "" 视为 false。

6.4 数字分隔符 (Numeric Separator)

ES2021 引入了数字分隔符（Numeric Separator），这是一种在数字中插入下划线 _ 的方式，以提高大数字的可读性。数字分隔符可以用于整数和浮点数中，但不能出现在数字的开头或结尾，也不能连续出现。

6.5 String.replaceAll

String.replaceAll 方法用于将字符串中所有匹配指定值或正则表达式的部分替换为另一个字符串。

String.replace 只会替换第一个匹配项，而 String.replaceAll 会替换所有匹配项
