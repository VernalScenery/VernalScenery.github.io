1 Proxy

Proxy 是 ES6 引入的一个新特性，它允许你创建一个代理对象，从而可以拦截并自定义对目标对象的操作。Proxy 可以用来实现数据的监听、验证、日志记录等功能。

1.2 Proxy 的捕获器

  • construct 捕获器: 用于拦截通过 new 关键字的实例化操作。

1.3 receiver 参数的作用

在 Proxy 的捕获器（如 get 和 set）中，receiver 参数是一个非常重要的参数。它的主要作用是确保在访问器属性（getter 和 setter）中，this 的指向是正确的。

  • receiver 是调用 get 或 set 捕获器时传入的第三个参数
  • 它通常是创建出来的代理对象（即 Proxy 实例）

receiver 的作用

  • 确保 this 的正确性：在访问器属性（getter 和 setter）中，this 应该指向调用该属性的对象。使用 receiver 可以确保这一点。
  • 避免 this 被错误地绑定到目标对象：如果不使用 receiver，this 可能会被错误地绑定到目标对象（target），而不是代理对象（Proxy 实例）。

第一次 get 方法被访问

  • 当你访问 objProxy.name 时，get 捕获器被触发。
  • key 是 name，receiver 是 objProxy。

第二次 get 方法被访问

  • 在 obj 的 name getter 中，return this._name; 会再次触发 get 捕获器。
  • 这次 key 是 _name，receiver 仍然是 objProxy。

如果不使用receiver，则this 在 name 的 getter 和 setter 中会指向 obj，而不是 objProxy。这可能会导致意外的行为，特别是在使用 Proxy 来增强或修改对象行为时。

2 Reflect

Reflect 是 ES6 引入的一个内置对象，它提供了许多与对象操作相关的方法，这些方法与 Proxy 的捕获器一一对应。Reflect 的主要目的是将一些原本在 Object 上的方法移到 Reflect 上，并且这些方法的行为与 Proxy 的捕获器一致，这使得 Reflect 成为一个更强大和一致的对象操作工具。

主要特点

  1. 一致性: Reflect 的方法与 Proxy 的捕获器方法一一对应，这使得它们的行为一致。
  2. 简洁性: Reflect 的方法通常比 Object 的相应方法更简洁和直观。
  3. 可读性: 使用 Reflect 可以提高代码的可读性和一致性。

2.3 Reflect 中 construct 的作用

Reflect.construct 方法的作用是使用指定的构造函数和参数列表来创建一个新对象。它类似于 new 操作符，但提供了更多的灵活性和控制。

具体来说，Reflect.construct 的作用包括：

  • 创建新对象：根据提供的构造函数和参数列表创建一个新的对象。
  • 指定原型：可以指定新对象的原型，这在某些情况下非常有用，例如当你希望新对象继承自某个特定的构造函数的原型时。

语法

  • target：目标构造函数，即你要调用的构造函数。
  • argsArray：传递给构造函数的参数数组。
  • newTarget（可选）：新对象的原型将被设置为 newTarget.prototype。如果不指定，newTarget 默认为 target。

案例

  • **Reflect.construct(Student, ["why", 18], Teacher)**：
    • Student 是目标构造函数。
    • ["why", 18] 是传递给 Student 构造函数的参数数组。
    • Teacher 是 newTarget，表示新对象的原型将被设置为 Teacher.prototype。

因此，teacher 对象虽然是通过 Student 构造函数创建的，但它的原型是 Teacher.prototype，而不是 Student.prototype。这就是 Reflect.construct 提供的灵活性。

3 响应式原理

响应式编程是一种编程范式，其核心思想是数据的变化会自动触发相关的计算或更新操作。在前端开发中，响应式通常指的是当数据发生变化时，视图或其他依赖于这些数据的部分能够自动更新，而不需要手动重新渲染或计算。

3.1 什么是响应式

  1. 数据绑定：数据源与视图之间的双向绑定，数据变化时视图自动更新，视图变化时数据源也自动更新。
  2. 依赖追踪：系统能够自动追踪数据的依赖关系，当数据变化时，能够找到所有依赖于该数据的计算或视图，并自动更新它们。
  3. 懒计算：只有在需要时才进行计算，提高性能。
