4 npm install 命令

4.1 基本使用

  • 全局安装（global install）： npm install webpack -g;
  • 项目（局部）安装（local install）： npm install webpack;

通常使用npm全局安装的包都是一些工具包：yarn、webpack等；并不是类似于 axios、express、koa等库文件；

4.2 原理

4.2.1 执行 npm install 它背后帮助我们完成了什么操作？

  • 解析 package.json 文件：读取 package.json 文件中的 dependencies 和 devDependencies 字段，确定需要安装的包及其版本范围。
  • 检查本地缓存：首先检查本地缓存中是否有需要的包，如果有则直接使用缓存中的包，减少网络请求。
  • 下载包：如果本地缓存中没有需要的包，则从 npm 仓库下载相应的包。
  • 解析包：下载完成后，解析包的内容，包括其依赖项。
  • 递归安装依赖：递归地安装每个包的依赖项，确保所有依赖都被正确安装。
  • 更新 node_modules 目录：将安装的包放入 node_modules 目录中。
  • 生成或更新 package-lock.json 文件：记录安装的具体版本信息，确保不同环境中安装的依赖版本一致。

4.2.2 package-lock.json 文件的作用是什么？

  • 锁定依赖版本：package-lock.json 文件记录了项目中所有依赖的具体版本号，确保在不同的开发和生产环境中安装的依赖版本一致，避免因版本不一致导致的问题。
  • 提高安装速度：由于 package-lock.json 文件中已经记录了所有依赖的具体版本和下载地址，npm 可以更快地解析和安装依赖，减少网络请求和解析时间。
  • 确保可重复构建：通过锁定依赖版本，确保每次构建的结果都是可重复的，这对于持续集成和部署非常重要。

package-lock.json

4.2.3 从 npm 5 开始，npm 支持缓存策略（来自 Yarn 的压力），缓存有什么作用？

  • 提高安装速度：缓存可以显著减少网络请求次数，加快包的下载和安装速度。
  • 节省带宽：已经下载过的包可以直接从缓存中获取，减少不必要的网络传输，节省带宽。
  • 离线安装：在没有网络连接的情况下，可以从缓存中获取已经下载过的包，实现离线安装。
  • 减少服务器负载：通过缓存，减少了对 npm 服务器的请求次数，减轻了服务器的负载。

npx

npx 是 npm 5.2.0 之后自带的一个命令，它可以自动查找并执行项目中的本地命令。即使在当前目录下没有全局安装某个命令，npx 也会尝试从项目中的 node_modules/.bin 目录中查找并执行。

  • 问题：在终端执行 webpack --version 时，实际使用的是全局安装的 webpack。
  • 原因：系统会按照当前目录 -> 全局路径的顺序查找并执行命令。
  • 解决方法：使用 npx 来确保执行的是项目中安装的 webpack。
