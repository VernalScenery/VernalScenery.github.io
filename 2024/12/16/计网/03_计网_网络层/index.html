IP 地址

1 概览

  • Mac 地址是用在一个网络内部的，比如 N1网络内部，但是跨网络就用不到了
  • 内部使用 Mac 地址，跨网络使用 IP 地址

这里以IPv4版本为例介绍

网络号相同的主机或路由器接口就处于同一网段或同一个局域网中

这里以前24位为网络号为例

2 IP 地址的分类

2.1 概览

  • A类地址：
    • 范围：0.0.0.0 到 127.255.255.255
    • 默认子网掩码：255.0.0.0
    • 适用于大型网络，前8位表示网络号，后24位表示主机号。
  • B类地址：
    • 范围：128.0.0.0 到 191.255.255.255
    • 默认子网掩码：255.255.0.0
    • 适用于中型网络，前16位表示网络号，后16位表示主机号。
  • C类地址：
    • 范围：192.0.0.0 到 223.255.255.255
    • 默认子网掩码：255.255.255.0
    • 适用于小型网络，前24位表示网络号，后8位表示主机号。
  • 回环地址：127.0.0.0 到 127.255.255.255，主要用于测试网络软件。

注意：

  • A、B和C类地址可以分配给网络中的主机或路由器的各接口。
  • 主机号“全0”的地址代表网络地址，不能分配给具体的主机或路由器接口。
  • 主机号“全1”的地址是广播地址，不能分配给具体的主机或路由器接口。

案例

2.2 可分配网络号数量

A类地址中，网络号为0和127的地址不能分配给实际的网络，原因如下：

  • 网络号为0：
    • 表示“本网络”：网络号全0（即 0.0.0.0 到 0.255.255.255）用于表示当前网络中的主机，而不是一个具体的网络。
    • 特殊用途：0.0.0.0 常用于表示默认路由，告诉路由器将数据包发送到默认网关进行进一步路由。
    • 因此，网络号为0的地址不能用于实际的网络分配，以避免混淆和潜在的路由问题。
  • 网络号为127：
    • 回环地址：网络号为127的地址范围（即 127.0.0.0 到 127.255.255.255）被预留用于回环（Loopback）测试。
    • 本地测试：127.0.0.1 是最常见的回环地址，用于测试本地主机上的网络软件和协议栈。
    • 因此，网络号为127的地址也不能用于实际的网络分配，以确保回环测试功能的正常运行。

可分配主机号数量都-2是因为：

  • 主机号全0和全0分别代表网络地址和广播地址

3 划分子网

3.1 子网划分的基本概念

  • 子网号：从主机号中借用一些比特位来表示子网号，这样可以将一个大的网络划分为多个子网
  • 子网掩码：用于标识网络号、子网号和主机号的长度。子网掩码是一个32位的二进制数，其中左起连续的1表示网络号和子网号，之后的0表示主机号。

3.2 默认子网掩码

  • A类地址的标准子网掩码：255.0.0.0（前8位为1）
  • B类地址的标准子网掩码：255.255.0.0（前16位为1）
  • C类地址的标准子网掩码：255.255.255.0（前24位为1

3.3 IP 地址结合子网掩码

假设有一个IP地址 192.168.1.10 和子网掩码 255.255.255.192，我们来计算该IP地址所在的网络地址。

1. 转换为二进制形式

  • IP地址：192.168.1.10

    • 192 = 11000000
    • 168 = 10101000
    • 1 = 00000001
    • 10 = 00001010
    • 组合起来：11000000.10101000.00000001.00001010

  • 子网掩码：255.255.255.192

    • 255 = 11111111
    • 255 = 11111111
    • 255 = 11111111
    • 192 = 11000000
    • 组合起来：11111111.11111111.11111111.11000000
2. 进行按位与操作
3. 将结果转换回十进制形式
  • 11000000 = 192
  • 10101000 = 168
  • 00000001 = 1
  • 00000000 = 0

组合起来，得到网络地址 192.168.1.0。

4 CIDR

CIDR（无类别域间路由，Classless Inter-Domain Routing）是一种用于IP地址分配和路由聚合的方法，它克服了传统IP地址分类（A、B、C类）的限制，提供了更灵活的IP地址管理。

CIDR使用斜杠 / 后跟一个数字来表示子网掩码的位数。

例如，192.168.1.0/24 表示IP地址 192.168.1.0 的子网掩码为 255.255.255.0。斜杠后面的数字表示网络号的位数，剩余的位表示主机号。

使用方法

  1. 确定子网大小：
    • 根据需要的子网数量和每个子网所需的主机数量，确定合适的子网掩码。
    • 例如，如果需要4个子网，每个子网至少需要8个主机地址，可以选择 /29 子网掩码（255.255.255.248），因为它提供8个可用主机地址（2^3 - 2 = 6个可用主机地址 + 2个保留地址）。
  2. 计算CIDR表示：
    • 将IP地址和子网掩码转换为CIDR表示法。
    • 例如，IP地址 192.168.1.0 和子网掩码 255.255.255.192 可以表示为 192.168.1.0/26。
  3. 计算子网范围：
    • 通过CIDR表示法，可以计算每个子网的网络地址、广播地址和可用主机地址范围。
    • 例如，192.168.1.0/26 表示：
      • 网络地址：192.168.1.0
      • 广播地址：192.168.1.63
      • 可用主机地址范围：192.168.1.1 到 192.168.1.62

5 同一网络的判断

主机号全为0代表网络地址

  1. 主机1：
    • IP地址：200.10.0.10
    • 子网掩码：255.255.255.0（即 /24）
    • 网络地址：200.10.0.0
    • 广播地址：200.10.0.255
    • 可用主机地址范围：200.10.0.1 到 200.10.0.254
  2. 主机2：
    • IP地址：200.10.10.10
    • 子网掩码：255.255.0.0（即 /16）
    • 网络地址：200.10.0.0
    • 广播地址：200.10.255.255
    • 可用主机地址范围：200.10.0.1 到 200.10.255.254

尽管两台主机的网络地址相同（200.10.0.0），但由于它们的子网掩码不同，它们实际上位于不同的子网中：

  • 主机1 在 200.10.0.0/24 子网。
  • 主机2 在 200.10.0.0/16 子网。

具体判断逻辑

  1. 发送方主机用自己的IP地址和子网掩码按位与：
    • 这一步用于确定发送方主机所在的网络地址。
  2. 发送方主机用目的主机的IP地址和自己的子网掩码按位与：
    • 这一步用于确定目的主机所在的网络地址。
  3. 比较两个结果：
    • 如果两个结果相同，说明目的主机与发送方主机在同一网络段，可以直接发送消息。
    • 如果两个结果不同，说明目的主机与发送方主机不在同一网络段，需要通过网关（路由器）发送消息。

因为子网掩码定义了网络地址和主机地址的边界，故在同一个网络段内的主机通常会使用相同的子网掩码

6 公网 IP 和 私网 IP

6.1 概览

为了缓解公网IP地址不足的问题，IPv4标准定义了三个私有IP地址范围，这些地址只能在内部网络中使用，不会与互联网上的任何其他设备冲突。

  • 公网IP地址：用于广域网，每个地址在全球范围内唯一。可以从互联网上的任何地方路由到这些IP地址，允许外部设备访问。
  • 私网IP地址：用于局域网，可以在不同网络中重复使用。私网IP地址不能直接从互联网路由到，只能在内部网络中使用。

A类私有IP地址

  • 网络地址：10.0.0.0/8
  • 可用主机范围：10.0.0.1 到 10.255.255.254

B类私有IP地址

  • 网络地址：172.16.0.0/12
  • 可用主机范围：172.16.0.1 到 172.31.255.254

C类私有IP地址

  • 网络地址：192.168.0.0/16
  • 可用主机范围：192.168.0.1 到 192.168.255.254

6.2 NAT（网络地址转换）

NAT（Network Address Translation，网络地址转换）是一种网络技术，用于将一个IP地址转换为另一个IP地址。NAT最常见于家庭和企业网络中，用于将内部网络中的私有IP地址转换为一个或多个公网IP地址，从而实现内部设备与互联网之间的通信。

  • 作用：NAT允许多个设备共享一个公网IP地址，从而减少对公网IP地址的需求。

6.2.1 工作原理

  • 内部网络中的设备使用私有IP地址。
  • 当内部设备需要访问互联网时，NAT设备（通常是路由器）将私有IP地址转换为公网IP地址。
  • 返回的数据包也会经过NAT设备，将公网IP地址转换回相应的私有IP地址。

6.2.2 工作流程

  1. 内部设备发起请求：
    • 内部设备（如 192.168.1.10）向外部服务器（如 8.8.8.8）发送数据包。
    • 数据包的源IP地址是 192.168.1.10，目标IP地址是 8.8.8.8。
  2. NAT设备进行转换：
    • NAT设备接收到数据包后，将源IP地址 192.168.1.10 替换为公网IP地址 203.0.113.1。
    • 同时，NAT设备记录下这次转换，以便后续的响应数据包可以正确地返回给内部设备。
    • 转换后的数据包的源IP地址变为 203.0.113.1，目标IP地址仍然是 8.8.8.8。
  3. 外部服务器响应：
    • 外部服务器（如 8.8.8.8）接收到数据包后，向 203.0.113.1 发送响应数据包。
    • 响应数据包的源IP地址是 8.8.8.8，目标IP地址是 203.0.113.1。
  4. NAT设备还原转换：
    • NAT设备接收到响应数据包后，查找之前的转换记录，将目标IP地址 203.0.113.1 还原为 192.168.1.10。
    • 还原后的数据包的源IP地址仍然是 8.8.8.8，目标IP地址变为 192.168.1.10。
  5. 内部设备接收响应：
    • 内部设备（如 192.168.1.10）接收到响应数据包，完成通信过程。

6.3 NAPT（网络地址端口转换）

NAPT（Network Address Port Translation，网络地址端口转换）是NAT的一种扩展形式，它不仅转换IP地址，还转换端口号，从而允许多个内部设备共享同一个公网IP地址。NAPT是最常见的NAT形式，广泛应用于家庭和企业网络中。

  • 节省公网IP地址：通过NAPT，多个内部设备可以共享一个公网IP地址，从而大大减少对公网IP地址的需求。

6.3.1 工作原理

  • 内部设备：使用私有IP地址和端口号。
  • NAPT设备（通常是路由器）：负责将私有IP地址和端口号转换为一个公网IP地址和不同的端口号，以及将返回的数据包转换回私有IP地址和端口号。

6.3.2 工作流程

  1. 内部设备发起请求：
    • 内部设备（如 192.168.1.10）向外部服务器（如 8.8.8.8）发送数据包。
    • 数据包的源IP地址是 192.168.1.10，源端口号是 12345，目标IP地址是 8.8.8.8，目标端口号是 80。
  2. NAPT设备进行转换：
    • NAPT设备接收到数据包后，将源IP地址 192.168.1.10 替换为公网IP地址 203.0.113.1。
    • 同时，NAPT设备将源端口号 12345 替换为一个新的端口号，例如 10000。
    • NAPT设备记录下这次转换，以便后续的响应数据包可以正确地返回给内部设备。
    • 转换后的数据包的源IP地址变为 203.0.113.1，源端口号变为 10000，目标IP地址仍然是 8.8.8.8，目标端口号仍然是 80。
  3. 外部服务器响应：
    • 外部服务器（如 8.8.8.8）接收到数据包后，向 203.0.113.1 的端口 10000 发送响应数据包。
    • 响应数据包的源IP地址是 8.8.8.8，源端口号是 80，目标IP地址是 203.0.113.1，目标端口号是 10000。
  4. NAPT设备还原转换：
    • NAPT设备接收到响应数据包后，查找之前的转换记录，将目标IP地址 203.0.113.1 和目标端口号 10000 还原为 192.168.1.10 和 12345。
    • 还原后的数据包的源IP地址仍然是 8.8.8.8，源端口号仍然是 80，目标IP地址变为 192.168.1.10，目标端口号变为 12345。
  5. 内部设备接收响应：
    • 内部设备（如 192.168.1.10）接收到响应数据包，完成通信过程。

2 IP 协议

IP（Internet Protocol，互联网协议）是网络层的核心协议，负责在不同网络之间传输数据包。IP协议定义了数据包的格式和传输规则，确保数据包能够从源节点可靠地传输到目的节点。

2.1 IP 协议的首部格式

首部格式的字节范围：20字节~60字节

详解如下

  • 版本（4位）：表示IP协议的版本，通常是4。
  • 头部长度（4位）：表示协议首部长度，乘4才是最终长度。
    • 二进制1111代表十进制15，乘4刚好为60，代表首部最大字节
  • 区分服务（8位）：用于指定数据包的优先级和传输质量。
  • 总长度（16位）：表示整个数据包的长度，以字节为单位。
  • 标识（16位）：用于唯一标识数据包，帮助重组分片。
    • 帧的数据部分不超过1500B,过大的数据包需要分片，再交给链路层，同一数据包的所有分片标识都是一样的
  • 标志（3位）：用于控制分片。
    • 第1个字节为保留位，留着后面用
    • 第2个字节为0，表示允许分片
    • 第3个字节为0，表示是最后一个分片
  • 片偏移（13位）：表示分片在原始数据包中的位置。
  • 生存时间（TTL，8位）：表示数据包在网络中可以经过的最大跳数。
    • 路由器收到待转发的P数据包时，将其首部中的该字段的值减1，若结果不为0就转发，否则就丢弃
  • 协议（8位）：表示上层协议类型，如TCP（6）、UDP（17）等。
    • 指明IP数据包的数据载荷是何种协议数据单元
  • 头部校验和（16位）：用于检测头部的完整性。
    • 用于检测在传输过程中P首部是否出现了差错
  • 源IP地址（32位）：表示发送方的IP地址。
  • 目的IP地址（32位）：表示接收方的IP地址。

3 ARP 协议

ARP（Address Resolution Protocol，地址解析协议）是网络层的一个重要协议，用于将IP地址解析为物理地址（MAC地址）。在局域网中，设备之间通过MAC地址进行通信，而ARP协议则负责在IP地址和MAC地址之间建立映射关系。

3.1 基本概念

  • MAC地址（Media Access Control Address）：
    • 定义：MAC地址是网络设备的唯一物理地址，通常由6个字节（48位）组成，表示为12个十六进制数字，例如 00:A0:C9:14:C8:29。
    • 作用：MAC地址用于在同一局域网（LAN）内的设备之间进行通信。它是数据链路层（OSI模型的第二层）的标识符，确保数据帧能够从一个设备传输到另一个设备。
  • IP地址（Internet Protocol Address）：
    • 定义：IP地址是互联网上每个设备的唯一标识符，用于在网络中定位设备。
    • 版本：
      • IPv4：32位地址，表示为四个十进制数字，每个数字范围从0到255，例如 192.168.1.1。
      • IPv6：128位地址，表示为八组四位十六进制数字，例如 2001:0db8:85a3:0000:0000:8a2e:0370:7334。
    • 作用：IP地址用于在不同网络之间传输数据包。它是网络层（OSI模型的第三层）的标识符，确保数据包能够从源节点传输到目的节点。

三者联系：

  • MAC地址用于局域网内的数据帧传输，确保数据帧能够从一个设备传输到另一个设备；
  • IP地址用于不同网络之间的数据包传输，确保数据包能够从源节点传输到目的节点。
  • ARP协议在这两者之间建立了桥梁，将IP地址解析为MAC地址，使得数据能够在局域网内正确传输。

3.2 为什么需要MAC地址

3.2.1 局域网内的通信

  • 数据链路层：在局域网（LAN）内，设备之间直接通信时使用的是MAC地址。MAC地址是数据链路层的标识符，确保数据帧能够从一个设备传输到另一个设备。
  • 以太网帧：以太网帧是数据链路层的数据单元，包含源MAC地址和目的MAC地址。这些地址用于在局域网内识别发送方和接收方。

3.2.2 跨网络的通信

  • 网络层：当数据需要跨多个网络传输时，IP地址用于在网络层进行路由。路由器根据IP地址选择最佳路径，将数据包从一个网络转发到另一个网络。
  • ARP协议：在每个网络内部，路由器或主机需要将IP地址解析为MAC地址，以便在局域网内进行实际的数据传输。ARP协议就是用来完成这个任务的。

3.2.3 具体流程

  1. 数据包的生成：
    • 应用层：生成数据。
    • 传输层：将数据封装成段（TCP/UDP）。
    • 网络层：将段封装成IP数据包，添加源IP地址和目的IP地址。
    • 数据链路层：将IP数据包封装成数据帧，添加源MAC地址和目的MAC地址。
  2. 路由选择：
    • 路由器：根据IP地址和路由表选择最佳路径，将数据包从一个网络转发到另一个网络。
  3. 局域网内的传输：
    • ARP请求：如果路由器或主机不知道目的设备的MAC地址，它会发送一个ARP请求广播，询问谁拥有目标IP地址。
    • ARP响应：目标设备收到ARP请求后，会发送一个ARP响应，告知自己的MAC地址。
    • 数据帧传输：路由器或主机将IP数据包封装成以太网帧，使用目的设备的MAC地址进行传输。

3.3 ARP协议的工作原理

3.3.1 ARP请求

  • 目的：当一个设备需要与另一个设备通信时，它需要知道对方的MAC地址。如果它不知道对方的MAC地址，它会发送一个ARP请求广播。
  • 格式：
    • 硬件类型：表示使用的硬件类型，例如以太网（1）。
    • 协议类型：表示使用的协议类型，例如IPv4（0x0800）。
    • 硬件地址长度：表示硬件地址的长度，以字节为单位（6字节）。
    • 协议地址长度：表示协议地址的长度，以字节为单位（4字节）。
    • 操作类型：表示操作类型，1表示ARP请求，2表示ARP响应。
    • 发送方硬件地址：发送方的MAC地址。
    • 发送方协议地址：发送方的IP地址。
    • 目标硬件地址：目标方的MAC地址（未知，通常为全0）。
    • 目标协议地址：目标方的IP地址。

3.3.2 ARP响应

  • 目的：当某个设备收到ARP请求并且知道目标IP地址对应的MAC地址时，它会发送一个ARP响应单播。
  • 格式：
    • 硬件类型：表示使用的硬件类型，例如以太网（1）。
    • 协议类型：表示使用的协议类型，例如IPv4（0x0800）。
    • 硬件地址长度：表示硬件地址的长度，以字节为单位（6字节）。
    • 协议地址长度：表示协议地址的长度，以字节为单位（4字节）。
    • 操作类型：表示操作类型，2表示ARP响应。
    • 发送方硬件地址：发送方的MAC地址。
    • 发送方协议地址：发送方的IP地址。
    • 目标硬件地址：目标方的MAC地址。
    • 目标协议地址：目标方的IP地址。

    • 设备A现在可以使用设备B的MAC地址进行通信。

3.5 ARP缓存

  • 定义：ARP缓存是一个临时存储IP地址和MAC地址映射关系的表，用于加快后续的通信。
  • 作用：减少ARP请求的次数，提高网络效率。
  • 生命周期：ARP缓存中的条目有一定的生存时间（TTL），过期后会被删除。

4 ICMP 协议

ICMP（Internet Control Message Protocol，互联网控制消息协议）是TCP/IP协议族中的一个重要协议，主要用于在网络设备之间传递控制消息和错误报告。ICMP协议帮助诊断网络问题，并提供了网络设备之间通信的反馈机制。

  • 定义：ICMP协议是IP协议的一个补充协议，用于在网络设备之间传递控制消息和错误报告。
  • 作用：ICMP协议提供网络设备之间的通信反馈机制，帮助诊断网络问题和提高网络的可靠性。常见的ICMP报文包括回声请求、回声应答、目的地不可达、时间超过等。

4.1 基本概念

4.1.1 ICMP报文

  • 定义：ICMP报文是ICMP协议的基本单元，用于在网络设备之间传递控制消息和错误报告。
  • 结构：
    • 类型（Type）：8位字段，表示ICMP报文的类型，例如回声请求、回声应答、目的地不可达等。
    • 代码（Code）：8位字段，表示ICMP报文类型的子类型或附加信息。
    • 校验和（Checksum）：16位字段，用于检测ICMP报文在传输过程中是否发生错误。
    • 数据（Data）：可变长度字段，包含具体的消息内容，例如回声请求的数据或错误报告的具体信息。

4.1.2 ICMP报文的封装过程

假设一个ICMP回声请求（Ping）报文从源节点 192.168.1.10 发送到目的节点 192.168.1.11。

  1. 生成ICMP回声请求报文：
    • 类型（Type）：8（回声请求）。
    • 代码（Code）：0。
    • 校验和（Checksum）：计算后的值。
    • 标识符（Identifier）：随机生成的值。
    • 序列号（Sequence Number）：递增的值。
    • 数据（Data）：可选的自定义数据，例如时间戳。
  2. 封装成IP数据报：
    • IP头部
    • 数据载荷：ICMP回声请求报文。
  3. 发送IP数据报：
    • 源节点 192.168.1.10 将封装好的IP数据报发送到目的节点 192.168.1.11。
  4. 接收和处理：
    • 目的节点 192.168.1.11 接收到IP数据报，解封装出ICMP回声请求报文。
    • 目的节点生成ICMP回声应答报文，封装成IP数据报，发送回源节点 192.168.1.10。

4.2 常见的ICMP报文类型

4.2.1 询问报文

回声请求（Echo Request）和回声应答（Echo Reply）
  • 用途：用于测试网络连通性，通常通过 ping 命令实现。
  • 类型：
    • 回声请求：类型值为8。
    • 回声应答：类型值为0。
  • 代码：通常为0。
  • 数据：包含一个标识符（Identifier）和序列号（Sequence Number），用于匹配请求和应答。

4.2.2 差错报告报文

tracert（在Windows中）或traceroute（在Unix/Linux中）命令用于探测IP数据报从源主机到达目的主机所经过的路由器。

这个命令通过发送一系列带有不同TTL（Time to Live，生存时间）值的ICMP报文来实现这一功能。

当数据报的TTL值减为0时，路由器会丢弃该数据报并返回一个ICMP“时间超过”（Time Exceeded）错误报文。

通过这种方式，tracert命令可以逐步确定每个路由器的IP地址。

目的地不可达（Destination Unreachable）
  • 用途：通知发送方目标地址无法到达。
  • 类型：类型值为3。
  • 代码：
    • 0：网络不可达。
    • 1：主机不可达。
    • 2：协议不可达。
    • 3：端口不可达。
    • 4：需要进行分片但设置了DF（Don’t Fragment）标志。
    • 5：源路由失败。
    • 6：目的地网络未知。
    • 7：目的地主机未知。
    • 8：源主机隔离。
    • 9：通信被管理禁止。
    • 10：主机先前提到的问题。
    • 11：主机的主机部分不可达。
    • 12：通信因主机先前提到的问题而被管理禁止。
  • 数据：包含引起错误的IP数据包的前8个字节。
时间超过（Time Exceeded）
  • 用途：通知发送方数据包的TTL（Time To Live，生存时间）值已耗尽。
  • 类型：类型值为11。
  • 代码：
    • 0：TTL超时。
    • 1：重组超时。
  • 数据：包含引起错误的IP数据包的前8个字节。

路由器

  • 路由器是网络设备，用于在不同的网络之间转发数据包。路由器通过分析数据包的目的IP地址，决定将数据包转发到哪个网络接口。
  • 路由器维护一个路由表，其中包含到达各个网络的路径信息。路由表中的条目可以是静态配置的或通过动态路由协议自动学习的。

路由器的工作原理

  1. 接收数据包：
    • 路由器从某个网络接口接收数据包。
  2. 解析IP头部：
    • 路由器解析数据包的IP头部，提取目的IP地址。
  3. 查找路由表：
    • 路由器在路由表中查找与目的IP地址匹配的路由条目。
  4. 选择最佳路径：
    • 路由器根据路由表中的信息选择最佳路径，通常选择具有最长前缀匹配的路由条目。
  5. 转发数据包：
    • 路由器将数据包从相应的网络接口转发出去。

1 路由配置

默认情况下，路由器只知道与它直连的网络的路由信息。

为了使路由器能够转发数据包到非直连网络，需要通过配置静态路由或动态路由协议来告知路由器如何到达这些网络。

  • 静态路由：由网络管理员手动配置，适用于小型、简单的网络环境。
  • 动态路由：通过路由协议自动学习和维护路由表，适用于大型、复杂的网络环境。

3 总结

路由器、集线器和交换机在它们各自工作的OSI模型层次上，能够理解和处理相应的协议头部信息。

  • 路由器：
    • 路由器工作在网络层（OSI模型的第三层），主要用于连接不同的网络，并根据IP地址进行数据包的转发。
    • 它能够执行复杂的路径选择算法，决定最佳的数据传输路径。
    • 路由器还具有防火墙功能，可以过滤不安全的数据流。
  • 集线器：
    • 集线器工作在物理层（OSI模型的第一层），它只是一个简单的信号放大设备，将接收到的所有数据无差别地广播给所有端口。
    • 没有数据过滤或管理功能，所有连接到集线器的设备都在同一个冲突域内，因此网络效率较低。
    • 在现代网络中，集线器已经被交换机取代。
  • 交换机：
    • 交换机工作在数据链路层（OSI模型的第二层），它可以识别MAC地址，实现数据帧的精确转发。
    • 与集线器不同，交换机可以减少网络中的冲突，提高数据传输效率。
    • 交换机支持VLAN划分，可以更灵活地管理网络资源。
