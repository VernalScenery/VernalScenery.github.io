物理层

物理层的标准

物理层涉及许多标准和协议，常见的包括：

  • IEEE 802.3：以太网标准，定义了局域网（LAN）的物理层和数据链路层。
  • IEEE 802.11：无线局域网（WLAN）标准，定义了无线网络的物理层和数据链路层。
  • RS-232：串行通信标准，用于点对点通信。
  • V.35：用于高速数据传输的标准，常用于路由器和调制解调器之间。
  • FDDI：光纤分布式数据接口，用于高速环形网络。
  • SONET/SDH：同步光网络/同步数字体系，用于长距离传输。

物理层的设备

物理层涉及的设备主要包括：

  • 中继器（Repeater）：用于放大和再生信号，延长传输距离。
  • 集线器（Hub）：用于连接多个设备，但不进行智能路由。
  • 调制解调器（Modem）：用于将数字信号转换为模拟信号，以便在电话线等模拟介质上传输。

数据链路层

1 概览

  • 数据链路层负责解决一个网络或一段链路上传输的问题，确保数据帧在相邻节点之间可靠传输。
  • 网络层负责解决跨网络的传输问题，通过路由选择和 IP 地址管理，确保数据包能够从源节点到达目的节点。

2 Mac 地址

MAC 地址（Media Access Control Address）是网络设备在数据链路层的唯一标识符。每个网络接口控制器（NIC）都有一个唯一的 MAC 地址，用于在网络中唯一标识设备。

2.1 MAC 地址的格式

  • 长度：MAC 地址通常是一个 48 位（6 字节）的二进制数。
  • 表示形式：通常以十六进制表示，每两个字节之间用冒号（:）、连字符（-）或不加分隔符表示。例如：
    • 00:1A:2B:3C:4D:5E
    • 00-1A-2B-3C-4D-5E
    • 001A2B3C4D5E

2.2 MAC 地址的类型

  • 单播地址：用于标识单个设备，是最常见的 MAC 地址类型。
  • 广播地址：所有位均为 1（FF:FF:FF:FF:FF:FF），用于向网络中的所有设备发送数据。
  • 多播地址：用于向一组特定的设备发送数据，多播地址的最高位为 1。
  • 全局唯一地址：由厂商分配，保证全球唯一。
  • 本地管理地址：由管理员手动配置，可能不是全球唯一的。

2.3 MAC 地址的作用

  • 数据链路层通信：在同一个局域网内，MAC 地址用于标识发送方和接收方。
  • ARP 协议：地址解析协议（ARP）用于将 IP 地址解析为 MAC 地址，以便在局域网内进行通信。
  • 帧的封装：数据链路层将数据包封装成帧时，会在帧头中包含源 MAC 地址和目的 MAC 地址。

2.4 MAC 地址的局限性

  • 局域网内有效：MAC 地址主要用于局域网内的通信，跨网络的通信需要依赖 IP 地址。
  • 不可路由：路由器在转发数据包时，只使用 IP 地址，不使用 MAC 地址。

2.5  MAC 地址和 IP 地址的关系

  • 层次关系：
    • 数据链路层：MAC 地址用于同一网络内的设备通信。
    • 网络层：IP 地址用于不同网络间的设备通信。
  • ARP 协议：
    • ARP（Address Resolution Protocol）：用于将 IP 地址解析为 MAC 地址。当一个设备需要向另一个设备发送数据时，它会使用 ARP 协议查询目标设备的 MAC 地址。
    • 工作流程：
      1. 发送 ARP 请求：源设备广播一个 ARP 请求，询问目标 IP 地址对应的 MAC 地址。
      2. 接收 ARP 回应：目标设备收到 ARP 请求后，回应一个包含其 MAC 地址的 ARP 回应。
      3. 缓存 ARP 表：源设备将目标设备的 IP 地址和 MAC 地址的对应关系存储在 ARP 缓存表中，以便后续通信。
  • RARP 协议：
    • RARP（Reverse Address Resolution Protocol）：用于将 MAC 地址解析为 IP 地址。在某些情况下，设备只知道自己的 MAC 地址，需要获取 IP 地址。
    • 工作流程：
      1. 发送 RARP 请求：设备广播一个 RARP 请求，询问其 MAC 地址对应的 IP 地址。
      2. 接收 RARP 回应：RARP 服务器收到请求后，回应一个包含 IP 地址的 RARP 回应。

注意：

传输数据前，会先发一个 ARP 协议的包，用于将 IP 地址解析为 MAC 地址

3 链路层的帧格式

以太网（Ethernet）帧格式

  • 前导码（Preamble）：56 比特（7 字节），用于接收方同步时钟。
  • 帧起始符（Start of Frame Delimiter, SFD）：8 比特（1 字节），表示帧的开始。
  • 目的地址（Destination Address）：48 比特（6 字节），接收方的 MAC 地址。
  • 源地址（Source Address）：48 比特（6 字节），发送方的 MAC 地址。
  • 类型/长度（Type/Length）：16 比特（2 字节），表示上层协议类型或帧的长度。
  • 数据（Data）：46 到 1500 字节，上层传递下来的数据包。
  • 帧校验序列（Frame Check Sequence, FCS）：32 比特（4 字节），用于错误检测。

4.1 以太网（Ethernet）

交叉线、集线器、交换机

1 交叉线

交叉线（Crossover Cable）是一种特殊的网络线缆，主要用于直接连接两台相同类型的网络设备，如两台计算机、两台交换机或两台路由器。传统的直通线（Straight-Through Cable）用于连接不同类型或层次的设备，而交叉线则用于连接相同类型的设备。

1.1 交叉线的工作原理

1.1.1 直通线 vs 交叉线

  • 直通线：
    • 用于连接不同类型的设备，如计算机和交换机、计算机和路由器。
    • 两端的引脚顺序相同，即 1 对 1、2 对 2、3 对 3 等。
    • 例如，计算机的网卡（NIC）通常使用直通线连接到交换机的端口。
  • 交叉线：
    • 用于连接相同类型的设备，如两台计算机、两台交换机或两台路由器。
    • 一端的发送引脚连接到另一端的接收引脚，反之亦然。
    • 例如，两台计算机直接连接时，需要使用交叉线。

1.2 交叉线的应用场景

  • 两台计算机直接连接：在没有交换机或路由器的情况下，使用交叉线可以直接连接两台计算机，实现文件共享、打印共享等功能。
  • 两台交换机直接连接：在需要连接两个交换机时，使用交叉线可以避免信号冲突，确保数据正常传输。
  • 两台路由器直接连接：在需要连接两个路由器时，使用交叉线可以实现网络之间的通信

2 集线器

集线器（Hub）是一种网络设备，用于连接多个网络设备，使它们能够在同一个物理网络中通信。集线器工作在 OSI 模型的物理层（第 1 层），它的主要功能是将接收到的信号进行放大和转发，而不进行任何智能处理。

2.1 工作原理

  • 信号放大：集线器接收到来自某个端口的信号后，会将信号放大并转发到所有其他端口。
  • 广播域：集线器将所有连接的设备放在同一个广播域内，这意味着任何发送到集线器的数据帧都会被转发到所有其他端口，无论目标设备是否在这些端口中。
  • 冲突域：由于集线器不进行任何智能处理，所有连接的设备都在同一个冲突域内。这意味着在同一时间只能有一个设备发送数据，否则会发生冲突。

2.2 缺点

  • 性能低下：由于所有设备都在同一个冲突域内，网络性能会随着连接设备数量的增加而显著下降。
  • 安全性差：所有数据帧都会被转发到所有端口，这可能导致数据泄露。
  • 不支持 VLAN：集线器不支持虚拟局域网（VLAN）技术，无法实现更细粒度的网络划分和管理。

3 交换机

交换机（Switch）是一种网络设备，用于连接多个网络设备并在它们之间传输数据。与集线器不同，交换机工作在数据链路层（OSI 模型的第 2 层），能够根据 MAC 地址智能地转发数据帧，从而提高网络性能和安全性

3.1 工作原理

  • MAC 地址表：交换机维护一个 MAC 地址表，记录每个端口连接的设备的 MAC 地址。
  • 学习过程：当交换机接收到数据帧时，它会记录数据帧的源 MAC 地址和进入的端口号，更新 MAC 地址表。
  • 转发过程：交换机根据数据帧的目标 MAC 地址查找 MAC 地址表，确定目标设备所在的端口，并将数据帧转发到该端口。
  • 广播：如果目标 MAC 地址未知，交换机会将数据帧广播到所有端口（除了接收数据帧的端口）。
  • 冲突域：每个端口都是一个独立的冲突域，减少了网络中的冲突，提高了网络性能。

整体流程：交换机先接收信息，进行解析，然后更新 MAC 地址表，之后再转发

假设有一台二层交换机，连接了三台设备 A、B 和 C，分别连接到端口 1、2 和 3。设备 A 向设备 B 发送一个数据帧。
