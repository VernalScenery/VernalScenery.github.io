1 什么是Promise?

Promise 是 JavaScript 中用于处理异步操作的一种对象。它代表了一个异步操作的最终完成（或失败）及其结果值。Promise 提供了一种更清晰、更可控的方式来处理异步操作，避免了传统的回调地狱（Callback Hell）问题。

1.1 Promise 的执行过程

  1. 创建 Promise 对象:
    • 使用 new Promise 创建一个新的 Promise 对象。
    • 构造函数接受一个函数作为参数，这个函数称为 executor。
  2. Executor 的执行:
    • Executor 函数会被立即执行。
    • Executor 函数接收两个参数：resolve 和 reject，这两个参数也是函数。
  3. resolve 和 reject 的作用:
    • resolve: 当异步操作成功时调用，将 Promise 的状态从 pending 转为 fulfilled，并传递成功的值。
    • reject: 当异步操作失败时调用，将 Promise 的状态从 pending 转为 rejected，并传递失败的原因。
  4. then 和 catch 方法:
    • then 方法用于指定成功和失败的回调函数。
      • 第一个参数是成功时的回调函数。
      • 第二个参数是失败时的回调函数。
    • catch 方法用于捕获错误，相当于 then(null, reject)。

  2. 不可变性:

  3. 链式调用:

2 resolve不同值的区别

resolve 函数在 Promise 的 executor 函数中被调用，用于将 Promise 的状态从 pending 转为 fulfilled。根据 resolve 函数传入的不同类型的参数，Promise 的行为也会有所不同。以下是三种常见的情况：

2.1 传入普通值或对象

  • 如果 resolve 传入一个普通的值或对象，这个值会作为 then 回调函数的参数。
  • Promise 的状态会从 pending 转为 fulfilled。

2.2 传入另一个 Promise

  • 如果 resolve 传入的是另一个 Promise，那么原 Promise 的状态会由传入的 Promise 来决定。
  • 传入的 Promise 如果成功（fulfilled），原 Promise 也会成功，并且其 then 回调函数会接收到传入 Promise 的成功值。
  • 传入的 Promise 如果失败（rejected），原 Promise 也会失败，并且其 catch 或 then 的第二个回调函数会接收到传入 Promise 的失败原因。

2.3 传入一个实现了 then 方法的对象（Thenable 对象）

  • 如果 resolve 传入的是一个实现了 then 方法的对象（称为 Thenable 对象），那么会执行该 then 方法。
  • then 方法的结果会决定原 Promise 的状态。
  • 如果 then 方法调用 resolve，原 Promise 会成功，并且其 then 回调函数会接收到 then 方法传递的值。
  • 如果 then 方法调用 reject，原 Promise 会失败，并且其 catch 或 then 的第二个回调函数会接收到 then 方法传递的失败原因。

3 Promise 实例方法(对象方法)

Promise有哪些实例方法

3.1 then 方法的特点

then 方法是 Promise 对象的一个重要方法，用于处理 Promise 的成功和失败情况。then 方法实际上定义在 Promise 的原型上，即 Promise.prototype.then。

3.1.1 接受两个参数

then方法接受两个参数：

  • onFulfilled：可选参数，当 Promise 状态变为 fulfilled 时调用的回调函数。该函数接收 Promise 成功的结果作为参数。
  • onRejected：可选参数，当 Promise 状态变为 rejected 时调用的回调函数。该函数接收 Promise 失败的原因作为参数。

3.1.2 同一个 Promise 可以被多次调用 then 方法

  • 当 Promise 的状态变为 fulfilled 或 rejected 时，所有注册的 then 方法的回调函数都会被调用。

3.1.3 then 方法的回调函数可以有返回值

then 方法本身返回一个新的 Promise，这使得 then 方法可以进行链式调用。

3.3.1 返回普通值

如果 then 方法的回调函数返回一个普通值（如数值、字符串、普通对象、undefined），这个值会被作为新的 Promise 的 resolve 值。

3.3.2 返回 Promise

如果 then 方法的回调函数返回一个 Promise，新的 Promise 的状态会由返回的 Promise 决定。

3.3.3 返回 thenable 对象

如果 then 方法的回调函数返回一个实现了 then 方法的对象（thenable 对象），新的 Promise 的状态会由该 then 方法的结果决定。

3.2 catch 方法的特点

catch 方法是 Promise 对象上的一个方法，用于处理 Promise 的失败情况。catch 方法实际上定义在 Promise 的原型上，即 Promise.prototype.catch。

catch 方法接受一个参数，即失败的回调函数

  • onRejected：当 Promise 状态变为 rejected 时调用的回调函数。该函数接收 Promise 失败的原因作为参数。

3.2.1 一个 Promise 的 catch 方法可以被多次调用

  • 当 Promise 的状态变为 rejected 时，所有注册的 catch 方法的回调函数都会被调用。

3.2.2 catch 方法的回调函数可以有返回值

catch 方法本身返回一个新的 Promise， 这使得 catch 方法可以进行链式调用

其返回值和 then一样，分为如下三种情况：

4.2.1 返回普通值

如果 catch 方法的回调函数返回一个普通值（如数值、字符串、普通对象、undefined），这个值会被作为新的 Promise 的 resolve 值。

4.2.2 返回 Promise

如果 catch 方法的回调函数返回一个 Promise，新的 Promise 的状态会由返回的 Promise 决定。

4.2.3 返回 thenable 对象

如果 catch 方法的回调函数返回一个实现了 then 方法的对象（thenable 对象），新的 Promise 的状态会由该 then 方法的结果决定。

3.2.3 捕获异常

当 executor 抛出异常时，也会调用错误（拒绝）捕获的回调函数

3.3 finally方法的特点

finally是在ES9（ES2018）中新增的一个特性：表示无论Promise对象无论变成fulfilled还是reject状态，最终都会执行

  • 无论 Promise 是 resolve 还是 reject，finally 方法都会被执行。
  • finally 方法不接收任何参数，因为它只是用来执行一些清理或收尾工作的代码。

4 Promise 静态方法(类方法)

4.1 Promise.resolve

Promise.resolve的用法相当于new Promise，并且执行resolve操作

resolve参数的3种情况：

  • 参数是一个普通的值或者对象
  • 参数本身是Promise
  • 参数是一个thenable

4.2 Promise.reject

reject方法类似于resolve方法，只是会将Promise对象的状态设置为reject状态

注意：Promise.reject传入的参数无论是什么形态，都会直接作为reject状态的参数传递到catch的

4.3 Promise.all

Promise.all 是 Promise 类的一个静态方法，用于将多个 Promise 实例包装成一个新的 Promise 实例。

新的 Promise 实例在以下两种情况下会改变状态：

4.3.1 所有传入的 Promise 实例都变为 fulfilled 状态

  • 新的 Promise 实例会变为 fulfilled 状态，并且其结果是一个数组，数组中的元素是每个 Promise 实例的结果，顺序与传入的 Promise 实例顺序一致。

注意：

  • 数组中'aaaa' 是一个普通值，Promise.all 会将其视为一个已经 resolve 的 Promise，结果为 'aaaa'。

4.3.2 任意一个传入的 Promise 实例变为 rejected 状态

  • 新的 Promise 实例会立即变为 rejected 状态，并且其结果是第一个被 rejected 的 Promise 实例的结果。

4.4 Promise.allSettled

Promise.allSettled 是 Promise 类的一个静态方法，用于将多个 Promise 实例包装成一个新的 Promise 实例。

新的 Promise 实例在以下情况下会改变状态：

  1. 所有传入的 Promise 实例都变为 settled 状态：
    • settled 状态包括 fulfilled 和 rejected。
    • 新的 Promise 实例会变为 fulfilled 状态，并且其结果是一个数组，数组中的每个元素是一个对象，表示每个 Promise 实例的结果。

结果数组的结构

  • 对于 fulfilled 的 Promise，结果对象包含 status: 'fulfilled' 和 value 属性。
  • 对于 rejected 的 Promise，结果对象包含 status: 'rejected' 和 reason 属性。

4.5 Promise.race

Promise.race 是 Promise 类的一个静态方法，用于将多个 Promise 实例包装成一个新的 Promise 实例。新的 Promise 实例在以下情况下会改变状态：

  1. 任意一个传入的 Promise 实例变为 settled 状态：
    • settled 状态包括 fulfilled 和 rejected。
    • 新的 Promise 实例会立即变为该 Promise 实例的状态，并且其结果是该 Promise 实例的结果。
  2. 结果：
    • 如果第一个变为 fulfilled，新的 Promise 实例也会变为 fulfilled。
    • 如果第一个变为 rejected，新的 Promise 实例也会变为 rejected。

不论第一个执行完的 Promise实例是resolve, 还是reject，新的 Promise 实例的状态都和它保持一致

4.6 Promise.any

Promise.any 是 Promise 类的一个静态方法，用于将多个 Promise 实例包装成一个新的 Promise 实例。新的 Promise 实例在以下情况下会改变状态：

  1. 任意一个传入的 Promise 实例变为 fulfilled 状态：
    • 新的 Promise 实例会立即变为 fulfilled 状态，并且其结果是该 Promise 实例的结果。
  2. 所有传入的 Promise 实例都变为 rejected 状态：
    • 新的 Promise 实例会变为 rejected 状态，并且其结果是一个 AggregateError 对象，包含所有 Promise 实例的 reject 值。

    • 当前实现不支持链式调用，即 then 方法返回一个新的 Promise 实例。
    • 解决方案：在 then 方法中返回一个新的 HYPromise 实例，并在 resolve 或 reject 时正确处理返回值。

5.4 finally方法的实现

finally 方法的实现目的是无论 Promise 对象最终是被解决（fulfilled）还是被拒绝（rejected），都会执行 finally 中的回调函数。

这个实现通过调用 then 方法，并传入两个回调函数来达到目的：

  • 第一个回调函数在 Promise 成功解决时执行。
  • 第二个回调函数在 Promise 被拒绝时执行。

在这两种情况下，都会调用 onFinally 函数，从而实现了 finally 的功能。

但是这个会有一个问题

  • 第一个 then 方法返回了 'aaaaa'，这个值会被作为第二个 catch 方法的 onFulfilled 回调函数的参数。
  • 但是，第二个 catch 方法没有提供 onFulfilled 回调函数，因此 onFulfilled 为 undefined。
  • 由于 onFulfilled 是 undefined，execFunctionWithCatchError 不会执行任何操作，resolve 也不会被调用。
  • 因此，第二个 catch 方法不会执行任何操作，也不会传递任何值。

由于.catch返回的第二个promise中的resolve 没有被调用，故.catch返回的第二个 promise的状态会一直是 pending，而 .finally是由.then实现的，必须需要 promise状态确定时，才会执行回调函数

5.6.1 all

这里传入的promises是一个由promise组成的数组

  1. 何时执行 resolve:
    • resolve 应该在所有传入的 Promise 都成功（即都变为 fulfilled 状态）时执行。
    • 具体来说，当每个 Promise 成功时，将其结果存储在 values 数组中，并检查 values 数组的长度是否等于传入的 Promise 数组的长度。如果是，则说明所有 Promise 都已成功，此时可以调用 resolve(values)。
  2. 何时执行 reject:
    • reject 应该在任意一个传入的 Promise 失败（即变为 rejected 状态）时立即执行。
    • 具体来说，只要有一个 Promise 调用了其 reject 回调，就应立即调用 reject(err)，并将错误传递出去。

5.6.2 allSettled

注意：

最后返回的 Promise 的结果一定是 fulfilled 的，且传入数组中的所有promise的状态都确定了，体现在代码上即results.length === promises.length

处理每个 Promise 的成功情况

  • 对于每个 Promise，调用其 then 方法。
  • 如果 Promise 成功解决（fulfilled），则将结果对象 { status: PROMISE_STATUS_FULFILLED, value: res } 推入 results 数组。
  • 检查 results 数组的长度是否等于传入的 promises 数组的长度。如果是，则说明所有 Promise 都已解决，调用 resolve(results) 返回结果数组。

处理每个 Promise 的失败情况

  • 如果 Promise 被拒绝（rejected），则将结果对象 { status: PROMISE_STATUS_REJECTED, value: err } 推入 results 数组。
  • 同样检查 results 数组的长度是否等于传入的 promises 数组的长度。如果是，则说明所有 Promise 都已解决，调用 resolve(results) 返回结果数组。

5.7.1 race

  • 对于每个 Promise，调用其 then 方法。
  • then 方法接受两个回调函数作为参数：一个用于处理成功的情况，另一个用于处理失败的情况。
  • 这里直接将 resolve 和 reject 作为 then 方法的参数传递，这意味着：
    • 如果 Promise 成功解决（fulfilled），则调用 resolve 并传递成功的结果。
    • 如果 Promise 被拒绝（rejected），则调用 reject 并传递错误信息。
  • 行为:
    • 如果传入的 Promise 中有一个成功解决，则新的 Promise 也会成功解决，并返回该成功的结果。
    • 如果传入的 Promise 中有一个被拒绝，则新的 Promise 也会被拒绝，并返回该错误信息。

5.7.2 any

  • 对于每个 Promise，使用 then 方法注册成功和失败的回调。
  • 成功回调直接调用 resolve，这意味着只要有一个 Promise 成功解决，整个 any 方法就会立即调用 resolve。
  • 失败回调将错误原因推入 reasons 数组，并检查 reasons 数组的长度是否等于 promises 数组的长度。
  • 如果 reasons 数组的长度等于 promises 数组的长度，说明所有 Promise 都已失败，此时调用 reject 并传递一个包含所有错误原因的 AggregateError。

AggregateError 是 JavaScript 中的一个内置错误对象，用于表示多个错误情况的集合。它通常在需要同时处理多个错误时使用，特别是在处理多个 Promise 的情况下。AggregateError 的主要用途是将多个错误封装在一个单一的对象中，以便更方便地进行错误处理和调试。
