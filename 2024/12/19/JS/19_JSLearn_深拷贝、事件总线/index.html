深拷贝

  • 引入的赋值：指向同一个对象，相互之间会影响；
  • 对象的浅拷贝：只是浅层的拷贝，内部引入对象时，依然会相互影响；
  • 对象的深拷贝：两个对象不再有任何关系，不会相互影响；

1 使用 JSON.parse(JSON.stringify(obj)) 的局限性

1.1 局限性

  1. 无法处理函数:
    • JSON.stringify 会自动忽略对象中的函数属性。
    • 因此，在深拷贝过程中，函数属性会被丢失。
  2. 无法处理 Symbol 类型:
    • JSON.stringify 会忽略对象中的 Symbol 键。
    • 因此，Symbol 类型的键及其对应的值不会出现在深拷贝的结果中。
  3. 无法处理循环引用:
    • JSON.stringify 在遇到循环引用时会抛出错误。
    • 循环引用会导致无限递归，最终引发栈溢出错误。

1.2 原因

JSON.parse(JSON.stringify(obj)) 存在的局限性主要是因为 JSON 格式的限制。

JSON（JavaScript Object Notation）是一种轻量级的数据交换格式，它具有以下几个特点和限制：

  1. 数据类型支持有限:
    • JSON 支持的基本数据类型包括字符串（String）、数字（Number）、布尔值（Boolean）、数组（Array）、对象（Object）和 null。
    • JSON 不支持 JavaScript 中的函数、Symbol、Date、RegExp 等特殊类型。
  2. 不支持循环引用:
    • JSON 要求数据必须是树形结构，不允许出现循环引用。
    • 当尝试序列化包含循环引用的对象时，JSON.stringify 会抛出错误，提示“Converting circular structure to JSON”。|
  3. 键只能是字符串:
    • JSON 对象的键必须是字符串。
    • 因此，即使对象中有 Symbol 类型的键，JSON.stringify 也会忽略这些键。

解决办法

  1. 初始化 WeakMap:
    • 在调用 deepClone 函数时，传递一个空的 WeakMap 实例
    • 这里不能将WeakMap 实例放在全局，不然谁调用，用的都是一个实例
  2. 检查并记录已克隆的对象:
    • 在每次递归调用开始时，首先检查当前对象是否已经在 WeakMap 中。
    • 如果存在，则直接返回对应的克隆对象，避免进一步递归。
  3. 记录新克隆的对象:
    • 如果当前对象尚未被克隆，则创建一个新的对象，并将其与原始对象一起存储在 WeakMap 中。

事件总线

认识事件总线

事件总线是一个中央集线器，负责接收来自不同来源的事件，并将这些事件分发给所有注册了相应事件处理程序的组件。这种机制类似于现实生活中的广播系统，任何人都可以发送消息，任何感兴趣的人都可以接收并处理这些消息。

主要功能

  1. 事件发布（Emit/Publish）:
    • 组件可以通过事件总线发布事件。发布的事件通常包含事件名称和可选的数据负载（payload）。
  2. 事件订阅（Subscribe/Listen）:
    • 组件可以订阅感兴趣的事件，并提供相应的回调函数来处理这些事件。
  3. 事件取消订阅（Unsubscribe）:
    • 当组件不再需要处理某些事件时，可以取消订阅，以减少不必要的性能开销。

使用场景

  • 跨组件通信: 在复杂的单页应用（SPA）中，组件之间的层级关系可能非常复杂，直接通信会导致代码难以维护。事件总线可以帮助组件之间进行解耦的通信。
  • 状态管理: 在某些情况下，事件总线可以用作简单的状态管理工具，特别是在小型项目中。
  • 全局事件处理: 适用于需要处理全局事件的场景，如键盘事件、窗口大小变化等。
