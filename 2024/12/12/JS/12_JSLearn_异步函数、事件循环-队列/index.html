2 异步函数 async function

async 关键字用于声明一个异步函数，这种函数总是返回一个 Promise，并且可以在其内部使用 await 关键字来暂停执行，直到一个 Promise 被解决或拒绝。

async异步函数可以有很多中写法

2.1 异步函数的执行流程

异步函数的内部代码执行过程和普通的函数是一致的，默认情况下也是会被同步执行

注意：

  • 异步函数有返回值时 ，和普通函数会有区别
  • 如果我们在async中抛出了异常，那么程序它并不会像普通函数一样报错，而是会作为Promise的reject来传递

2.2 异步函数的返回值

  • 如果返回值不是 Promise 或 thenable，则会被包裹到 Promise.resolve 中。
  • 如果返回值是 Promise，则 async 函数返回的 Promise 的状态由返回的 Promise 决定。
  • 如果返回值是 thenable 对象，则 JavaScript 会将其转换为 Promise 并调用其 then 方法。

2.3 异步函数的异常处理

  • 如果在 async 函数内部抛出异常，异常会被捕获并作为 Promise 的 reject 来传递。可以通过 catch捕获

3 await关键字

async函数另外一个特殊之处就是可以在它内部使用await关键字

await 关键字是 async 函数中的一个重要特性，用于等待一个异步操作的结果。以下是 await 关键字的主要特点：

  1. 等待 Promise 完成：
    • await 后面通常会跟上一个表达式，这个表达式会返回一个 Promise。
    • await 会 暂停async 函数的执行，直到 Promise 的状态变为 fulfilled 或 rejected。
    • 如果 Promise 的状态变为 fulfilled，await 会返回 Promise 的结果。
    • 如果 Promise 的状态变为 rejected，await 会抛出一个异常，这个异常会被 catch 捕获。
  2. 处理普通值：
    • 如果 await 后面是一个普通的值（而不是 Promise 或 thenable 对象），await 会直接返回这个值，不会暂停函数的执行。
  3. 处理 thenable 对象：
    • 如果 await 后面是一个 thenable 对象（即具有 then 方法的对象），await 会调用这个对象的 then 方法，并根据 then 方法的回调来决定后续的值。
  4. 异常处理：
    • 如果 await 后面的 Promise 被 reject，await 会抛出一个异常，这个异常会被 catch 捕获。
    • 如果没有 catch 处理，这个异常会导致整个 async 函数返回一个 rejected 的 Promise。

1、等待 Promise完成

  • await 关键字会暂停 foo 函数的执行，直到 requestData 返回的 Promise 完成。

2、处理普通值

  • await 123 会直接返回 123，不会暂停函数的执行。

3、处理 thenable 对象

  • await { then: function(resolve, reject) { resolve("abc") } } 会调用 then 方法，并传入 resolve 回调。
  • resolve("abc") 会使 await 返回 "abc"。

4、异常处理

  • await requestData() 会暂停 foo 函数的执行，等待 Promise 的状态变化。
  • 2 秒后，Promise 被 reject 为 1111，await 抛出异常 1111。console.log("res1:", res1)便不会执行了

事件循环-队列

1 进程和线程

2 浏览器中的JavaScript线程

  • 大多数浏览器都是多进程
  • JS 是单线程的
  • JS 中真正耗时的操作，实际上并不是由 JS 运行的单线程在执行的，而是其他线程来完成这个耗时的操作
  • 而当其他线程完成这个耗时的操作后，JS 线程如何调用其结果，就需要用到事件循环机制

3 事件循环机制

  1. 单线程模型：
    • JavaScript 是单线程的，这意味着同一时间只能执行一段代码。
    • 单线程模型有助于避免复杂的线程同步问题。
  2. 耗时操作的处理：
    • 大多数耗时操作（如网络请求、文件读取、定时器等）不是由 JavaScript 引擎的主线程执行的。
    • 这些操作通常由浏览器或其他线程（如 Web Workers）来执行。
  3. 事件循环的作用：
    • 当耗时操作完成时，相应的回调函数会被放入任务队列中。
    • 事件循环负责从任务队列中取出任务并执行。
    • 任务队列分为两种主要类型：
      • 宏任务队列（macrotask queue）：包括 setTimeout、setInterval、setImmediate（Node.js）、I/O 操作、UI 渲染等。
      • 微任务队列（microtask queue）：包括 Promise 的回调、MutationObserver、process.nextTick（Node.js）等。
  4. 事件循环的工作流程：
    • 执行栈为空时，事件循环会先检查微任务队列是否有任务，如果有则依次执行。
    • 微任务队列为空后，事件循环会从宏任务队列中取出一个任务执行。
    • 重复上述步骤，直到所有任务执行完毕。

案例1

  1. setTimeout 的调用时机：
    • 当 JavaScript 引擎执行到 setTimeout(function() { console.log("setTimeout callback"); }, 1000); 这一行时，它会立即返回，并将回调函数注册到浏览器的定时器系统中。
    • 这意味着 setTimeout 函数本身不会阻塞后续代码的执行。
  2. 回调函数的入队时机：
    • 浏览器的定时器系统会在 1 秒后将回调函数放入宏任务队列中。
    • 这个过程是由浏览器的内部机制管理的，与 JavaScript 主线程的执行无关。
  3. 事件循环的处理：
    • 当主线程上的所有同步代码执行完毕后，若微任务队列中的任务为空，事件循环会检查宏任务队列。
    • 如果宏任务队列中有任务，事件循环会取出一个任务并执行。
    • 在这个例子中，setTimeout 的回调函数会在 1 秒后被放入宏任务队列，然后在同步代码执行完毕后被执行。

总结

  1. 宏任务（Macrotask）：
    • setTimeout 是一个典型的宏任务，它的回调函数会在指定的时间后被放入宏任务队列。
    • 宏任务包括：setTimeout、setInterval、I/O、UI 渲染 等。
  2. 微任务（Microtask）：
    • queueMicrotask 和 Promise.then 是典型的微任务，它们的回调函数会被立即放入微任务队列，并在当前任务完成后立即执行。
    • 微任务包括：process.nextTick（Node.js）、MutationObserver、Promise.then 等。
  3. 事件循环（Event Loop）：
    • 事件循环负责管理和调度宏任务和微任务的执行。
    • 每次主线程上的所有同步代码执行完毕后，事件循环会先处理微任务队列中的所有任务，然后再检查宏任务队列。
