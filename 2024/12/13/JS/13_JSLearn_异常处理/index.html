13_JSLearn_异常处理

1 throw关键字

在 JavaScript 中，throw 关键字用于抛出异常。你可以抛出任何类型的值作为异常信息，包括基本数据类型和对象类型。

1.1 throw关键字后面可以跟上的类型

基本数据类型：

  • number
  • string
  • boolean
  • null
  • undefined
  • symbol

对象类型：

  • 任何对象都可以作为异常信息，这使得你可以传递更多的信息。
  • 最常见的做法是抛出 Error 对象或其子类的实例

一般都是抛出一个对象类型，为此JavaScript 提供了内置的Error 类及其一些子类

2 Error 类

Error 类是 JavaScript 中所有错误的基础类。当你创建一个 Error 对象时，它可以包含以下属性：

  1. **message**：创建 Error 对象时传入的错误消息。
  2. **name**：错误的名称，默认为 "Error"，但可以通过继承 Error 类来自定义。
  3. **stack**：错误的堆栈跟踪信息，包括函数的调用栈，当直接打印 Error 对象时，通常会显示 stack 属性。

2.1 Error 类的子类

JavaScript 提供了一些内置的 Error 子类，用于处理不同类型的错误：

  1. **RangeError**：当一个值不在允许的范围内时抛出。
  2. **SyntaxError**：当解析代码时发生语法错误时抛出。
  3. **TypeError**：当变量或参数不是预期的类型时抛出。
  4. **ReferenceError**：当引用未声明的变量时抛出。
  5. **URIError**：当使用全局 URI 处理函数时传入无效格式的 URI 时抛出。
  6. **EvalError**：当 eval() 函数使用不当时抛出（在现代 JavaScript 中很少使用）。

3 try...catch来捕获异常

  • try...catch 块可以捕获并处理在 try 块中抛出的异常。
  • 一旦异常被捕获并处理，程序会继续执行 catch 块之后的代码。
  • 这种机制使得程序能够在遇到错误时进行适当的处理，而不是直接终止执行。
